Pillar 1 · Foundation Layer

AI Models and Hosting

Which large language models power SAM-AI, how the platform wraps every model in a consistent internal architecture, and the compliance rules that bind every model call.

Hosted vs. Direct Model

The first decision the platform makes for any given workload: call a hosted third-party provider, or run a model directly under our own infrastructure.

SAM-AI is deliberately model-agnostic. Rather than binding the platform to a single foundation model, the architecture treats the model itself as a swappable component behind a stable internal interface. This gives the platform three practical advantages: it can route different tasks to whichever model performs them best, it can fail over to an alternate provider if one becomes unavailable or degrades, and it can negotiate cost and latency independently of any single vendor's pricing or roadmap.

"Hosted" access means calling a provider's managed API — the provider owns the infrastructure, the scaling, and the underlying weights, and SAM-AI simply sends requests and receives completions. "Direct" access means running or fine-tuning a model on infrastructure the platform controls, trading operational overhead for tighter control over data residency, latency, and customization. The current roster of third-party AI providers spans both modes:

ProviderTypical RoleAccess Mode
Fireworks Fast, cost-efficient inference for open and fine-tuned models Hosted API
ChatGPT5 General-purpose reasoning, drafting, and complex analysis Hosted API
Deepseek High-throughput reasoning and coding tasks at lower cost Hosted or direct deployment
et al. Additional providers evaluated and added as the model landscape evolves Hosted or direct, case by case

Why more than one provider? No single foundation model is best at everything, and pricing and availability shift constantly. Keeping the model layer pluggable means SAM-AI can adopt a better or cheaper model the day it becomes available, without redesigning anything above it.

Core Internal Architecture

Whichever model answers a request, it answers inside the same internal scaffolding — this is what makes SAM-AI feel like one coherent system rather than a collection of raw model calls.

FM System Prompts

Every call to a foundation model is preceded by a system prompt layer that establishes who the AI is, how it should behave, what it should produce, and what it must never do. These four components are configured centrally and applied consistently, so switching the underlying model doesn't change the AI's identity or its guardrails.

Role Definition

Establishes the AI's identity, expertise level, and purpose for the task at hand — telling the model whether it is acting as an analyst, a research assistant, a compliance checker, or another defined role before it processes a single word of the user's request.

Behavior Guidelines

Rules for interaction style, response length, and engagement patterns. These guidelines keep tone and verbosity consistent across every conversation, regardless of which model or provider generates the underlying response.

Response Format

Templates for structured outputs, formatting preferences, and data presentation — from tables and bullet structures to strict JSON schemas — so downstream systems can parse and display AI output reliably.

Constraints & Rules

Boundaries for acceptable responses, safety protocols, and ethical guidelines. This is the layer that stops the model from acting outside its authorized scope, regardless of how a request is phrased.

Session Management

A session is the container for an individual user's interaction with SAM-AI over time. Session management tracks that container from open to close, keeping conversations coherent and cleaning up after they end.

User Sessions

Tracking individual user interactions across multiple conversations, so context and preferences carry forward without the user having to repeat themselves.

Context Persistence

Maintaining conversation state across sessions, so a user can pick up a thread of analysis or research where they left off.

Session Timeout

Managing session lifecycle and cleanup — expiring inactive sessions and releasing resources to keep the platform responsive and secure.

Memory Management

Memory determines what the AI can recall, and for how long. SAM-AI layers three types of memory, each suited to a different time horizon.

Short-term Memory

Context window management for the current conversation — what the model can "see" in the immediate exchange without retrieval.

Long-term Memory

Persistent storage of user preferences and historical interactions, carried forward across sessions rather than lost when a conversation ends.

Episodic Memory

Recall of specific past conversations and contexts — allowing the AI to reference a particular prior exchange rather than just a general summary of history.

Security & Compliance

Every model call, regardless of provider or hosting mode, is bound by the same regulatory floor.

Regulatory Compliance

GDPR, CCPA, HIPAA, and industry-specific regulations govern what data can reach a model, how it must be handled once it does, and what must be logged, retained, or deleted afterward. This applies uniformly across hosted and direct providers — the compliance layer sits above the model choice, not inside any single vendor's terms of service.
