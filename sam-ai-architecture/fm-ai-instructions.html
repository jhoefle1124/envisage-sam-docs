SAM-AI — FM Configured AI Instructions

The instruction layer that sits between the foundation model and every other pillar of the platform — the tools the AI can call, the agents it can coordinate with, and the prompt engineering that shapes how it reasons.

Why This Layer Runs "Across the System"

Fund-manager-configured AI instructions are not tied to one feature or one page — they are the connective tissue that every other pillar depends on. Tool Integration gives the model hands: the functions, APIs, and external services it can actually call to get work done. System Prompt Engineering gives it judgment: the context management, prompt design, and behavior rules that determine how it interprets a request before it ever touches a tool. Together, they are configured once by the fund manager and then applied consistently to every query, regardless of whether that query ultimately touches SAM Data, SAM Team Data, or both.

Tool Integration

The mechanisms SAM-AI uses to act on the world beyond generating text.

Custom Functions

Business Logic

Domain-specific rules, calculations, and decision-making encoded directly into callable functions, so the model applies the firm's own logic rather than improvising it.

Data Processing

ETL pipelines, data cleaning, and format conversion — the plumbing that turns raw inputs into something a model or downstream system can actually use.

Python Scripts

Custom data processing, analysis, and transformation logic for tasks too specific or too computational to leave to a language model alone.

API Tools

REST APIs

Standard web service integrations for data retrieval and actions — the default way SAM-AI reaches most internal and external systems.

Webhooks

Real-time event-driven integrations and notifications, letting the platform react to something happening elsewhere rather than only responding when asked.

GraphQL

Efficient data querying with flexible field selection, used where a request needs precisely shaped data without over-fetching.

External Services

Third-party APIs

Weather, maps, news, financial data, and other specialized services that extend the platform's awareness beyond its own data.

SaaS Platforms

CRM, ERP, and other business system integrations that let the AI read from and write to the tools the firm already runs on.

Cloud Services

AWS, Google Cloud, and Azure integrations for scalable infrastructure — compute, storage, and managed services the platform draws on as needed.

Agent Coordination

Role-based Agents

Agents with specific expertise and responsibilities, each scoped to a narrower task than a general-purpose assistant would handle alone.

Multi-agent Systems

Multiple specialized AI agents working together, dividing a complex task into pieces that individual agents are better suited to solve.

Agent Communication

Inter-agent messaging and coordination protocols that let agents share intermediate results and hand off work without losing context.

System Prompt Engineering

The configuration layer that determines how the model interprets and responds to a request — independent of any single tool call.

Context Management

Window Management

Optimizing context window usage for maximum relevance, so the limited space available to the model is spent on what actually matters to the current request.

Relevance Scoring

Ranking and selecting the most relevant information for responses, filtering out material that would otherwise dilute the model's attention.

Context Compression

Summarizing and condensing large contexts efficiently, preserving meaning while reducing the volume of text the model has to process.

Prompt Design

Template Creation

Reusable prompt structures for consistent responses, so the same category of question is handled the same way every time it's asked.

Variable Injection

Dynamic content insertion based on user context, filling a template with the specific details of the request at hand.

Chain-of-Thought

Structured reasoning processes for complex problem-solving, prompting the model to work through intermediate steps rather than jumping to a conclusion.

Behavior Configuration

Personality Traits

Defining AI character, expertise level, and interaction preferences so the assistant feels consistent across every conversation.

Response Style

Detailed vs. concise, technical vs. layman explanations — calibrated to who is asking and what they need from the answer.

Tone Settings

Formal, casual, professional, or conversational styles, selected to match the context of the interaction.

Instruction Sets

Multi-step Instructions

Complex workflows requiring sequential processing, where a request is broken into an ordered series of smaller instructions.

Conditional Logic

If-then rules for dynamic response generation, letting the same instruction set adapt its output to the specifics of the input.

Task-specific Prompts

Specialized instructions for different types of queries, tuned to the particular demands of a given task rather than a generic default.

Features

Functions

Describes the combination of tools and data sets needed to complete a function, giving the model a map of what's available before it acts.

Context

Provides context and insights for what to look for in a user prompt, helping the model recognize intent even when a request is loosely phrased.

Examples

Provides samples of what the feature should deliver, giving the model a concrete reference point for the expected shape of a good answer.
