New in SAM · Feature Guide

Excel Processing Enhancements

Excel uploads now get intelligent row sampling instead of naive truncation, much larger file size limits, and a new GLM-4.5 model option — plus a form-submission bug fix so pressing Enter doesn't interrupt an upload in progress.

Smart Sampling Algorithm

Rather than reading only the first N rows of a large spreadsheet, uploads now sample intelligently:

  • First 100 rows — captures headers and initial data patterns.
  • Last 50 rows — captures the most recent data.
  • Smart middle sampling — time-based distribution for date columns, category distribution for categorical data (up to 50 unique values), or even spacing as a fallback.

Summaries are enriched with statistical analysis, a column overview with data types, date ranges, and categorical value distributions. Chunk size for Excel content doubled from 2,000 to 4,000 characters for better context retention.

File Handling

ChangeBeforeAfter
File size limit (unified)25MB100MB
Async processing threshold5MB2MB (faster UX trigger)

MIME type detection was also improved to correctly handle Excel/DOCX files that get misidentified as application/zip or application/octet-stream by the underlying detection library.

Reliability

Comprehensive test coverage for small/large DataFrames, duplicate removal, empty DataFrame edge cases, date detection, and category-based sampling. Also fixes: Enter-key no longer submits the form while a document is processing, and smart sampling now properly de-duplicates rows when combining its three samples.

Data Handling Still Applies

Larger file limits don't change what's safe to upload. A 100MB spreadsheet of client portfolio data is still Tier 3 — see Data Security before uploading anything beyond generic, public data.

การปรับปรุงการประมวลผลไฟล์ Excel

การอัปโหลดไฟล์ Excel ในตอนนี้ใช้การสุ่มตัวอย่างแถวข้อมูลอย่างชาญฉลาด (intelligent row sampling) แทนการตัดข้อมูลแบบง่าย ๆ (naive truncation) มีขีดจำกัดขนาดไฟล์ที่ใหญ่ขึ้นมาก และมีตัวเลือกโมเดลใหม่ GLM-4.5 พร้อมทั้งแก้ไขบั๊กการส่งฟอร์มที่ทำให้การกด Enter ไม่ไปขัดจังหวะการอัปโหลดที่กำลังดำเนินอยู่

อัลกอริทึมการสุ่มตัวอย่างอย่างชาญฉลาด

แทนที่จะอ่านเฉพาะ N แถวแรกของสเปรดชีตขนาดใหญ่ ตอนนี้การอัปโหลดจะสุ่มตัวอย่างข้อมูลอย่างชาญฉลาด:

  • 100 แถวแรก — จับข้อมูล header และรูปแบบข้อมูลเริ่มต้น
  • 50 แถวสุดท้าย — จับข้อมูลล่าสุด
  • การสุ่มตัวอย่างช่วงกลางอย่างชาญฉลาด — กระจายตามช่วงเวลาสำหรับคอลัมน์วันที่ กระจายตามหมวดหมู่สำหรับข้อมูลเชิงหมวดหมู่ (สูงสุด 50 ค่าที่ไม่ซ้ำกัน) หรือกระจายแบบเท่า ๆ กันเป็นทางเลือกสำรอง

สรุปข้อมูลได้รับการเสริมด้วยการวิเคราะห์เชิงสถิติ ภาพรวมคอลัมน์พร้อมประเภทข้อมูล ช่วงวันที่ และการกระจายค่าเชิงหมวดหมู่ ขนาด chunk สำหรับเนื้อหา Excel เพิ่มขึ้นเป็นสองเท่าจาก 2,000 เป็น 4,000 ตัวอักษร เพื่อรักษา context ได้ดีขึ้น

การเปลี่ยนแปลงก่อนหลัง

การตรวจจับประเภท MIME ก็ได้รับการปรับปรุงเช่นกัน เพื่อจัดการไฟล์ Excel/DOCX ที่ถูกไลบรารีตรวจจับที่ใช้งานอยู่ระบุผิดเป็น application/zip หรือ application/octet-stream ให้ถูกต้องยิ่งขึ้น

ความน่าเชื่อถือ

ครอบคลุมการทดสอบอย่างละเอียด สำหรับ DataFrame ขนาดเล็กและขนาดใหญ่ การลบข้อมูลซ้ำ กรณี edge case ของ DataFrame ที่ว่างเปล่า การตรวจจับวันที่ และการสุ่มตัวอย่างตามหมวดหมู่ นอกจากนี้ยังแก้ไข: ปุ่ม Enter จะไม่ส่งฟอร์มอีกต่อไปขณะที่เอกสารกำลังประมวลผลอยู่ และการสุ่มตัวอย่างอย่างชาญฉลาดตอนนี้ลบแถวข้อมูลที่ซ้ำกันได้อย่างถูกต้องเมื่อรวมตัวอย่างทั้งสามชุดเข้าด้วยกัน

กฎการจัดการข้อมูลยังคงบังคับใช้

ขีดจำกัดไฟล์ที่ใหญ่ขึ้นไม่ได้เปลี่ยนแปลงว่าข้อมูลใดปลอดภัยที่จะอัปโหลด สเปรดชีตขนาด 100MB ที่มีข้อมูลพอร์ตโฟลิโอของลูกค้ายังคงเป็น Tier 3 — ดู Data Security ก่อนอัปโหลดข้อมูลใด ๆ ที่เกินกว่าข้อมูลทั่วไปและข้อมูลสาธารณะ

Excel ဖိုင် ပြုစုမှု တိုးတက်ကောင်းမွန်စေမှုများ

Excel ဖိုင် အပ်လုဒ်များသည် ယခုအခါ naive truncation အစား ဉာဏ်ရည်ရှိသော row sampling (intelligent row sampling) ကို ရရှိနိုင်ပြီဖြစ်ပြီး၊ ဖိုင်အရွယ်အစား ကန့်သတ်ချက် ပိုမိုကြီးမားလာသည့်အပြင် GLM-4.5 ဟူသော model ရွေးချယ်စရာအသစ်တစ်ခုလည်း ရရှိလာသည် — ထို့ပြင် Enter နှိပ်လိုက်ခြင်းသည် လက်ရှိ တင်ပို့နေသော အပ်လုဒ်ကို အနှောင့်အယှက်မဖြစ်စေတော့သည့် form-submission bug ပြင်ဆင်မှုတစ်ခုလည်း ပါဝင်သည်။

ဉာဏ်ရည်ရှိသော Sampling အယ်လဂိုရီသမ်

spreadsheet ကြီးတစ်ခု၏ ပထမ N row များကိုသာ ဖတ်မည့်အစား ယခုအခါ အပ်လုဒ်များသည် ဉာဏ်ရည်ရှိစွာ sampling ပြုလုပ်ပါသည် —

  • ပထမဆုံး row ၁၀၀ — header များနှင့် အစပိုင်း ဒေတာပုံစံများကို ဖမ်းယူသည်
  • နောက်ဆုံး row ၅၀ — မကြာသေးမီက ဒေတာကို ဖမ်းယူသည်
  • ဉာဏ်ရည်ရှိသော အလယ်ပိုင်း sampling — ရက်စွဲ column များအတွက် အချိန်အခြေခံ ဖြန့်ဝေမှု၊ categorical ဒေတာအတွက် အမျိုးအစားအလိုက် ဖြန့်ဝေမှု (unique value ၅၀ အထိ) သို့မဟုတ် fallback အဖြစ် တစ်ညီတစ်ညာ ခြားနားမှု

အကျဉ်းချုပ်များကို စာရင်းအင်း ခွဲခြမ်းစိတ်ဖြာမှု၊ ဒေတာအမျိုးအစားများ၊ ရက်စွဲအပိုင်းအခြားများ နှင့် categorical value ဖြန့်ဝေမှုများပါသော column ခြုံငုံသုံးသပ်ချက်ဖြင့် ထပ်ပေါင်းသိပ်သည်။ Excel content အတွက် chunk အရွယ်အစားကို context ကို ပိုမိုကောင်းမွန်စွာ ထိန်းသိမ်းနိုင်ရန် ၂,၀၀၀ မှ ၄,၀၀၀ စာလုံးအထိ နှစ်ဆ တိုးမြှင့်လိုက်သည်။

ဖိုင် ကိုင်တွယ်မှု

ပြောင်းလဲမှုယခင်ယခု
ဖိုင်အရွယ်အစား ကန့်သတ်ချက် (တစ်ခုတည်း ပေါင်းစည်းထားသော)25MB100MB
Async ပြုစုမှု အတိုင်းအတာ5MB2MB (UX ပိုမိုမြန်ဆန်စွာ စတင်စေရန်)

underlying detection library က application/zip သို့မဟုတ် application/octet-stream အဖြစ် အမှားရှုပ်ထွေးစွာ ခွဲခြားမိသော Excel/DOCX ဖိုင်များကို မှန်ကန်စွာ ကိုင်တွယ်နိုင်ရန် MIME type ရှာဖွေတွေ့ရှိမှုကိုလည်း ပိုမိုတိုးတက်စေထားသည်။

ယုံကြည်စိတ်ချရမှု

ကျယ်ပြန့်သော test coverage — DataFrame သေးငယ်/ကြီးမားမှုများ၊ ထပ်နေသော ဒေတာဖယ်ရှားခြင်း၊ DataFrame အလွတ် edge case များ၊ ရက်စွဲရှာဖွေတွေ့ရှိခြင်း နှင့် category-based sampling တို့အတွက် ပါဝင်သည်။ ထို့ပြင် ပြင်ဆင်ချက်များ — စာရွက်စာတမ်းတစ်ခု ပြုစုနေဆဲအချိန်တွင် Enter ခလုတ်က form ကို နောက်ထပ် မတင်ပို့တော့ပါ၊ ထို့အပြင် ဉာဏ်ရည်ရှိသော sampling သည် ၎င်း၏ sample သုံးခုကို ပေါင်းစည်းသည့်အခါ ထပ်နေသော row များကို မှန်ကန်စွာ ဖယ်ရှားပေးနိုင်ပြီ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်ဆဲ

ဖိုင်အရွယ်အစား ကန့်သတ်ချက် ပိုမိုကြီးမားလာခြင်းသည် အပ်လုဒ်လုပ်ရန် ဘာလုံခြုံသည်ဆိုသည်ကို ပြောင်းလဲမပေးပါ။ ဖောက်သည် portfolio ဒေတာပါသော 100MB spreadsheet သည် Tier 3 အဖြစ် ဆက်လက်တည်ရှိနေဆဲဖြစ်သည် — အထွေထွေနှင့် အများပြည်သူ ဒေတာထက်ပိုသော မည်သည့်အရာကိုမဆို အပ်လုဒ်မလုပ်မီ Data Security ကို ကြည့်ပါ။
