Citation Source ID Bug Fix

A follow-up fix for citations loaded from saved chat sessions, where deserialized dictionaries — not citation objects — caused a crash when the code tried to read a source ID.

The Problem

When a chat session is reloaded from the database or a checkpoint, citations come back as plain dictionaries rather than citation objects. Code that assumed source.source_id was always an attribute failed on those dictionaries with a 'dict' object has no attribute 'source_id' error.

The Fix

  • Citation validation now handles both citation objects and dictionaries when extracting source IDs.
  • Error state now clears at session initialization, so a past error can't persist across sessions.
  • The citation processing logic was extracted into a dedicated helper, cutting the enclosing function's complexity from 53 statements / 13 branches down to roughly 25 statements / 8 branches — resolving the linter's complexity warnings in the process.

Reliability

New tests cover citation objects, dictionary citations (the fix case), mixed collections of both, pre-processed citations, and error handling. All existing tests continue to pass, and linting is clean.

Data Handling Still Applies

A backend deserialization fix — no change to what citations may reference or how tiered data flows through the system. See Data Security for the underlying rules.

การแก้ไขบั๊ก Source ID ของการอ้างอิง (Citation)

การแก้ไขต่อเนื่องสำหรับการอ้างอิง (citation) ที่โหลดมาจากเซสชันแชทที่บันทึกไว้ ซึ่งในกรณีนี้ dictionary ที่ผ่านการ deserialize แล้ว — ไม่ใช่ citation object — เป็นสาเหตุให้เกิด crash เมื่อโค้ดพยายามอ่านค่า source ID

ปัญหา

เมื่อเซสชันแชทถูกโหลดใหม่จากฐานข้อมูลหรือ checkpoint การอ้างอิง (citation) จะกลับมาในรูปแบบ dictionary ธรรมดา แทนที่จะเป็น citation object โค้ดที่สมมติไว้ว่า source.source_id เป็น attribute เสมอ จึงล้มเหลวเมื่อเจอ dictionary เหล่านี้ ด้วย error 'dict' object has no attribute 'source_id'

การแก้ไข

  • ระบบตรวจสอบความถูกต้องของ citation ตอนนี้รองรับทั้ง citation object และ dictionary เมื่อดึงค่า source ID
  • สถานะ error ตอนนี้จะถูกล้างตั้งแต่ขั้นตอนเริ่มต้นเซสชัน เพื่อไม่ให้ error ที่เกิดขึ้นก่อนหน้าค้างข้ามไปยังเซสชันอื่น
  • ตรรกะการประมวลผล citation ถูกแยกออกมาเป็น helper function เฉพาะ ทำให้ความซับซ้อนของฟังก์ชันหลักลดลงจากประมาณ 53 statements / 13 branches เหลือประมาณ 25 statements / 8 branches ซึ่งช่วยแก้คำเตือนเรื่องความซับซ้อนจาก linter ไปพร้อมกันด้วย

ความน่าเชื่อถือ

การทดสอบชุดใหม่ครอบคลุม citation object, citation แบบ dictionary (กรณีที่ได้รับการแก้ไข), ชุดข้อมูลที่ผสมกันทั้งสองแบบ, citation ที่ผ่านการประมวลผลมาก่อนแล้ว และการจัดการ error การทดสอบเดิมทั้งหมดยังคงผ่านตามปกติ และ linting ผ่านเรียบร้อยไม่มีคำเตือน

การจัดการข้อมูลยังคงเป็นไปตามเดิม

นี่เป็นการแก้ไขด้าน deserialization ฝั่ง backend เท่านั้น — ไม่มีการเปลี่ยนแปลงว่า citation สามารถอ้างอิงถึงข้อมูลใดได้บ้าง หรือข้อมูลตาม tier ไหลผ่านระบบอย่างไร ดูกฎเกณฑ์พื้นฐานได้ที่ Data Security

Citation Source ID ချို့ယွင်းချက် ပြင်ဆင်မှု

သိမ်းဆည်းထားသော chat session များမှ ရယူသော citation များအတွက် ဆက်လက်ပြင်ဆင်ခြင်းတစ်ခုဖြစ်သည် — ဤနေရာတွင် deserialize လုပ်ထားသော dictionary များသည် citation object များ မဟုတ်ဘဲ ကုဒ်မှ source ID ကို ဖတ်ရန် ကြိုးစားစဉ် crash ဖြစ်စေခဲ့သည်။

ပြဿနာ

chat session တစ်ခုကို database သို့မဟုတ် checkpoint မှ ပြန်လည်တင်သောအခါ citation များသည် citation object များအဖြစ်မဟုတ်ဘဲ ရိုးရိုး dictionary များအဖြစ် ပြန်လာသည်။ source.source_id ကို attribute တစ်ခုအဖြစ် အမြဲရှိနေသည်ဟု ယူဆထားသော ကုဒ်သည် ထို dictionary များနှင့် ကြုံသောအခါ 'dict' object has no attribute 'source_id' ဟူသော error ဖြင့် ကျရှုံးခဲ့သည်။

ပြင်ဆင်မှု

  • Citation validation သည် ယခုအခါ source ID များကို ထုတ်ယူသည့်အခါ citation object များနှင့် dictionary များ နှစ်မျိုးလုံးကို ကိုင်တွယ်နိုင်ပြီ
  • Error state ကို session initialization အဆင့်တွင် ယခုအခါ ရှင်းလင်းပေးသဖြင့် အတိတ်က error များသည် session များကူး၍ မကျန်ရှိနိုင်တော့ပါ
  • Citation ပြုပြင်ဆောင်ရွက်မှု logic ကို သီးသန့် helper function တစ်ခုအဖြစ် ခွဲထုတ်လိုက်ပြီး၊ ၎င်းက ခြုံငုံထားသော function ၏ ရှုပ်ထွေးမှုကို statement 53 ခန့် / branch 13 ခန့်မှ statement 25 ခန့် / branch 8 ခန့်အထိ လျှော့ချနိုင်ခဲ့ပြီး — linter ၏ ရှုပ်ထွေးမှုဆိုင်ရာ သတိပေးချက်များကိုပါ တစ်ပါတည်း ဖြေရှင်းပေးခဲ့သည်

ယုံကြည်စိတ်ချရမှု

စမ်းသပ်မှုအသစ်များသည် citation object များ၊ dictionary citation များ (ပြင်ဆင်မှု ဖြစ်ရပ်)၊ နှစ်မျိုးလုံး ရောနှောထားသော collection များ၊ ကြိုတင်ပြုပြင်ပြီးသား citation များနှင့် error ကိုင်တွယ်မှုတို့ကို လွှမ်းခြုံစမ်းသပ်သည်။ လက်ရှိစမ်းသပ်မှုအားလုံး ဆက်လက်အောင်မြင်ပြီး linting သည် သတိပေးချက် တစ်ခုမှမရှိဘဲ သန့်ရှင်းနေသည်

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

Backend deserialization ပြင်ဆင်မှုတစ်ခုသာဖြစ်ပြီး — citation များ မည်သည့်အရာကို ရည်ညွှန်းနိုင်သည် သို့မဟုတ် tier အလိုက် ဒေတာသည် စနစ်ကို မည်သို့စီးဆင်းသည်ဆိုသည်တွင် ပြောင်းလဲမှု မရှိပါ။ အခြေခံစည်းမျဉ်းများအတွက် Data Security ကိုကြည့်ပါ
