New in SAM · Technical Reference

Company Financials Batch Retrieval

Company financials can now be retrieved for multiple companies in a single request, with smart batch defaults — part of a broader consolidation that folded the standalone portfolio analyst module into company_financials.py.

What's New

  • Batch retrieval: fetch financial data for multiple companies in one call, with dynamic limits based on which fields are requested.
  • Optional description control: a parameter to exclude descriptive text from responses, reducing token usage when only figures are needed.
  • Stronger stock screener guidance: tool documentation now prohibits combining company_list with other filters, preventing a class of misuse.

Cleanup

~2,800 lines of obsolete portfolio-analyst files (error handler, exceptions, repository, service, types, validator) were removed after their functionality was fully migrated into company_financials.py, simplifying the codebase without changing behavior.

Reliability

Comprehensive test coverage for the new batch functionality and helper methods, with backward compatibility verified for single-company requests. No breaking changes — new parameters are optional.

Data Handling Still Applies

Batching more companies into one call doesn't change what tier the underlying financials carry — see Data Security for the standard rules.

ฟีเจอร์ใหม่ใน SAM · เอกสารอ้างอิงทางเทคนิค

การดึงข้อมูลการเงินบริษัทแบบ Batch

ขณะนี้สามารถดึงข้อมูลการเงินของบริษัทได้พร้อมกันหลายบริษัทในคำขอเดียว โดยมีค่าเริ่มต้นของ batch ที่ฉลาดขึ้น — เป็นส่วนหนึ่งของการรวมโค้ดครั้งใหญ่ที่ย้ายโมดูล portfolio analyst แบบแยกเดี่ยวเข้าไปรวมกับ company_financials.py

มีอะไรใหม่

  • Batch retrieval: ดึงข้อมูลการเงินของหลายบริษัทได้ในการเรียกครั้งเดียว โดยมีขีดจำกัดแบบไดนามิกขึ้นอยู่กับฟิลด์ที่ร้องขอ
  • ตัวเลือกควบคุมคำอธิบาย (Optional description control): พารามิเตอร์สำหรับไม่รวมข้อความอธิบายในผลลัพธ์ ช่วยลดการใช้ token เมื่อต้องการเฉพาะตัวเลข
  • คำแนะนำที่เข้มงวดขึ้นสำหรับ stock screener: เอกสารประกอบเครื่องมือระบุห้ามใช้ company_list ร่วมกับตัวกรองอื่น เพื่อป้องกันการใช้งานผิดประเภท

การล้างโค้ด

โค้ดของไฟล์ portfolio-analyst ที่ไม่ใช้งานแล้วประมาณ 2,800 บรรทัด (error handler, exceptions, repository, service, types, validator) ถูกลบออก หลังจากฟังก์ชันการทำงานทั้งหมดถูกย้ายเข้าไปรวมกับ company_financials.py เรียบร้อยแล้ว ช่วยให้โค้ดเบสเรียบง่ายขึ้นโดยไม่เปลี่ยนพฤติกรรมการทำงาน

ความน่าเชื่อถือ

มีการทดสอบครอบคลุมฟังก์ชัน batch ใหม่และ helper method ต่าง ๆ อย่างครบถ้วน พร้อมยืนยันความเข้ากันได้แบบย้อนหลัง (backward compatibility) สำหรับคำขอแบบบริษัทเดียว ไม่มีการเปลี่ยนแปลงที่กระทบระบบเดิม (breaking changes) — พารามิเตอร์ใหม่เป็นแบบทางเลือกทั้งหมด

กฎการจัดการข้อมูลยังคงบังคับใช้

การรวมหลายบริษัทไว้ในคำขอเดียวไม่ได้เปลี่ยนแปลง tier ของข้อมูลการเงินที่อยู่เบื้องหลัง — ดู Data Security สำหรับกฎมาตรฐาน

ကုမ္ပဏီ ဘဏ္ဍာရေးအချက်အလက် Batch ရယူခြင်း

ကုမ္ပဏီ ဘဏ္ဍာရေးအချက်အလက်များကို တောင်းဆိုမှုတစ်ခုတည်းဖြင့် ကုမ္ပဏီများစွာအတွက် တစ်ပြိုင်နက် ရယူနိုင်ပြီဖြစ်သည်၊ ဉာဏ်ရည်ရှိသော batch default များနှင့်အတူ — ၎င်းသည် portfolio analyst module သီးခြားကို company_financials.py ထဲသို့ ပေါင်းစည်းလိုက်သည့် ကျယ်ပြန့်သော ပေါင်းစည်းမှု၏ တစ်စိတ်တစ်ပိုင်း ဖြစ်သည်။

အသစ်ပါဝင်သည့်အချက်များ

  • Batch retrieval: တောင်းဆိုသည့် field အပေါ်မူတည်၍ dynamic limit များဖြင့် ကုမ္ပဏီများစွာ၏ ဘဏ္ဍာရေးအချက်အလက်ကို ခေါ်ဆိုမှုတစ်ခုတည်းဖြင့် ရယူနိုင်သည်။
  • ရွေးချယ်နိုင်သော description ထိန်းချုပ်မှု: ဂဏန်းအချက်အလက်သာ လိုအပ်သောအခါ token အသုံးပြုမှုလျှော့ချရန် response များမှ ဖော်ပြချက်စာသားကို ဖယ်ထုတ်ပေးသော parameter။
  • stock screener အတွက် ပိုမိုတင်းကျပ်သော လမ်းညွှန်ချက်: tool documentation တွင် company_list ကို filter အခြားများနှင့် ပေါင်းသုံးခြင်းကို ယခုအခါ တားမြစ်ထားသဖြင့် အလွဲသုံးမှုအမျိုးအစားတစ်ခုကို ကာကွယ်ပေးသည်။

ရှင်းလင်းမှု

အသုံးမကျတော့သည့် portfolio-analyst ဖိုင်များ (error handler, exceptions, repository, service, types, validator) ၏ လိုင်းအရေအတွက် ~2,800 ကို ၎င်းတို့၏ function များ company_financials.py ထဲသို့ အပြည့်အဝ ပြောင်းရွှေ့ပြီးနောက် ဖယ်ရှားခဲ့သည်၊ ဤသည်က behavior ကို မပြောင်းလဲဘဲ codebase ကို ရိုးရှင်းအောင် ပြုလုပ်ပေးသည်။

ယုံကြည်စိတ်ချရမှု

batch function အသစ်နှင့် helper method များအတွက် စမ်းသပ်မှု coverage အပြည့်အဝရှိပြီး၊ ကုမ္ပဏီတစ်ခုတည်း တောင်းဆိုမှုများအတွက် backward compatibility ကို အတည်ပြုထားသည်။ breaking changes မရှိပါ — parameter အသစ်များသည် ရွေးချယ်ခွင့်ရှိသည့် (optional) parameter များသာ ဖြစ်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်ဆဲ

ကုမ္ပဏီများစွာကို ခေါ်ဆိုမှုတစ်ခုတည်းထဲ batch ပြုလုပ်ခြင်းသည် အခြေခံဘဏ္ဍာရေးအချက်အလက်၏ tier ကို ပြောင်းလဲမပေးပါ — စံနှုန်းစည်းမျဉ်းများအတွက် Data Security ကို ကြည့်ပါ။
