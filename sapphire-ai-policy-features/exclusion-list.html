New in SAM · Feature Guide

Exclusion List Capabilities

The stock screener's exclusion logic now matches partially, not just exactly — so "exclude Pineapple" correctly drops every company with Pineapple in the name, and "exclude AAPL" drops it under any exchange prefix. This makes exclusion phrasing in natural-language queries behave the way you'd expect.

What's New

  • Partial name matching: name exclusions now use NOT LIKE instead of an exact match, so a single exclusion term catches every variant of that name.
  • Partial ticker matching: ticker exclusions match partially too, so an exclusion works regardless of exchange prefix.
  • Better handling of mixed queries: improved logic for queries that combine inclusions and exclusions in the same request.

How It Works

Name exclusion

Excluding "Pineapple"

Now also excludes "Pineapple Resources Berhad," "Pineapple Financial," and any other company with "Pineapple" in the name — not just a company named exactly "Pineapple."

Ticker exclusion

Excluding "AAPL"

Now also excludes exchange-prefixed variants like "XNGS:AAPL," so the exclusion holds regardless of which exchange prefix a result uses.

NULL-safe. The underlying NOT LIKE logic explicitly handles missing data, so companies with a blank name or ticker field aren't accidentally dropped by an exclusion filter.

What You Can Ask

Example

"Show me Apple stocks but exclude anything with Pineapple."

Example

"Tech companies excluding MSFT."

Example

"All banks except those with 'Community' in the name."

Inclusion vs. Exclusion Logic

Inclusion filters and exclusion filters are combined differently under the hood: inclusion criteria are joined with OR logic (match any of these), while exclusion criteria are joined with AND logic (must not match any of these). That distinction is what lets a single query mix "show me X" with "but not Y" correctly.

Reliability

Backward compatible. Existing exact-match behavior for other filter types is unaffected. Testing covered partial name and ticker exclusions, exchange-prefix handling, industry exclusions, and combined inclusion/exclusion queries.

Data Handling Still Applies

Exclusion criteria based on public company names or tickers are generally Tier 1, same as any other open-market screen.

Usually fine

Public screening exclusions

"Exclude companies with X in the name" on its own, with no client framing, stays close to Tier 1.

Check first

Exclusions tied to a mandate

If the exclusion list reflects a client's specific investment restrictions or exclusion policy, treat that framing as Tier 2/3 and keep it inside SAM and approved workflows, per Data Security.

ความสามารถของ Exclusion List

ตอนนี้ตรรกะการยกเว้น (exclusion) ของ stock screener จะจับคู่แบบบางส่วนได้ ไม่ใช่แค่การจับคู่แบบตรงทั้งหมดเหมือนก่อน — ดังนั้น "exclude Pineapple" จะตัดบริษัททุกแห่งที่มีคำว่า Pineapple อยู่ในชื่อออกอย่างถูกต้อง และ "exclude AAPL" จะตัดออกไม่ว่าจะมีคำนำหน้าตลาดหลักทรัพย์ใดก็ตาม ทำให้การเขียน exclusion ในคำสั่งภาษาธรรมชาติทำงานได้ตามที่คาดหวัง

มีอะไรใหม่

  • การจับคู่ชื่อแบบบางส่วน: การยกเว้นตามชื่อตอนนี้ใช้ NOT LIKE แทนการจับคู่แบบตรงทั้งหมด ทำให้คำยกเว้นเพียงคำเดียวครอบคลุมทุกรูปแบบของชื่อนั้น
  • การจับคู่ ticker แบบบางส่วน: การยกเว้นตาม ticker ก็จับคู่แบบบางส่วนได้เช่นกัน ทำให้การยกเว้นทำงานได้ไม่ว่าจะมีคำนำหน้าตลาดหลักทรัพย์ใดก็ตาม
  • การจัดการ query แบบผสมที่ดีขึ้น: ปรับปรุงตรรกะสำหรับ query ที่รวมทั้งการรวมและการยกเว้นไว้ในคำขอเดียวกัน

วิธีการทำงาน

การยกเว้นตามชื่อ

การยกเว้น "Pineapple"

ตอนนี้จะยกเว้น "Pineapple Resources Berhad," "Pineapple Financial," และบริษัทอื่น ๆ ที่มีคำว่า "Pineapple" อยู่ในชื่อด้วย ไม่ใช่แค่บริษัทที่ชื่อ "Pineapple" ตรงตัวเท่านั้น

การยกเว้นตาม Ticker

การยกเว้น "AAPL"

ตอนนี้จะยกเว้นรูปแบบที่มีคำนำหน้าตลาดหลักทรัพย์ด้วย เช่น "XNGS:AAPL," ดังนั้นการยกเว้นจะยังคงมีผลไม่ว่าผลลัพธ์จะใช้คำนำหน้าตลาดหลักทรัพย์ใดก็ตาม

ปลอดภัยจากค่า NULL ตรรกะ NOT LIKE ที่อยู่เบื้องหลังจะจัดการกับข้อมูลที่ขาดหายไปอย่างชัดเจน ดังนั้นบริษัทที่มีฟิลด์ชื่อหรือ ticker ว่างเปล่าจะไม่ถูกตัดออกโดยไม่ได้ตั้งใจจากตัวกรอง exclusion

ตรรกะการรวมเทียบกับการยกเว้น

ตัวกรองแบบรวม (inclusion) และตัวกรองแบบยกเว้น (exclusion) ถูกรวมกันด้วยวิธีที่ต่างกันในเบื้องหลัง เกณฑ์การรวมจะเชื่อมด้วยตรรกะ OR (ตรงกับข้อใดข้อหนึ่งก็ได้) ในขณะที่เกณฑ์การยกเว้นจะเชื่อมด้วยตรรกะ AND (ต้องไม่ตรงกับข้อใดเลย) ความแตกต่างนี้เองที่ทำให้ query เดียวสามารถผสม "แสดง X" กับ "แต่ไม่เอา Y" ได้อย่างถูกต้อง

Exclusion List စွမ်းရည်များ

Stock screener ၏ exclusion (ချန်လှပ်ခြင်း) logic သည် ယခုအခါ တိကျစွာသာမက တစ်စိတ်တစ်ပိုင်း တိုက်ဆိုင်မှုကိုပါ ရှာဖွေနိုင်ပါပြီ — ထို့ကြောင့် "exclude Pineapple" ဆိုလျှင် အမည်တွင် Pineapple ပါဝင်သော ကုမ္ပဏီအားလုံးကို မှန်ကန်စွာ ဖယ်ထုတ်ပေးပြီး "exclude AAPL" ဆိုလျှင် stock exchange ရှေ့ဆက်မည်သို့ပင်ရှိစေ ဖယ်ထုတ်ပေးသည်။ ဤသည်က natural-language query များတွင် exclusion အသုံးအနှုန်းများ မျှော်လင့်ထားသည့်အတိုင်း အလုပ်လုပ်စေသည်။

အသစ်ပါဝင်လာသည့်အချက်များ

  • အမည် တစ်စိတ်တစ်ပိုင်း တိုက်ဆိုင်မှု — အမည်ဖြင့် ဖယ်ထုတ်ခြင်းများသည် ယခုအခါ တိကျစွာ တိုက်ဆိုင်မှုအစား NOT LIKE ကို အသုံးပြုသဖြင့် exclusion စကားလုံးတစ်ခုတည်းဖြင့် ထိုအမည်၏ ကွဲပြားမှုအားလုံးကို ဖမ်းယူနိုင်သည်
  • Ticker တစ်စိတ်တစ်ပိုင်း တိုက်ဆိုင်မှု — Ticker ဖြင့် ဖယ်ထုတ်ခြင်းများသည်လည်း တစ်စိတ်တစ်ပိုင်း တိုက်ဆိုင်နိုင်သဖြင့် exchange ရှေ့ဆက် မည်သို့ပင်ရှိစေ exclusion သည် အလုပ်လုပ်သည်
  • ရောနှောထားသော query များကို ပိုမိုကောင်းမွန်စွာ ကိုင်တွယ်ခြင်း — inclusion နှင့် exclusion ကို တောင်းဆိုမှုတစ်ခုတည်းတွင် ပေါင်းစပ်ထားသော query များအတွက် logic ကို ပိုမိုကောင်းမွန်အောင် ပြုလုပ်ထားသည်

အလုပ်လုပ်ပုံ

အမည်ဖြင့် ဖယ်ထုတ်ခြင်း

"Pineapple" ကို ဖယ်ထုတ်ခြင်း

ယခုအခါ "Pineapple Resources Berhad,"၊ "Pineapple Financial," နှင့် အမည်တွင် "Pineapple" ပါဝင်သည့် အခြားကုမ္ပဏီများကိုပါ ဖယ်ထုတ်ပေးသည် — "Pineapple" ဟု အတိအကျ အမည်ရှိသော ကုမ္ပဏီတစ်ခုတည်းကိုသာ မဟုတ်တော့ပါ

Ticker ဖြင့် ဖယ်ထုတ်ခြင်း

"AAPL" ကို ဖယ်ထုတ်ခြင်း

ယခုအခါ "XNGS:AAPL" ကဲ့သို့ exchange ရှေ့ဆက်ပါသော ပုံစံများကိုပါ ဖယ်ထုတ်ပေးသဖြင့် ရလဒ်တွင် မည်သည့် exchange ရှေ့ဆက်ကို အသုံးပြုသည်ဖြစ်စေ exclusion သည် ဆက်လက်သက်ရောက်နေမည်

NULL တန်ဖိုးအတွက် ဘေးကင်းသည် အခြေခံ NOT LIKE logic သည် ပျောက်ဆုံးနေသော ဒေတာကို ရှင်းလင်းစွာ ကိုင်တွယ်သောကြောင့် အမည် သို့မဟုတ် ticker field အလွတ်ရှိသော ကုမ္ပဏီများသည် exclusion filter တစ်ခုကြောင့် မတော်တဆ ကျန်ရစ်ခြင်း မရှိတော့ပါ

မေးမြန်းနိုင်သည့် ဥပမာများ

ဥပမာ

"Apple ရှယ်ယာများကို ပြပါ၊ သို့သော် Pineapple နှင့်ဆိုင်သော မည်သည့်အရာကိုမျှ မထည့်ပါနှင့်။"

ဥပမာ

"MSFT ကို မထည့်ဘဲ နည်းပညာကုမ္ပဏီများ"

ဥပမာ

"အမည်တွင် 'Community' ပါသော ဘဏ်များမှလွဲ၍ ဘဏ်အားလုံး"

Inclusion နှင့် Exclusion Logic

Inclusion filter နှင့် exclusion filter များကို နောက်ကွယ်တွင် မတူညီသောနည်းလမ်းဖြင့် ပေါင်းစပ်ထားသည် — inclusion criteria များကို OR logic ဖြင့် ချိတ်ဆက်ထားပြီး (တစ်ခုခုနှင့် တိုက်ဆိုင်ရန်) exclusion criteria များကို AND logic ဖြင့် ချိတ်ဆက်ထားသည် (မည်သည့်တစ်ခုနှင့်မျှ မတိုက်ဆိုင်ရ)။ ဤကွာခြားမှုကြောင့်ပင် query တစ်ခုတည်းတွင် "X ကို ပြပါ" နှင့် "သို့သော် Y မဟုတ်ပါ" ကို မှန်ကန်စွာ ပေါင်းစပ်နိုင်ခြင်း ဖြစ်သည်။

ယုံကြည်စိတ်ချမှု

Backward compatible ဖြစ်သည် အခြား filter အမျိုးအစားများ၏ ရှိပြီးသား တိကျသော တိုက်ဆိုင်မှု အပြုအမူသည် ထိခိုက်မှု မရှိပါ။ စမ်းသပ်မှုသည် အမည်နှင့် ticker တစ်စိတ်တစ်ပိုင်း ဖယ်ထုတ်ခြင်း၊ exchange ရှေ့ဆက် ကိုင်တွယ်မှု၊ လုပ်ငန်းအလိုက် ဖယ်ထုတ်ခြင်း နှင့် inclusion/exclusion ပေါင်းစပ် query များကို လွှမ်းခြုံသည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်သည်

အများပြည်သူ ကုမ္ပဏီအမည် သို့မဟုတ် ticker အပေါ် အခြေခံသည့် exclusion criteria များသည် ယေဘုယျအားဖြင့် Tier 1 ဖြစ်ပြီး အခြား open-market screen များနှင့် အတူတူပင်ဖြစ်သည်

များသောအားဖြင့် ကိစ္စမရှိပါ

အများပြည်သူ screening exclusion

ဖောက်သည် framing တစ်စုံတစ်ရာ မပါဘဲ "အမည်တွင် X ပါသော ကုမ္ပဏီများကို ဖယ်ထုတ်ပါ" ဆိုသည့် ညွှန်ကြားချက်တစ်ခုတည်းသည် Tier 1 နှင့် နီးကပ်စွာ ကျန်ရှိသည်
