New in SAM · Feature Guide

Scorecard Team Averages & Spider Charts

Scorecard comparisons can now include a team average for context, spider/polar charts got intelligent scaling for clearer visual comparison, and the AI screener learned to parse abbreviated numbers like \"1B\" or \"500M\" in market cap filters.

What's New

  • Scorecard team averages: an optional include_team_average parameter adds a team average calculation alongside individual scorecard comparisons.
  • Advanced spider/polar charts: intelligent y-axis scaling and automatic scale adjustment based on data distribution, for clearer factor-by-factor visual comparison.
  • AI screener parsing: better handling of large-number notations (B/M/K) for market cap filters, plus corrected debt-to-equity field mapping and technology stock filtering logic.

Reliability

Team average calculation tested in scorecard tool tests; chart visualization verified across varied data distributions; AI screener parsing tested with 1B, 500M, and 100K style inputs. All changes are backward compatible with existing API contracts.

Data Handling Still Applies

A team average is still built from team-scoped scorecard data — Tier 3, same as the individual scores it's derived from. See Data Security.

ใหม่ใน SAM · คู่มือฟีเจอร์

ค่าเฉลี่ยทีมสกอร์การ์ดและกราฟใยแมงมุม

การเปรียบเทียบสกอร์การ์ดในขณะนี้สามารถแสดงค่าเฉลี่ยของทีมเพื่อเป็นบริบทเพิ่มเติมได้ กราฟใยแมงมุม/กราฟเรดาร์ (spider/polar charts) ได้รับการปรับสเกลอย่างชาญฉลาดเพื่อการเปรียบเทียบภาพที่ชัดเจนขึ้น และตัวคัดกรองหุ้นที่ขับเคลื่อนด้วย AI สามารถแปลความหมายตัวเลขแบบย่อ เช่น "1B" หรือ "500M" ในตัวกรองมูลค่าตลาดได้แล้ว

มีอะไรใหม่

  • ค่าเฉลี่ยทีมในสกอร์การ์ด: พารามิเตอร์เสริม include_team_average เพิ่มการคำนวณค่าเฉลี่ยของทีมควบคู่ไปกับการเปรียบเทียบสกอร์การ์ดรายบุคคล
  • กราฟใยแมงมุม/เรดาร์ขั้นสูง: การปรับสเกลแกน y อย่างชาญฉลาดและการปรับสเกลอัตโนมัติตามการกระจายตัวของข้อมูล เพื่อการเปรียบเทียบเชิงภาพแบบปัจจัยต่อปัจจัยที่ชัดเจนยิ่งขึ้น
  • การแปลความหมายของตัวคัดกรอง AI: การจัดการที่ดีขึ้นสำหรับสัญกรณ์ตัวเลขจำนวนมาก (B/M/K) ในตัวกรองมูลค่าตลาด รวมถึงการแก้ไขการแมปฟิลด์อัตราส่วนหนี้สินต่อทุน (debt-to-equity) และตรรกะการกรองหุ้นกลุ่มเทคโนโลยี

ความน่าเชื่อถือ

การคำนวณค่าเฉลี่ยของทีมได้รับการทดสอบในชุดทดสอบเครื่องมือสกอร์การ์ด การแสดงผลกราฟได้รับการตรวจสอบในหลากหลายรูปแบบการกระจายตัวของข้อมูล และการแปลความหมายของตัวคัดกรอง AI ได้รับการทดสอบด้วยข้อมูลรูปแบบ 1B, 500M และ 100K การเปลี่ยนแปลงทั้งหมดเข้ากันได้กับ API contract ที่มีอยู่เดิม

หลักการจัดการข้อมูลยังคงใช้บังคับ

ค่าเฉลี่ยของทีมยังคงถูกสร้างจากข้อมูลสกอร์การ์ดในขอบเขตของทีม — Tier 3 เช่นเดียวกับคะแนนรายบุคคลที่นำมาคำนวณ ดู ความปลอดภัยข้อมูล

Scorecard အသင်းပျမ်းမျှနှင့် Spider Chart

Scorecard နှိုင်းယှဉ်မှုများသည် ယခုအခါ context အတွက် အသင်း (team) ပျမ်းမျှတန်ဖိုးကို ထည့်သွင်းနိုင်ပြီ၊ spider/polar chart များသည် ပိုမိုရှင်းလင်းသော ဒေတာနှိုင်းယှဉ်မှုအတွက် ဉာဏ်ရည်ရှိသော scaling ရရှိလာပြီ၊ ထို့အပြင် AI screener သည် market cap filter များတွင် "1B" သို့မဟုတ် "500M" ကဲ့သို့ အတိုကောက်ဂဏန်းများကို ဖတ်တတ်လာပြီဖြစ်သည်။

အသစ်ဘာတွေရှိလဲ

  • Scorecard အသင်းပျမ်းမျှ: ရွေးချယ်နိုင်သော include_team_average parameter သည် တစ်ဦးချင်း scorecard နှိုင်းယှဉ်မှုများနှင့်အတူ အသင်းပျမ်းမျှတန်ဖိုး တွက်ချက်မှုကို ထပ်ထည့်ပေးသည်။
  • အဆင့်မြင့် Spider/Polar Chart များ: ဒေတာ ဖြန့်ကျဲမှုအပေါ် အခြေခံ၍ ဉာဏ်ရည်ရှိသော y-axis scaling နှင့် အလိုအလျောက် scale ချိန်ညှိမှု — factor တစ်ခုချင်းစီအလိုက် ပိုမိုရှင်းလင်းသော ရုပ်ပုံနှိုင်းယှဉ်မှုအတွက်။
  • AI Screener ဖတ်ခြင်း: market cap filter များအတွက် ဂဏန်းအကြီးစား သင်္ကေတ (B/M/K) ကိုင်တွယ်မှု ပိုမိုကောင်းမွန်လာပြီး debt-to-equity field mapping အမှားပြင်ဆင်မှုနှင့် technology stock filtering logic ကိုပါ ပြင်ဆင်ထားသည်။

ယုံကြည်စိတ်ချရမှု

အသင်းပျမ်းမျှတန်ဖိုး တွက်ချက်မှုကို scorecard tool test များတွင် စစ်ဆေးထားပြီး၊ chart visualization ကို ဒေတာဖြန့်ကျဲမှု အမျိုးမျိုးတွင် အတည်ပြုထားကာ၊ AI screener ဖတ်ခြင်းကို 1B, 500M နှင့် 100K ပုံစံ input များဖြင့် စစ်ဆေးထားသည်။ ပြောင်းလဲမှုအားလုံးသည် လက်ရှိ API contract များနှင့် backward compatible ဖြစ်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

အသင်းပျမ်းမျှတန်ဖိုးသည် အသင်းအလိုက် scorecard ဒေတာမှသာ ဆက်လက် တည်ဆောက်ထားခြင်းဖြစ်သည် — ၎င်းကို ရယူသည့် တစ်ဦးချင်း score များကဲ့သို့ပင် Tier 3 ဖြစ်သည်။ ဒေတာလုံခြုံရေး ကို ကြည့်ပါ။
