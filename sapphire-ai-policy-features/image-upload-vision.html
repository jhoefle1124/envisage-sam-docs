New in SAM · Feature Guide

Image Upload & Vision Processing

SAM can now see. Upload a JPG, PNG, GIF, or WebP image and a vision model converts it to a text description that any chat model can then reason about — charts, screenshots, and general images all get context-aware handling.

What's New

  • Vision model processing: a Fireworks.ai vision model (Qwen3 VL 235B) converts uploaded images into text descriptions with context-aware prompts for charts, screenshots, or general images.
  • Async processing: images are processed via a Celery task with retry support, with a new "analyzing" status shown while it happens.
  • EXIF orientation normalization: phone photos that rely on EXIF metadata for correct orientation now display correctly instead of sideways or upside down.

Limits & Configuration

Images are capped at 5MB (versus 100MB for documents), validated by MIME type. Minimum dimension clamping prevents errors on extreme aspect-ratio images.

Reliability

48+ new tests covering validation and preprocessing, vision model integration, Celery retry behavior, extreme aspect ratios, corrupt images, palette-mode conversion, S3 storage compatibility, and EXIF normalization.

Data Handling Still Applies

An uploaded image is treated the same as any other document for tier purposes. A screenshot containing client data or person-of-interest material is Tier 3 the moment it's uploaded — see Data Security before uploading anything beyond generic, public images.

ใหม่ใน SAM · คู่มือฟีเจอร์

อัปโหลดรูปภาพและการประมวลผลด้วย Vision

ตอนนี้ SAM มองเห็นภาพได้แล้ว อัปโหลดไฟล์ JPG, PNG, GIF หรือ WebP แล้ว vision model จะแปลงเป็นคำอธิบายข้อความที่โมเดลแชทใดก็ตามสามารถนำไปวิเคราะห์ต่อได้ — ทั้งกราฟ ภาพหน้าจอ และภาพทั่วไป ล้วนได้รับการจัดการอย่างเข้าใจบริบท

มีอะไรใหม่

  • การประมวลผลด้วย Vision Model: vision model จาก Fireworks.ai (Qwen3 VL 235B) แปลงรูปภาพที่อัปโหลดให้เป็นคำอธิบายข้อความ พร้อม prompt ที่เข้าใจบริบทสำหรับกราฟ ภาพหน้าจอ หรือภาพทั่วไป
  • การประมวลผลแบบ Async: รูปภาพถูกประมวลผลผ่าน Celery task ที่รองรับการลองใหม่ (retry) พร้อมสถานะ "กำลังวิเคราะห์" แบบใหม่ที่แสดงระหว่างดำเนินการ
  • การปรับทิศทางภาพตาม EXIF: ภาพถ่ายจากโทรศัพท์ที่พึ่งพา EXIF metadata สำหรับทิศทางที่ถูกต้อง ตอนนี้แสดงผลถูกต้องแล้ว แทนที่จะเอียงข้างหรือกลับหัว

ข้อจำกัดและการตั้งค่า

รูปภาพถูกจำกัดไว้ที่ 5MB (เทียบกับ 100MB สำหรับเอกสาร) และตรวจสอบด้วยประเภท MIME การจำกัดขนาดมิติขั้นต่ำช่วยป้องกันข้อผิดพลาดกับภาพที่มีอัตราส่วนภาพสุดโต่ง

ความน่าเชื่อถือ

การทดสอบใหม่มากกว่า 48 รายการ ครอบคลุมการตรวจสอบและการประมวลผลล่วงหน้า การเชื่อมต่อ vision model พฤติกรรมการ retry ของ Celery อัตราส่วนภาพสุดโต่ง ภาพที่เสียหาย การแปลงโหมด palette ความเข้ากันได้กับ S3 storage และการปรับทิศทางตาม EXIF

กฎการจัดการข้อมูลยังคงบังคับใช้เหมือนเดิม

รูปภาพที่อัปโหลดจะถูกปฏิบัติเหมือนเอกสารอื่น ๆ ในแง่ของการจัดชั้นข้อมูล ภาพหน้าจอที่มีข้อมูลลูกค้าหรือข้อมูล person-of-interest จัดเป็น Tier 3 ทันทีที่อัปโหลด — ดู Data Security ก่อนอัปโหลดสิ่งใดก็ตามที่เกินกว่าภาพทั่วไปสาธารณะ

ပုံတင်ခြင်းနှင့် Vision ဖြင့် ပုံဖတ်ခြင်း

SAM သည် ယခုအခါ ပုံများကို မြင်နိုင်ပြီ။ JPG, PNG, GIF သို့မဟုတ် WebP ပုံဖိုင်ကို တင်လိုက်ပါက vision model တစ်ခုသည် ၎င်းကို chat model မည်သည့်ခြင်းမဆို ဆင်ခြင်တွေးခေါ်နိုင်သည့် စာသားဖော်ပြချက်အဖြစ် ပြောင်းလဲပေးသည် — chart၊ screenshot နှင့် ပုံအထွေထွေအားလုံးအတွက် context ကို နားလည်သော ကိုင်တွယ်မှု ရရှိသည်။

အသစ်ပါဝင်သည့်အရာများ

  • Vision Model ဖြင့် ပုံဖတ်ခြင်း: Fireworks.ai ၏ vision model (Qwen3 VL 235B) သည် တင်ထားသော ပုံများကို chart၊ screenshot သို့မဟုတ် ပုံအထွေထွေအတွက် context သိသော prompt များဖြင့် စာသားဖော်ပြချက်များအဖြစ် ပြောင်းလဲပေးသည်
  • Async ကိုင်တွယ်မှု: ပုံများကို retry ပံ့ပိုးထားသော Celery task မှတစ်ဆင့် ကိုင်တွယ်ပြီး လုပ်ဆောင်နေစဉ် "analyzing" ဟူသော status အသစ်ကို ပြသသည်
  • EXIF orientation ပုံမှန်ပြုပြင်ခြင်း: မှန်ကန်သော ဦးတည်ချက်အတွက် EXIF metadata အပေါ် မှီခိုနေသော ဖုန်းဓာတ်ပုံများ ယခုအခါ ဘေးတိုက် သို့မဟုတ် အောက်ခြေမှ အပေါ်သို့ မဟုတ်တော့ဘဲ မှန်ကန်စွာ ပြသပါပြီ

ကန့်သတ်ချက်များနှင့် ပြင်ဆင်သတ်မှတ်ချက်

ပုံများကို 5MB ဖြင့် ကန့်သတ်ထားသည် (စာတမ်းများအတွက် 100MB နှင့် နှိုင်းယှဉ်ပါက) MIME အမျိုးအစားဖြင့် အတည်ပြုသည်။ အနည်းဆုံးအရွယ်အစား clamping က ပုံအချိုးအစား အလွန်အကျွံရှိသော ပုံများတွင် error ဖြစ်ခြင်းကို ကာကွယ်ပေးသည်

ယုံကြည်စိတ်ချရမှု

Test အသစ် ၄၈ ခုကျော် ဖြင့် validation နှင့် preprocessing၊ vision model ပေါင်းစည်းမှု၊ Celery retry အပြုအမူ၊ ပုံအချိုးအစား အလွန်အကျွံရှိသော ပုံများ၊ ပျက်စီးနေသော ပုံများ၊ palette-mode ပြောင်းလဲမှု၊ S3 storage ဟန်ချက်ညီမှုနှင့် EXIF ပုံမှန်ပြုပြင်ခြင်းတို့ကို ခြုံငုံစစ်ဆေးထားသည်

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေသည်

Tier သတ်မှတ်ရာတွင် တင်ထားသော ပုံတစ်ပုံကို အခြားစာတမ်းတစ်စောင်နည်းတူ ဆက်ဆံသည်။ ဖောက်သည်ဒေတာ သို့မဟုတ် person-of-interest အချက်အလက်ပါသော screenshot တစ်ခုသည် တင်လိုက်သည်နှင့် Tier 3 ချက်ချင်း ဖြစ်သွားသည် — အထွေထွေ အများပြည်သူပုံများထက် ပိုသော မည်သည့်အရာမဆို မတင်မီ Data Security ကို ကြည့်ပါ
