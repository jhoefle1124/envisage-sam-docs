Prompt Template Library

SAM is getting a built-in library of admin-managed prompt templates for common analysis tasks — organized by category, written in markdown, and built around fill-in-the-blank placeholders like {{Company}}. This page explains how the library works and how to use it well.

What Are Prompt Templates?

Prompt templates are pre-written, reusable prompts for recurring analysis work — things like company overviews, investment theses, or competitor comparisons — so analysts don't have to rebuild the same prompt structure from scratch every time. Templates are:

  • Globally managed by site admins through the custom admin interface, not created ad hoc by individual users.
  • Written in markdown with a proper editor, so formatting (headings, bold, bullet lists) carries through into the final prompt.
  • Organized by category — e.g. investment research, competitive analysis — so they're easy to browse and discover in SAM.
  • Available to all users once published, with admins controlling which templates are active and in what order they appear.

How Placeholders Work

Templates use double-curly-brace placeholders that you fill in before running the prompt. Placeholders are detected automatically when a template is saved and are matched case-insensitively, so {{Company}} and {{company}} behave the same way.

PlaceholderMeaningExample
{{Company}}A single company selection{{Company}} → Petronas
{{Companies}}Multiple companies, comma-separated{{Companies}} → Petronas, Shell, TotalEnergies
{{Date}}Auto-filled with the current date{{Date}} → today's date
{{User}}Auto-filled with the current user's name{{User}} → your name

Any word works as a placeholder. Template authors aren't limited to the four above — {{Competitor1}}, {{Sector}}, or any other {{...}} pattern is recognized the same way. The four listed are the ones SAM understands well enough to offer smart replacement (company lookups, auto-filled date/user); anything else is a plain fill-in-the-blank.

Example Templates

Investment Thesis Template

Category: Investment Research
## Background and Thesis for {{Company}}

Write a detailed overview of {{Company}}, tailored for long-term
fundamental investors with an investment horizon of 3-5 years.
The summary should include:

- Company Description: primary business activities, key products
  or services, market focus, and geographical presence, broken
  down by segment and geography where possible.
- Investment Thesis: why the company is attractive for a 3-5 year
  horizon — structural growth drivers, competitive advantages,
  market positioning, alignment with global trends.
- Risk: macro risks, competitive dynamics, regulatory concerns,
  company-specific operational or financial risks.
- Ownership: shareholder structure — insider, institutional,
  strategic investors, and free float.

Where to Find Them in SAM

Templates live in a library inside the chatbot interface, grouped by category so you can browse rather than hunt. Selecting a template drops the markdown into your prompt with its placeholders intact — you replace {{Company}}, {{Competitor1}}, and similar tokens with real values before sending.

Rolling out in stages. The template library and admin management tools are backend infrastructure landing first. The in-chat selection modal, company autocomplete for placeholders, and real-time preview are planned as a follow-on frontend release — check with your team lead on current availability before relying on it for live work.

Data Handling Still Applies

A blank template is Tier 1 — it's boilerplate structure with no ENVISAGE-specific content. What you fill into {{Company}} and the placeholders around it determines the tier of the resulting prompt, exactly as with any other AI input.

Usually fine

Public company name only

Filling {{Company}} with a publicly known company name for a general research overview keeps the prompt close to Tier 1.

Check first

Client or engagement framing added

If the filled-in prompt reveals which client asked for the analysis, why, or ties it to a live engagement, treat it as Tier 2/3 and route it accordingly — see Data Security.

คลัง Prompt Template

SAM กำลังจะมีคลัง prompt template ที่ดูแลโดยแอดมินในตัว สำหรับงานวิเคราะห์ที่ทำเป็นประจำ — จัดหมวดหมู่ เขียนด้วย markdown และสร้างขึ้นจาก placeholder แบบเติมคำในช่องว่าง เช่น {{Company}} หน้านี้อธิบายวิธีการทำงานของคลังนี้และวิธีใช้งานให้ได้ผลดี

Prompt Template คืออะไร

Prompt template คือ prompt ที่เขียนไว้ล่วงหน้าและนำกลับมาใช้ซ้ำได้ สำหรับงานวิเคราะห์ที่เกิดขึ้นซ้ำ ๆ เช่น ภาพรวมบริษัท investment thesis หรือการเปรียบเทียบคู่แข่ง — เพื่อให้นักวิเคราะห์ไม่ต้องสร้างโครงสร้าง prompt แบบเดียวกันขึ้นมาใหม่ทุกครั้ง Template มีลักษณะดังนี้:

  • บริหารจัดการโดยแอดมินของไซต์ในระดับส่วนกลาง ผ่านหน้า admin interface ที่สร้างขึ้นเฉพาะ ไม่ใช่สิ่งที่ผู้ใช้แต่ละคนสร้างขึ้นเองตามใจ
  • เขียนด้วย markdown ผ่านตัวแก้ไขที่เหมาะสม ทำให้การจัดรูปแบบ (หัวข้อ ตัวหนา bullet list) ยังคงอยู่ใน prompt สุดท้าย
  • จัดหมวดหมู่ — เช่น investment research, competitive analysis — เพื่อให้ค้นหาและเรียกดูใน SAM ได้ง่าย
  • ใช้งานได้กับผู้ใช้ทุกคน เมื่อเผยแพร่แล้ว โดยแอดมินเป็นผู้ควบคุมว่า template ใดเปิดใช้งานอยู่และเรียงลำดับการแสดงผลอย่างไร

Placeholder ทำงานอย่างไร

Template ใช้ placeholder แบบวงเล็บปีกกาคู่ที่คุณเติมข้อมูลก่อนรัน prompt ระบบจะตรวจจับ placeholder โดยอัตโนมัติเมื่อบันทึก template และจับคู่โดยไม่สนตัวพิมพ์เล็ก-ใหญ่ ดังนั้น {{Company}} และ {{company}} จะทำงานเหมือนกัน

Placeholderความหมายตัวอย่าง
{{Company}}การเลือกบริษัทเดียว{{Company}} → Petronas
{{Companies}}หลายบริษัท คั่นด้วยจุลภาค{{Companies}} → Petronas, Shell, TotalEnergies
{{Date}}เติมวันที่ปัจจุบันให้อัตโนมัติ{{Date}} → วันที่ของวันนี้
{{User}}เติมชื่อผู้ใช้ปัจจุบันให้อัตโนมัติ{{User}} → ชื่อของคุณ

คำใดก็ได้สามารถใช้เป็น placeholder ได้ ผู้เขียน template ไม่ได้จำกัดอยู่แค่สี่แบบข้างต้น — {{Competitor1}}, {{Sector}} หรือรูปแบบ {{...}} อื่นใดก็ถูกจดจำในลักษณะเดียวกัน สี่แบบที่ระบุไว้เป็นแบบที่ SAM เข้าใจดีพอที่จะเสนอการแทนที่อัจฉริยะ (การค้นหาบริษัท เติมวันที่/ผู้ใช้อัตโนมัติ) ส่วนแบบอื่น ๆ เป็นเพียงช่องว่างธรรมดาให้เติมคำ

จะหาได้ที่ไหนใน SAM

Template อยู่ในคลังภายในหน้า chatbot interface จัดกลุ่มตามหมวดหมู่เพื่อให้เรียกดูได้แทนที่จะต้องค้นหาเอง การเลือก template จะวางเนื้อหา markdown ลงใน prompt ของคุณพร้อม placeholder ที่ยังคงอยู่ครบ — คุณต้องแทนที่ {{Company}}, {{Competitor1}} และ token ที่คล้ายกันด้วยค่าจริงก่อนส่ง

ทยอยเปิดใช้งานเป็นระยะ คลัง template และเครื่องมือบริหารจัดการสำหรับแอดมินเป็นโครงสร้างพื้นฐานฝั่ง backend ที่จะออกมาก่อน ส่วน modal เลือก template ในหน้าแชท การเติมชื่อบริษัทอัตโนมัติสำหรับ placeholder และการแสดงตัวอย่างแบบเรียลไทม์ มีแผนจะตามมาในรุ่น frontend ถัดไป — ตรวจสอบกับหัวหน้าทีมของคุณเกี่ยวกับความพร้อมใช้งานปัจจุบันก่อนพึ่งพาใช้งานจริง

การจัดการข้อมูลยังคงบังคับใช้เช่นเดิม

Template เปล่าคือ Tier 1 — เป็นเพียงโครงสร้างต้นแบบที่ไม่มีเนื้อหาเฉพาะของ ENVISAGE สิ่งที่คุณเติมลงใน {{Company}} และ placeholder โดยรอบเป็นตัวกำหนด tier ของ prompt ที่ได้ เช่นเดียวกับ input อื่น ๆ ที่ป้อนให้ AI

Prompt Template Library

SAM သည် ပုံမှန်ဖြစ်ပေါ်လေ့ရှိသော ခွဲခြမ်းစိတ်ဖြာမှု လုပ်ငန်းများအတွက် admin မှ စီမံခန့်ခွဲသော prompt template များ စုစည်းထားသည့် built-in library တစ်ခုကို ရရှိတော့မည် — အမျိုးအစားအလိုက် စုစည်းထားပြီး markdown ဖြင့် ရေးသားထားကာ {{Company}} ကဲ့သို့သော ဖြည့်စွက်ရန် placeholder များပေါ်တွင် တည်ဆောက်ထားသည်။ ဤစာမျက်နှာသည် library ဘယ်လိုအလုပ်လုပ်သည်နှင့် ကောင်းစွာ အသုံးချနည်းကို ရှင်းပြသည်။

Prompt Template ဆိုတာ ဘာလဲ

Prompt Template များသည် ကုမ္ပဏီ overview၊ investment thesis သို့မဟုတ် ပြိုင်ဘက်နှိုင်းယှဉ်ချက် ကဲ့သို့ ထပ်ခါထပ်ခါ ဖြစ်ပေါ်သော ခွဲခြမ်းစိတ်ဖြာမှု လုပ်ငန်းများအတွက် ကြိုတင်ရေးသားထားပြီး ထပ်မံအသုံးပြုနိုင်သော prompt များ ဖြစ်သည် — ၎င်းကြောင့် analyst များသည် တူညီသော prompt ဖွဲ့စည်းပုံကို အကြိမ်တိုင်း အသစ်ပြန်တည်ဆောက်စရာ မလိုတော့ပါ။ Template များသည် —

  • Site admin များက ဗဟိုချုပ်ကိုင် စီမံခန့်ခွဲသည် custom admin interface မှတစ်ဆင့် ဖြစ်ပြီး၊ တစ်ဦးချင်းစီ user များက လိုအလိုလို ဖန်တီးသည် မဟုတ်ပါ။
  • Markdown ဖြင့် ရေးသား သင့်လျော်သော editor ဖြင့် ဖြစ်သဖြင့် formatting (heading, bold, bullet list) များသည် နောက်ဆုံး prompt ထဲသို့ ဆက်လက်ရောက်ရှိသည်။
  • အမျိုးအစားအလိုက် စုစည်းထား — ဥပမာ investment research, competitive analysis — SAM ထဲတွင် ရှာဖွေကြည့်ရှုရ လွယ်ကူစေရန်။
  • ထုတ်ပြန်ပြီးပါက user အားလုံး အသုံးပြုနိုင် admin များက မည်သည့် template များကို active ဖြစ်စေမည်၊ မည်သည့်အစီအစဉ်ဖြင့် ပေါ်စေမည်ကို ထိန်းချုပ်သည်။

Placeholder များ မည်သို့ အလုပ်လုပ်သနည်း

Template များသည် prompt ကို run မလုပ်မီ ဖြည့်စွက်ရမည့် curly-brace နှစ်ထပ် placeholder များကို သုံးသည်။ Template ကို သိမ်းဆည်းသည့်အခါ placeholder များကို အလိုအလျောက် ရှာဖွေတွေ့ရှိပြီး စာလုံးအကြီးအသေး ခွဲခြားခြင်းမရှိဘဲ ကိုက်ညီစေသည်၊ ထို့ကြောင့် {{Company}} နှင့် {{company}} သည် အလားတူ အလုပ်လုပ်သည်။

Placeholderအဓိပ္ပာယ်ဥပမာ
{{Company}}ကုမ္ပဏီတစ်ခု ရွေးချယ်မှု{{Company}} → Petronas
{{Companies}}ကုမ္ပဏီများစွာ ကော်မာဖြင့် ပိုင်းခြား{{Companies}} → Petronas, Shell, TotalEnergies
{{Date}}လက်ရှိရက်စွဲကို အလိုအလျောက် ဖြည့်ပေး{{Date}} → ယနေ့ရက်စွဲ
{{User}}လက်ရှိ user အမည်ကို အလိုအလျောက် ဖြည့်ပေး{{User}} → သင့်အမည်

မည်သည့်စကားလုံးမဆို placeholder အဖြစ် သုံးနိုင်သည်။ Template ရေးသားသူများသည် အထက်ပါ လေးခုတွင်သာ ကန့်သတ်မထားပါ — {{Competitor1}}, {{Sector}} သို့မဟုတ် အခြား {{...}} ပုံစံ မည်သည်ကိုမဆို အလားတူ မှတ်သားနိုင်သည်။ ဖော်ပြထားသော လေးခုသည် SAM က ကောင်းစွာ နားလည်ပြီး smart replacement (ကုမ္ပဏီရှာဖွေမှု၊ ရက်စွဲ/user အလိုအလျောက်ဖြည့်ခြင်း) ပေးနိုင်သော အရာများ ဖြစ်ပြီး ကျန်အရာများမှာ ရိုးရိုး fill-in-the-blank ဖြစ်သည်။

SAM ထဲတွင် ဘယ်နေရာမှာ ရှာနိုင်သနည်း

Template များသည် chatbot interface အတွင်း library တစ်ခုတွင် ရှိပြီး၊ အမျိုးအစားအလိုက် အုပ်စုဖွဲ့ထားသဖြင့် ရှာဖွေရမည့်အစား ကြည့်ရှုရွေးချယ်နိုင်သည်။ Template တစ်ခုကို ရွေးချယ်လိုက်လျှင် markdown ကို placeholder များ ပြည့်စုံစွာ ပါဝင်လျက် သင့် prompt ထဲသို့ ချထည့်ပေးသည် — {{Company}}, {{Competitor1}} နှင့် အလားတူ token များကို ပို့မီ တကယ့်တန်ဖိုးများဖြင့် အစားထိုးရမည်။

အဆင့်ဆင့် ထွက်ရှိနေသည်။ Template library နှင့် admin management tool များသည် ဦးစွာ ရောက်ရှိလာမည့် backend infrastructure ဖြစ်သည်။ In-chat selection modal၊ placeholder များအတွက် ကုမ္ပဏီ autocomplete နှင့် real-time preview တို့ကို နောက်ဆက်တွဲ frontend release တွင် ထည့်သွင်းရန် စီစဉ်ထားသည် — live အလုပ်တွင် အားထားမီ သင့်အဖွဲ့ခေါင်းဆောင်နှင့် လက်ရှိရရှိနိုင်မှုကို စစ်ဆေးပါ။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်

အလွတ် template တစ်ခုသည် Tier 1 ဖြစ်သည် — ENVISAGE-ဆိုင်ရာ အကြောင်းအရာ မပါဝင်သော boilerplate ဖွဲ့စည်းပုံသာ ဖြစ်သည်။ {{Company}} နှင့် ၎င်းပတ်ဝန်းကျင်ရှိ placeholder များထဲသို့ သင်ဖြည့်စွက်လိုက်သည့်အရာက ရလဒ် prompt ၏ tier ကို အခြား AI input များနည်းတူ ဆုံးဖြတ်ပေးသည်။

ပုံမှန်အားဖြင့် ပြဿနာမရှိ

အများပြည်သူသိ ကုမ္ပဏီအမည်သာ

{{Company}} ကို ယေဘုယျ သုတေသန overview အတွက် အများသိ ကုမ္ပဏီအမည်ဖြင့် ဖြည့်ခြင်းသည် prompt ကို Tier 1 အနီးတွင် ထားနိုင်သည်။

အရင်စစ်ဆေးပါ

ဖောက်သည် သို့မဟုတ် ပရောဂျက် framing ထပ်ပေါင်း

ဖြည့်စွက်ထားသော prompt သည် မည်သည့်ဖောက်သည်က ဤခွဲခြမ်းစိတ်ဖြာမှုကို တောင်းဆိုသည်၊ ဘာကြောင့်၊ သို့မဟုတ် လက်ရှိလုပ်ဆောင်နေသော engagement နှင့် ဆက်စပ်ကြောင်း ဖော်ပြပါက Tier 2/3 အဖြစ် သတ်မှတ်ပြီး လျော်ညီစွာ ပို့ဆောင်ပါ — Data Security ကို ကြည့်ပါ။
