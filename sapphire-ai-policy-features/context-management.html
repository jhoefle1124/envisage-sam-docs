Context Budget & Model Updates

The engineering behind SAM's context window management — a 769-line budget tracker, batched history summarization, model upgrades to GLM-4.6 and DeepSeek v3.1, and a breaking change to the company financials batch limit.

Context Window Management

  • Context budget tracking: monitors conversation size in real time and manages it before limits are hit, with provider-aware token estimation using tiktoken.
  • History summarization: long conversations are summarized in batches rather than cut off, with sanitized messages persisted so summaries don't get reinjected on the next turn.
  • Graceful overflow handling: a clear, actionable message instead of a silent or confusing failure when a conversation still exceeds the limit. See Context Management Enhancements for the full user-facing writeup.

Model & Configuration Updates

ChangeDetail
Default chat modelGLM-4.5 → GLM-4.6
DeepSeekUpgraded to v3.1 terminus
Reasoning effortNew configurable parameter for chat provider settings

Breaking change: company financials batch limit reduced from 5 to 3 companies per request. Workflows assuming a 5-company batch need to request in smaller groups.

Reliability

1,482 lines of context budget tests, 358 lines of error-handling tests, and 219 lines of summarizer tests. Migration 0013_add_context_budget_to_session.py included; no manual data migration required.

Data Handling Still Applies

Summarization manages conversation size, not sensitivity — a summarized conversation can still carry Tier 2/3 content if the original did. See Data Security.

งบประมาณบริบทและการอัปเดตโมเดล

งานด้านวิศวกรรมเบื้องหลังการจัดการหน้าต่างบริบท (context window) ของ SAM — ตัวติดตามงบประมาณจำนวน 769 บรรทัด การสรุปประวัติการสนทนาแบบเป็นชุด (batch) การอัปเกรดโมเดลเป็น GLM-4.6 และ DeepSeek v3.1 และการเปลี่ยนแปลงที่ส่งผลกระทบ (breaking change) ต่อขีดจำกัดการดึงข้อมูลการเงินบริษัทแบบเป็นชุด

การจัดการหน้าต่างบริบท

  • การติดตามงบประมาณบริบท: ตรวจสอบขนาดของบทสนทนาแบบเรียลไทม์และจัดการก่อนที่จะถึงขีดจำกัด โดยใช้การประมาณจำนวนโทเคนที่คำนึงถึงผู้ให้บริการ (provider) ด้วย tiktoken
  • การสรุปประวัติการสนทนา: บทสนทนาที่ยาวจะถูกสรุปเป็นชุด (batch) แทนที่จะถูกตัดทิ้ง โดยข้อความที่ผ่านการทำความสะอาดแล้ว (sanitized) จะถูกบันทึกไว้ เพื่อไม่ให้สรุปเหล่านั้นถูกใส่กลับเข้าไปใหม่ในรอบถัดไป
  • การจัดการ overflow อย่างเหมาะสม: แสดงข้อความที่ชัดเจนและสามารถนำไปปฏิบัติต่อได้ แทนที่จะปล่อยให้ล้มเหลวแบบเงียบหรือสร้างความสับสน เมื่อบทสนทนายังคงเกินขีดจำกัด ดูรายละเอียดฉบับเต็มสำหรับผู้ใช้งานได้ที่ การปรับปรุงการจัดการบริบท

การเปลี่ยนแปลงรายละเอียด

ความพยายามในการใช้เหตุผล (Reasoning effort)พารามิเตอร์ใหม่ที่ปรับตั้งค่าได้สำหรับการตั้งค่าผู้ให้บริการแชท

การเปลี่ยนแปลงที่ส่งผลกระทบ (Breaking change): ขีดจำกัดการดึงข้อมูลการเงินบริษัทแบบเป็นชุดลดลงจาก 5 บริษัทเหลือ 3 บริษัทต่อคำขอ workflow ที่ตั้งสมมติฐานว่าดึงได้ครั้งละ 5 บริษัทจำเป็นต้องแบ่งคำขอเป็นกลุ่มย่อยลง

ความน่าเชื่อถือ

1,482 บรรทัด ของการทดสอบงบประมาณบริบท 358 บรรทัด ของการทดสอบการจัดการข้อผิดพลาด และ 219 บรรทัด ของการทดสอบตัวสรุปความ มี migration 0013_add_context_budget_to_session.py รวมอยู่ด้วย ไม่จำเป็นต้อง migrate ข้อมูลด้วยตนเอง

กฎการจัดการข้อมูลยังคงมีผลบังคับใช้

การสรุปความจัดการเรื่องขนาดของบทสนทนา ไม่ใช่ระดับความอ่อนไหวของข้อมูล — บทสนทนาที่ถูกสรุปแล้วยังคงมีเนื้อหาระดับ Tier 2/3 ได้ หากต้นฉบับมีเนื้อหาเช่นนั้นอยู่ ดู Data Security

Context Budget နှင့် Model အပ်ဒိတ်များ

SAM ၏ context window စီမံခန့်ခွဲမှု နောက်ကွယ်ရှိ engineering — 769 line budget tracker, batch အလိုက် history summarization, GLM-4.6 နှင့် DeepSeek v3.1 သို့ model အဆင့်မြှင့်တင်ခြင်း၊ company financials batch ကန့်သတ်ချက်၏ breaking change တစ်ခု။

Context Window စီမံခန့်ခွဲမှု

  • Context budget tracking: tiktoken ကို အသုံးပြု၍ provider-aware token estimation ဖြင့် ဆွေးနွေးမှု အရွယ်အစားကို real-time စောင့်ကြည့်ပြီး ကန့်သတ်ချက်မထိမီ စီမံပေးသည်
  • History summarization: ဆွေးနွေးမှု ရှည်လျားသည်များကို ဖြတ်တောက်မည့်အစား batch အလိုက် အနှစ်ချုပ်ပေးပြီး၊ summary များ နောက်တစ်ကြိမ်တွင် ပြန်မထည့်စေရန် သန့်ရှင်းပြီးသား message များကို သိမ်းဆည်းထားသည်
  • Overflow ကို သင့်လျော်စွာ ကိုင်တွယ်ခြင်း: ဆွေးနွေးမှုသည် ကန့်သတ်ချက်ထက် ကျော်လွန်နေဆဲဖြစ်လျှင် ဆိတ်ငြိမ်သော သို့မဟုတ် ရှုပ်ထွေးစေသော failure အစား ရှင်းလင်းပြီး လုပ်ဆောင်နိုင်သော message ကို ပြသသည်။ User-facing အသေးစိတ်အတွက် Context Management တိုးတက်မှုများ တွင် ကြည့်ပါ။
