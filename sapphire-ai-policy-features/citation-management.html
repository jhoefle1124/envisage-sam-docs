New in SAM · Feature Guide

Citation System Launch

SAM responses now carry transparent source attribution — clickable citations for uploaded documents, a single descriptive citation for database queries, and publication-dated citations for news, so you can verify a claim without hunting for where it came from.

What's New

  • Document citations: clickable superscript numbers (e.g. [1], [2]) that jump straight to the source document; each document gets a unique number per conversation.
  • Database citations: a single descriptive citation per query (e.g. "Data from SAM database: stock_data tables") instead of clutter.
  • News citations: enhanced with the source's actual publication date, so you can judge how current the information is.

How It Works

A shared custom_tool_wrapper attaches citations to tool responses behind a global citation counter, so numbers stay unique across multiple tool calls in the same response. The processor accepts both structured citation objects and plain dictionaries for compatibility, then renders them into a user-friendly format in the chat UI.

MetricResult
Citation processing overhead< 5ms per response
Memory usage~1KB per citation
Test coverage78% → 85%

Reliability

Zero-downtime, feature-flagged rollout. No database migration required, fully backward compatible with existing chat sessions, and toggleable via an ENABLE_CITATIONS setting if issues arise. Alongside this, a session-history persistence bug and a test-isolation bug (both unrelated to citations) were fixed in the same pass.

Data Handling Still Applies

A citation is a pointer to a source, not a change in that source's sensitivity. A citation to Tier 3 SAM data is still Tier 3 — see Data Security for the underlying rules.

ใหม่ใน SAM · คู่มือฟีเจอร์

การเปิดตัวระบบ Citation

ขณะนี้คำตอบของ SAM มาพร้อมการอ้างอิงแหล่งที่มาที่โปร่งใส — citation ที่คลิกได้สำหรับเอกสารที่อัปโหลด, citation เชิงพรรณนาเพียงรายการเดียวสำหรับการ query ฐานข้อมูล และ citation ที่ระบุวันที่เผยแพร่สำหรับข่าว เพื่อให้คุณตรวจสอบข้อความอ้างสิทธิ์ได้โดยไม่ต้องไล่หาว่าข้อมูลนั้นมาจากไหน

มีอะไรใหม่

  • การอ้างอิงเอกสาร: ตัวเลขตัวยกที่คลิกได้ (เช่น [1], [2]) ซึ่งพาไปยังเอกสารต้นฉบับโดยตรง เอกสารแต่ละฉบับจะได้รับหมายเลขเฉพาะของตนเองในแต่ละบทสนทนา
  • การอ้างอิงฐานข้อมูล: citation เชิงพรรณนาเพียงรายการเดียวต่อหนึ่ง query (เช่น "Data from SAM database: stock_data tables") แทนที่จะแสดงผลรกเกะกะเหมือนก่อน
  • การอ้างอิงข่าว: ปรับปรุงให้แสดงวันที่เผยแพร่จริงของแหล่งข่าว เพื่อให้คุณประเมินได้ว่าข้อมูลนั้นทันสมัยเพียงใด

วิธีการทำงาน

custom_tool_wrapper ที่ใช้ร่วมกันจะแนบ citation เข้ากับการตอบกลับของ tool โดยอิงตามตัวนับ citation ส่วนกลาง (global citation counter) ทำให้หมายเลขไม่ซ้ำกันแม้จะมีการเรียก tool หลายครั้งในคำตอบเดียวกัน ตัวประมวลผลรองรับทั้ง citation object ที่มีโครงสร้างและ dictionary ธรรมดาเพื่อความเข้ากันได้ ก่อนจะแสดงผลออกมาในรูปแบบที่ใช้งานง่ายบน chat UI

ตัวชี้วัดผลลัพธ์
ภาระการประมวลผล citationน้อยกว่า 5ms ต่อการตอบกลับหนึ่งครั้ง
การใช้หน่วยความจำประมาณ 1KB ต่อ citation หนึ่งรายการ
ความครอบคลุมของการทดสอบ78% → 85%

ความน่าเชื่อถือ

เปิดใช้งานแบบไม่มี downtime ผ่านระบบ feature flag ไม่ต้อง migrate ฐานข้อมูล เข้ากันได้ (backward compatible) กับ chat session เดิมทั้งหมด และสามารถเปิด/ปิดได้ผ่านการตั้งค่า ENABLE_CITATIONS หากเกิดปัญหา นอกจากนี้ยังมีการแก้ไขบั๊กเรื่องการเก็บรักษาประวัติ session (session-history persistence) และบั๊กเรื่อง test isolation (ทั้งสองอย่างไม่เกี่ยวข้องกับ citation) ในการปรับปรุงรอบเดียวกันนี้ด้วย

หลักการจัดการข้อมูลยังคงบังคับใช้

Citation คือตัวชี้ไปยังแหล่งที่มา ไม่ใช่การเปลี่ยนระดับความอ่อนไหวของแหล่งที่มานั้น citation ที่อ้างถึงข้อมูล SAM ระดับ Tier 3 ก็ยังคงเป็น Tier 3 อยู่เช่นเดิม — ดู Data Security สำหรับกฎเกณฑ์พื้นฐานที่เกี่ยวข้อง

SAM တွင် အသစ် · ဝန်ဆောင်မှု လမ်းညွှန်

Citation System စတင်မိတ်ဆက်ခြင်း

SAM ၏ ပြန်ကြားချက်များတွင် ယခုအခါ ပွင့်လင်းသော source attribution ပါဝင်လာပြီ — အပ်လုဒ်လုပ်ထားသည့် စာရွက်စာတမ်းများအတွက် နှိပ်နိုင်သော citation များ၊ ဒေတာဘေ့စ် query များအတွက် ဖော်ပြချက်ပါ citation တစ်ခုတည်း၊ နှင့် သတင်းများအတွက် ထုတ်ဝေသည့်ရက်စွဲပါ citation များ ပါဝင်သဖြင့် အချက်အလက်တစ်ခု မည်သည့်နေရာမှ လာသည်ကို ရှာနေစရာမလိုဘဲ စစ်ဆေးနိုင်ပါပြီ။

အသစ်ပါဝင်သည့်အရာများ

  • စာရွက်စာတမ်း Citation များ: နှိပ်နိုင်သော superscript နံပါတ်များ (ဥပမာ [1], [2]) ဖြစ်ပြီး မူရင်း document သို့ တိုက်ရိုက်ပို့ဆောင်ပေးသည်။ document တစ်ခုစီသည် conversation တစ်ခုစီအလိုက် သီးသန့်နံပါတ်တစ်ခု ရရှိသည်။
  • ဒေတာဘေ့စ် Citation: query တစ်ခုစီအတွက် ဖော်ပြချက်ပါ citation တစ်ခုတည်း (ဥပမာ "Data from SAM database: stock_data tables") ဖြစ်ပြီး ရှုပ်ထွေးမှုကို လျှော့ချပေးသည်။
  • သတင်း Citation: source ၏ အမှန်တကယ် ထုတ်ဝေသည့်ရက်စွဲဖြင့် တိုးတက်ကောင်းမွန်စေထားပြီး၊ အချက်အလက်သည် မည်မျှ လတ်ဆတ်သည်ကို အကဲဖြတ်နိုင်စေသည်။

အလုပ်လုပ်ပုံ

custom_tool_wrapper တစ်ခုတည်းကို share လုပ်ပြီး tool ၏ response များတွင် global citation counter နောက်ကွယ်မှ citation များကို ပူးတွဲပေးသဖြင့် response တစ်ခုတည်းအတွင်း tool ကို အကြိမ်များစွာ ခေါ်ဆိုလျှင်ပင် နံပါတ်များ ထပ်နေခြင်း မရှိပါ။ processor သည် compatibility အတွက် structured citation object များနှင့် ရိုးရိုး dictionary များ နှစ်မျိုးလုံးကို လက်ခံပြီး၊ ထို့နောက် chat UI တွင် အသုံးပြုရလွယ်ကူသော format အဖြစ် render ပြုလုပ်ပေးသည်။

တိုင်းတာချက်ရလဒ်
Citation ပြုပြင်ဆောင်ရွက်မှု Overheadresponse တစ်ခုလျှင် 5ms အောက်
Memory အသုံးပြုမှုcitation တစ်ခုလျှင် ~1KB ခန့်
Test Coverage78% → 85%

ယုံကြည်စိတ်ချရမှု

Downtime မရှိဘဲ feature flag ဖြင့် rollout ပြုလုပ်ခြင်း။ Database migration မလိုအပ်ပါ၊ ရှိပြီးသား chat session များနှင့် အပြည့်အဝ backward compatible ဖြစ်ပြီး၊ ပြဿနာတစ်စုံတစ်ရာ ဖြစ်ပေါ်လာပါက ENABLE_CITATIONS setting မှတစ်ဆင့် ဖွင့်/ပိတ် လုပ်နိုင်သည်။ ၎င်းနှင့်အတူ session-history persistence ချို့ယွင်းချက်တစ်ခု နှင့် test-isolation ချို့ယွင်းချက်တစ်ခု (နှစ်ခုစလုံး citation နှင့် မသက်ဆိုင်ပါ) ကိုလည်း အလုပ်တစ်ခုတည်းတွင် ပြင်ဆင်ပြီးပါပြီ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

Citation တစ်ခုသည် source တစ်ခုသို့ ညွှန်ပြသော pointer တစ်ခုသာဖြစ်ပြီး ထို source ၏ sensitivity ကို ပြောင်းလဲစေခြင်း မဟုတ်ပါ။ Tier 3 SAM ဒေတာကို ညွှန်ပြသော citation သည်လည်း Tier 3 အဖြစ်ပင် ဆက်လက်ရှိနေဆဲဖြစ်သည် — အခြေခံစည်းမျဉ်းများအတွက် Data Security ကိုကြည့်ပါ။
