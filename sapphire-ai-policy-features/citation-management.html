New in SAM · Feature Guide

Citation System Launch

SAM responses now carry transparent source attribution — clickable citations for uploaded documents, a single descriptive citation for database queries, and publication-dated citations for news, so you can verify a claim without hunting for where it came from.

What's New

  • Document citations: clickable superscript numbers (e.g. [1], [2]) that jump straight to the source document; each document gets a unique number per conversation.
  • Database citations: a single descriptive citation per query (e.g. "Data from SAM database: stock_data tables") instead of clutter.
  • News citations: enhanced with the source's actual publication date, so you can judge how current the information is.

How It Works

A shared custom_tool_wrapper attaches citations to tool responses behind a global citation counter, so numbers stay unique across multiple tool calls in the same response. The processor accepts both structured citation objects and plain dictionaries for compatibility, then renders them into a user-friendly format in the chat UI.

MetricResult
Citation processing overhead< 5ms per response
Memory usage~1KB per citation
Test coverage78% → 85%

Reliability

Zero-downtime, feature-flagged rollout. No database migration required, fully backward compatible with existing chat sessions, and toggleable via an ENABLE_CITATIONS setting if issues arise. Alongside this, a session-history persistence bug and a test-isolation bug (both unrelated to citations) were fixed in the same pass.

Data Handling Still Applies

A citation is a pointer to a source, not a change in that source's sensitivity. A citation to Tier 3 SAM data is still Tier 3 — see Data Security for the underlying rules.
