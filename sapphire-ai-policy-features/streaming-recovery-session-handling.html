New in SAM · Technical Reference

Streaming Content Extraction

A robustness pass on how SAM extracts text from Claude's streaming responses — every content-block shape the API can return is now handled through one centralized path instead of scattered logic.

What's New

  • Robust content extraction: new shared utilities handle plain text, delta updates, lists of mixed content blocks, and more, all through one orchestrating extract_message_content() function.
  • Tool block detection: tool-related content (tool_use, tool_result, input_json_delta) is now properly filtered out of what you see in chat.
  • Cleaner thinking-tag removal: an optimized pattern strips both raw and HTML-encoded thinking tags in a single pass.

Reliability

46 new tests (27 for the LLM factory, 19 for streaming event processing), alongside the full existing suite, which continues to pass. See also Improving Anthropic Content and Complexity.

Data Handling Still Applies

This is a rendering reliability fix — it doesn't change what data flows through SAM. Standard Data Security rules continue to apply to everything typed into chat.

ใหม่ใน SAM · เอกสารอ้างอิงทางเทคนิค

การดึงเนื้อหาแบบสตรีมมิง

การปรับปรุงความทนทานของวิธีที่ SAM ดึงข้อความจากการตอบสนองแบบสตรีมมิงของ Claude — รูปแบบ content-block ทุกแบบที่ API สามารถส่งกลับมาได้ ตอนนี้ได้รับการจัดการผ่านเส้นทางศูนย์กลางเดียว แทนที่จะกระจัดกระจายอยู่ตามลอจิกต่าง ๆ

มีอะไรใหม่

  • การดึงเนื้อหาที่แข็งแกร่งขึ้น: ยูทิลิตีที่ใช้ร่วมกันตัวใหม่จัดการข้อความธรรมดา การอัปเดตแบบเดลต้า รายการของ content block ที่ผสมกัน และอื่น ๆ ทั้งหมดผ่านฟังก์ชันควบคุมกลางเดียวคือ extract_message_content()
  • การตรวจจับ tool block: เนื้อหาที่เกี่ยวข้องกับเครื่องมือ (tool_use, tool_result, input_json_delta) ตอนนี้ถูกกรองออกจากสิ่งที่คุณเห็นในแชทอย่างถูกต้องแล้ว
  • การลบแท็ก thinking ที่สะอาดขึ้น: รูปแบบที่ปรับให้เหมาะสมสามารถลบทั้งแท็ก thinking แบบดิบและแบบเข้ารหัส HTML ได้ในครั้งเดียว

ความน่าเชื่อถือ

การทดสอบใหม่ 46 รายการ (27 รายการสำหรับ LLM factory, 19 รายการสำหรับการประมวลผลเหตุการณ์สตรีมมิง) ควบคู่ไปกับชุดทดสอบเดิมทั้งหมดที่ยังคงผ่านการทดสอบ ดูเพิ่มเติมที่ การปรับปรุงเนื้อหาและความซับซ้อนของ Anthropic

หลักการจัดการข้อมูลยังคงใช้บังคับ

นี่คือการแก้ไขด้านความน่าเชื่อถือของการแสดงผล — ไม่ได้เปลี่ยนแปลงข้อมูลที่ไหลผ่าน SAM กฎมาตรฐานของ ความปลอดภัยข้อมูล ยังคงมีผลบังคับใช้กับทุกสิ่งที่พิมพ์ลงในแชท

SAM ရှိ အသစ် · နည်းပညာ ရည်ညွှန်းချက်

Streaming အကြောင်းအရာ ထုတ်ယူခြင်း

SAM သည် Claude ၏ streaming response များမှ စာသားထုတ်ယူပုံနည်းလမ်းအား ခိုင်မာအောင် ပြင်ဆင်ထားပြီး — API မှ ပြန်ပေးနိုင်သည့် content-block ပုံစံ အားလုံးကို ယခုအခါ ကွဲပြားနေသော logic များအစား ဗဟိုချုပ်ကိုင်သည့် လမ်းကြောင်းတစ်ခုတည်းမှတစ်ဆင့် ကိုင်တွယ်ထားသည်။

အသစ်ဘာတွေရှိလဲ

  • ခိုင်မာသော အကြောင်းအရာ ထုတ်ယူခြင်း: မျှဝေအသုံးပြုသည့် utility အသစ်များသည် extract_message_content() ဟူသော ဗဟိုကြပ်မတ်ဆောင်ရွက်သည့် function တစ်ခုတည်းမှတစ်ဆင့် ရိုးရိုးစာသား၊ delta update များ၊ ရောနှောထားသော content block စာရင်းများနှင့် အခြားအရာများကို ကိုင်တွယ်သည်။
  • Tool Block ရှာဖွေတွေ့ရှိခြင်း: tool နှင့်ဆိုင်သော အကြောင်းအရာ (tool_use, tool_result, input_json_delta) ကို chat တွင် မြင်ရသောအရာများမှ ယခုအခါ မှန်ကန်စွာ စစ်ထုတ်ပယ်ထုတ်ထားသည်။
  • ပိုမိုသန့်ရှင်းသော Thinking-Tag ဖယ်ရှားခြင်း: ပိုမိုကောင်းမွန်အောင် ပြုလုပ်ထားသော pattern တစ်ခုသည် ရိုးရိုး thinking tag နှင့် HTML-encoded thinking tag နှစ်မျိုးလုံးကို တစ်ကြိမ်တည်းနှင့် ဖယ်ရှားပေးသည်။

ယုံကြည်စိတ်ချရမှု

Test အသစ် ၄၆ ခု (LLM factory အတွက် ၂၇ ခု၊ streaming event ကိုင်တွယ်မှုအတွက် ၁၉ ခု) အပြင် လက်ရှိ test suite အပြည့်အစုံသည်လည်း ဆက်လက်၍ အောင်မြင်နေသေးသည်။ Anthropic အကြောင်းအရာနှင့် ရှုပ်ထွေးမှု တိုးတက်မှု ကိုလည်း ကြည့်ပါ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

ဤသည်မှာ ဖော်ပြမှု ယုံကြည်စိတ်ချရရေး ပြင်ဆင်ချက် တစ်ခုသာဖြစ်သည် — SAM ကို ဖြတ်သန်းသွားသော ဒေတာကို ပြောင်းလဲခြင်း မရှိပါ။ Chat တွင် ရိုက်ထည့်သမျှ အရာအားလုံးအတွက် standard ဒေတာလုံခြုံရေး စည်းမျဉ်းများ ဆက်လက် သက်ရောက်နေဆဲဖြစ်သည်။
