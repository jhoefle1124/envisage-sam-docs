New in SAM · Feature Guide

Citation, UI & Screener Improvements

A broad pass across three areas: citation accuracy and tool-specific attribution, a snappier streaming and auto-scroll experience, and a modernized stock screener with query complexity protection.

Citation System Enhancements

  • Tool-specific attribution: citations now carry prefixes like "SAM Database:" or "Stock Screener:" for clarity.
  • Generic SAM database citations: standardized to prevent duplication across a response.
  • Smart prioritization: database citations get lower numbers, which prevents the LLM from generating its own duplicate source list.
  • Safer dict handling: citations arriving as plain dictionaries no longer raise an AttributeError.

Performance & UX

Artificial streaming delays were removed, auto-scrolling is smoother during token streaming, and the LangGraph recursion limit was raised from 50 to 75 with graceful handling — including automatic title generation and response persistence when the limit is hit, so a long conversation never silently loses its answer.

Stock Screener Modernization

ChangeDetail
Modular architectureReorganized into a dedicated module with a field configuration registry and an extracted SQL builder class
Query complexity limitsAdded database protection through query complexity scoring
Result limitsMaximum results increased from 50 to 100
Quick filtersRemoved — an underutilized feature that added complexity

A new FinancialFormatter class standardizes financial data formatting across the app, and company_financials was migrated from raw SQL to Django ORM.

Critical Bug Fixes

Dividend yield was displaying 100x too high (41.69% instead of 0.42%) because the database stores yields as already-percentage values while the tool treated them as decimals. Fixed with corrected documentation and proper formatting functions per field.

Also fixed: a citation AttributeError that crashed multi-turn conversations when citations arrived as dictionaries instead of objects.

Data Handling Still Applies

Better citations and a modernized screener don't change data sensitivity. Screener results built from public company fundamentals stay close to Tier 1; results tied to a client portfolio or watchlist follow the same Tier 2/3 rules as everywhere else on Data Security.
