Broker Notes Date Filtering & Opus 4.5

Broker notes search gained date-range filtering and configurable result limits, alongside a cache key fix and an upgrade of the Claude Opus model powering the chatbot to version 4.5.

What's New

  • Date filtering: start_date and end_date parameters (YYYY-MM-DD) narrow results to a date range, defaulting to the last 12 months when unspecified.
  • Result limits: a new max_results parameter (1–50, default 10) controls how many results come back.
  • Validation: date range validation ensures the start date is on or before the end date, with tolerance for LLM-generated artifacts like quoted strings or stray whitespace.

Reliability Improvements

  • Null values in API responses (title, text, url) now fall back correctly instead of surfacing errors.
  • Cache keys now hash the entire JSON payload (query, result count, dates) instead of just the query string, preventing cache collisions.

Claude Opus was upgraded from 4.1 to 4.5, bringing improved reasoning and performance to the chatbot.

Reliability

20+ new unit tests covering date validation, range validation, result-limit bounds, null handling, and cache key generation. Backward compatible — existing calls without date or limit parameters keep working with sensible defaults.

Data Handling Still Applies

Filtering by date doesn't change what's safe to do with the results — see Broker Notes MCP Integration and Data Security.

การกรองวันที่ของ Broker Notes และ Opus 4.5

ระบบค้นหา broker notes ได้เพิ่มการกรองตามช่วงวันที่และการกำหนดจำนวนผลลัพธ์สูงสุดที่ปรับได้ พร้อมทั้งแก้ไขปัญหา cache key และอัปเกรดโมเดล Claude Opus ที่ขับเคลื่อนแชทบอทเป็นเวอร์ชัน 4.5

มีอะไรใหม่

  • การกรองตามวันที่: พารามิเตอร์ start_date และ end_date (รูปแบบ YYYY-MM-DD) ใช้จำกัดผลลัพธ์ให้อยู่ในช่วงวันที่ที่กำหนด โดยค่าเริ่มต้นจะใช้ช่วง 12 เดือนล่าสุดหากไม่ได้ระบุไว้
  • การจำกัดจำนวนผลลัพธ์: พารามิเตอร์ใหม่ max_results (1–50 ค่าเริ่มต้น 10) ใช้ควบคุมจำนวนผลลัพธ์ที่ส่งกลับมา
  • การตรวจสอบความถูกต้อง: การตรวจสอบช่วงวันที่ช่วยให้แน่ใจว่าวันที่เริ่มต้นต้องมาก่อนหรือเท่ากับวันที่สิ้นสุด โดยรองรับความคลาดเคลื่อนที่เกิดจาก LLM เช่น ข้อความที่มีเครื่องหมายคำพูดติดมาหรือช่องว่างเกิน

การปรับปรุงความน่าเชื่อถือ

  • ค่า null ในการตอบกลับจาก API (title, text, url) ขณะนี้จะ fallback ได้อย่างถูกต้องแทนที่จะแสดงข้อผิดพลาด
  • Cache key ขณะนี้จะ hash ข้อมูล JSON ทั้งหมด (query จำนวนผลลัพธ์ และวันที่) แทนที่จะใช้แค่ query string เพียงอย่างเดียว ซึ่งช่วยป้องกันการชนกันของ cache

Claude Opus ได้รับการอัปเกรดจากเวอร์ชัน 4.1 เป็น 4.5 ซึ่งช่วยเพิ่มความสามารถด้านการให้เหตุผลและประสิทธิภาพให้กับแชทบอท

ความน่าเชื่อถือ

Unit test ใหม่กว่า 20 รายการ ครอบคลุมการตรวจสอบวันที่ การตรวจสอบช่วงวันที่ ขอบเขตการจำกัดผลลัพธ์ การจัดการค่า null และการสร้าง cache key ยังคง backward compatible — การเรียกใช้งานเดิมที่ไม่มีพารามิเตอร์วันที่หรือจำนวนผลลัพธ์ยังคงทำงานได้ตามค่าเริ่มต้นที่เหมาะสม

การจัดการข้อมูลยังคงบังคับใช้

การกรองตามวันที่ไม่ได้เปลี่ยนแปลงว่าอะไรปลอดภัยที่จะทำกับผลลัพธ์เหล่านี้ — ดู Broker Notes MCP Integration และ Data Security

Broker Notes ရက်စွဲစစ်ထုတ်မှု နှင့် Opus 4.5

Broker notes ရှာဖွေမှုတွင် ရက်စွဲအပိုင်းအခြားအလိုက် စစ်ထုတ်ခြင်းနှင့် သတ်မှတ်နိုင်သော ရလဒ်အကန့်အသတ်များ ထပ်တိုးလာပြီး cache key ချို့ယွင်းချက် ပြင်ဆင်မှု နှင့် chatbot ကို ကွန်ပျူတာလုပ်ဆောင်ပေးနေသော Claude Opus မော်ဒယ်ကို ဗားရှင်း 4.5 သို့ အဆင့်မြှင့်တင်မှု ပါဝင်သည်။

အသစ်ပါဝင်လာသည့်အချက်များ

  • ရက်စွဲစစ်ထုတ်ခြင်း — start_date နှင့် end_date parameter များ (YYYY-MM-DD ပုံစံ) ဖြင့် ရလဒ်များကို ရက်စွဲအပိုင်းအခြားအတွင်း ကျဉ်းစေသည်။ မသတ်မှတ်ပါက ပုံသေအနေဖြင့် လွန်ခဲ့သော ၁၂ လအတွင်းကို အသုံးပြုသည်။
  • ရလဒ်အကန့်အသတ် — parameter အသစ် max_results (၁–၅၀၊ ပုံသေတန်ဖိုး ၁၀) က ပြန်ရသော ရလဒ်အရေအတွက်ကို ထိန်းချုပ်သည်။
  • အတည်ပြုစစ်ဆေးမှု — ရက်စွဲအပိုင်းအခြား အတည်ပြုမှုက စတင်ရက်စွဲသည် ပြီးဆုံးရက်စွဲနှင့် တူညီသော် (သို့) ယင်းထက် စောသော ရက်စွဲဖြစ်ကြောင်း သေချာစေသည်။ LLM မှ ထွက်ပေါ်လာတတ်သော quote character သို့မဟုတ် space အပို ကဲ့သို့ artifact များကိုလည်း ခွင့်ပြုသည်။

ယုံကြည်စိတ်ချမှု တိုးတက်မှုများ

  • API response များ (title, text, url) ရှိ null တန်ဖိုးများသည် ယခုအခါ error မပြသဘဲ မှန်ကန်စွာ fallback ပြုလုပ်နိုင်ပြီ
  • Cache key များသည် ယခုအခါ query string တစ်ခုတည်းအစား JSON payload တစ်ခုလုံး (query၊ ရလဒ်အရေအတွက်၊ ရက်စွဲများ) ကို hash ပြုလုပ်ပြီး cache ပဋိပက္ခများကို ကာကွယ်သည်

Claude Opus ကို 4.1 မှ 4.5 သို့ အဆင့်မြှင့်လိုက်ပြီး chatbot အတွက် ပိုမိုကောင်းမွန်သော ကျိုးကြောင်းဆင်ခြင်မှုနှင့် စွမ်းဆောင်ရည်ကို ယူဆောင်လာသည်

ယုံကြည်စိတ်ချမှု

Unit test အသစ် ၂၀ ကျော် — ရက်စွဲအတည်ပြုမှု၊ အပိုင်းအခြားအတည်ပြုမှု၊ ရလဒ်အကန့်အသတ်နယ်နိမိတ်၊ null ကိုင်တွယ်မှု နှင့် cache key ဖန်တီးမှုတို့ကို လွှမ်းခြုံသည်။ Backward compatible ဖြစ်သည် — ရက်စွဲ (သို့) limit parameter မပါသော ရှိပြီးသား ခေါ်ဆိုမှုများသည် ကောင်းမွန်သင့်တော်သော ပုံသေတန်ဖိုးများနှင့် ဆက်လက်အလုပ်လုပ်နေဆဲ ဖြစ်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်သည်

ရက်စွဲအလိုက် စစ်ထုတ်ခြင်းသည် ရလဒ်များနှင့် ဘာလုပ်လျှင် ဘေးကင်းသည်ဆိုသည်ကို ပြောင်းလဲမှု မရှိပါ — Broker Notes MCP Integration နှင့် Data Security ကို ကြည့်ပါ
