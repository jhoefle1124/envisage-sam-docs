Portfolio Analyst: Scorecard Rankings

The portfolio analyst tool now pulls scorecard data directly into portfolio and watchlist queries, so you can rank companies by internal, external, or ESG factors — and check up to 50 companies at once instead of 3. This page explains what changed and how to use it.

What's New

Portfolio analyst can now answer factor-based ranking questions directly, without a separate lookup step. The new scorecard_rankings query type integrates scorecard data with portfolio and watchlist queries in a single request:

  • Integrated scorecard data: portfolio/watchlist queries automatically pull in scorecard data — no separate lookup needed.
  • Factor-based rankings: rank companies by internal, external, or ESG scorecard factors.
  • Comprehensive filtering: filter by scorecard type, detail level, and latest-only.

What You Can Ask SAM

Example

"Which portfolio companies are rated highest on internal factors?"

Example

"Show watchlist companies with their scorecard ratings."

Example

"Get top companies by ESG scorecard scores."

Each of these now resolves in a single tool call — SAM fetches the portfolio or watchlist, pulls in scorecard ratings for every company, and ranks the result, rather than looking up each company one at a time.

Batch Processing: 3 Companies to 50

The underlying batch fetcher went from handling 3 companies per query to 50 — and from one tool call per company to a single tool call for the whole batch.

BeforeAfter
3 companies per queryUp to 50 companies per query
N tool calls for N companies1 tool call for N companies
Single-query lookups onlySingle query, multiple queries, or direct company IDs

For an analyst asking about a 30-company watchlist, this is the difference between one fast response and 30 sequential lookups.

Filtering Options

FilterWhat It Controls
Scorecard typeInternal, external, or ESG factors — or a combination
Detail levelHow much scorecard detail comes back with each company
Latest-onlyRestrict results to the most recent scorecard rating per company, rather than full history

Reliability

No breaking changes. Full backward compatibility is maintained — existing portfolio and watchlist queries behave exactly as before. The new ranking and batch capabilities are additive, backed by a 100+ test suite covering standard queries, batch operations, and edge cases.

Data Handling Still Applies

Portfolio and watchlist composition, and the scorecard ratings attached to them, are SAM client data — the same tier as any other fund holding or decision record on the Data Security page.

Tier 3

Stays inside SAM

Portfolio composition, scorecard ratings, and rankings are processed only inside SAM and approved Claude workflows under the engagement's terms — never pasted into free tools or verification models.

Check before quoting externally

Summaries and commentary

If you're drafting commentary that references a ranking or score outside SAM, treat client and company identifiers the same way you would any other Tier 3 content — anonymize or route through the approved workflow.

Portfolio Analyst: การจัดอันดับสกอร์การ์ด

เครื่องมือนักวิเคราะห์พอร์ตโฟลิโอ (portfolio analyst) ในขณะนี้ดึงข้อมูลสกอร์การ์ดเข้ามาในคำค้นหาพอร์ตโฟลิโอและ watchlist ได้โดยตรง ทำให้คุณสามารถจัดอันดับบริษัทตามปัจจัยภายใน ภายนอก หรือ ESG ได้ — และตรวจสอบได้ถึง 50 บริษัทในครั้งเดียวแทนที่จะเป็น 3 บริษัท หน้านี้อธิบายว่ามีอะไรเปลี่ยนแปลงและวิธีใช้งาน

มีอะไรใหม่

Portfolio analyst สามารถตอบคำถามเกี่ยวกับการจัดอันดับตามปัจจัย (factor-based ranking) ได้โดยตรงแล้ว โดยไม่ต้องมีขั้นตอนค้นหาแยกต่างหาก query type ใหม่ scorecard_rankings รวมข้อมูลสกอร์การ์ดเข้ากับคำค้นหาพอร์ตโฟลิโอและ watchlist ไว้ในคำขอเดียว:

  • ข้อมูลสกอร์การ์ดแบบรวม: คำค้นหาพอร์ตโฟลิโอ/watchlist จะดึงข้อมูลสกอร์การ์ดเข้ามาโดยอัตโนมัติ — ไม่ต้องค้นหาแยกอีกต่อไป
  • การจัดอันดับตามปัจจัย: จัดอันดับบริษัทตามปัจจัยสกอร์การ์ดภายใน ภายนอก หรือ ESG
  • การกรองข้อมูลอย่างครอบคลุม: กรองตามประเภทสกอร์การ์ด ระดับรายละเอียด และเฉพาะข้อมูลล่าสุดเท่านั้น

สิ่งที่คุณสามารถถาม SAM ได้

ตัวอย่าง

"บริษัทในพอร์ตโฟลิโอบริษัทใดได้คะแนนสูงสุดในปัจจัยภายใน?"

ตัวอย่าง

"แสดงบริษัทใน watchlist พร้อมคะแนนสกอร์การ์ดของแต่ละบริษัท"

ตัวอย่าง

"ดึงบริษัทอันดับต้น ๆ ตามคะแนนสกอร์การ์ด ESG"

คำถามเหล่านี้แต่ละข้อจะได้รับการประมวลผลด้วยการเรียกเครื่องมือเพียงครั้งเดียว — SAM จะดึงพอร์ตโฟลิโอหรือ watchlist ดึงคะแนนสกอร์การ์ดของทุกบริษัทเข้ามา และจัดอันดับผลลัพธ์ แทนที่จะค้นหาทีละบริษัท

การประมวลผลแบบกลุ่ม: จาก 3 บริษัท สู่ 50 บริษัท

ตัวดึงข้อมูลแบบกลุ่ม (batch fetcher) ที่อยู่เบื้องหลัง เปลี่ยนจากรองรับ 3 บริษัทต่อคำค้นหา เป็น 50 บริษัท — และจากการเรียกเครื่องมือหนึ่งครั้งต่อหนึ่งบริษัท เป็นการเรียกเครื่องมือเพียงครั้งเดียวสำหรับทั้งกลุ่ม

ก่อนหลัง
3 บริษัทต่อคำค้นหาสูงสุด 50 บริษัทต่อคำค้นหา
เรียกเครื่องมือ N ครั้งสำหรับ N บริษัทเรียกเครื่องมือ 1 ครั้งสำหรับ N บริษัท
ค้นหาได้เฉพาะคำค้นหาเดียวเท่านั้นคำค้นหาเดียว หลายคำค้นหา หรือระบุรหัสบริษัทโดยตรง

สำหรับนักวิเคราะห์ที่สอบถามเกี่ยวกับ watchlist ที่มี 30 บริษัท นี่คือความแตกต่างระหว่างการตอบสนองที่รวดเร็วเพียงครั้งเดียว กับการค้นหาต่อเนื่อง 30 ครั้ง

ตัวเลือกการกรองข้อมูล

ตัวกรองสิ่งที่ควบคุม
ประเภทสกอร์การ์ดปัจจัยภายใน ภายนอก หรือ ESG — หรือรวมกัน
ระดับรายละเอียดรายละเอียดสกอร์การ์ดที่จะส่งกลับมาพร้อมกับแต่ละบริษัทมากน้อยเพียงใด
เฉพาะข้อมูลล่าสุดจำกัดผลลัพธ์ให้แสดงเฉพาะคะแนนสกอร์การ์ดล่าสุดของแต่ละบริษัท แทนที่จะแสดงประวัติทั้งหมด

ความน่าเชื่อถือ

ไม่มีการเปลี่ยนแปลงที่กระทบฟังก์ชันการทำงานเดิม ยังคงความเข้ากันได้แบบย้อนหลังอย่างสมบูรณ์ — คำค้นหาพอร์ตโฟลิโอและ watchlist ที่มีอยู่เดิมทำงานเหมือนเดิมทุกประการ ความสามารถด้านการจัดอันดับและการประมวลผลแบบกลุ่มใหม่เป็นการเพิ่มเติมเท่านั้น โดยได้รับการรองรับจากชุดทดสอบกว่า 100 รายการที่ครอบคลุมคำค้นหามาตรฐาน การดำเนินการแบบกลุ่ม และกรณีขอบเขต (edge case)

หลักการจัดการข้อมูลยังคงใช้บังคับ

องค์ประกอบของพอร์ตโฟลิโอและ watchlist รวมถึงคะแนนสกอร์การ์ดที่ผูกอยู่กับข้อมูลเหล่านี้ ถือเป็นข้อมูลลูกค้าของ SAM — มี tier เดียวกันกับการถือครองกองทุนหรือบันทึกการตัดสินใจอื่น ๆ ตามหน้า ความปลอดภัยข้อมูล

Portfolio Analyst: Scorecard အဆင့်သတ်မှတ်ခြင်း

Portfolio analyst ကိရိယာသည် ယခုအခါ scorecard ဒေတာကို portfolio နှင့် watchlist query များထဲသို့ တိုက်ရိုက် ဆွဲသွင်းနိုင်ပြီ ဖြစ်သဖြင့် ကုမ္ပဏီများကို internal, external သို့မဟုတ် ESG factor များအလိုက် အဆင့်သတ်မှတ်နိုင်ပြီး — တစ်ကြိမ်လျှင် ကုမ္ပဏီ 3 ခုအစား 50 ခုအထိ စစ်ဆေးနိုင်သည်။ ဤစာမျက်နှာသည် ဘာပြောင်းလဲသွားသည်နှင့် အသုံးပြုနည်းကို ရှင်းပြသည်။

အသစ်ဘာတွေရှိလဲ

Portfolio analyst သည် သီးခြား lookup အဆင့်မလိုဘဲ factor-based ranking မေးခွန်းများကို တိုက်ရိုက် ဖြေဆိုနိုင်ပြီ။ Query type အသစ် scorecard_rankings သည် scorecard ဒေတာကို portfolio နှင့် watchlist query များနှင့် တစ်ကြိမ်တည်း request တစ်ခုတွင် ပေါင်းစပ်ပေးသည် —

  • ပေါင်းစပ် scorecard ဒေတာ: portfolio/watchlist query များသည် scorecard ဒေတာကို အလိုအလျောက် ဆွဲသွင်းသည် — သီးခြား lookup မလိုတော့ပါ။
  • Factor-based အဆင့်သတ်မှတ်ခြင်း: ကုမ္ပဏီများကို internal, external သို့မဟုတ် ESG scorecard factor များဖြင့် အဆင့်သတ်မှတ်ပါ။
  • ပြည့်စုံသော filtering: scorecard အမျိုးအစား၊ အသေးစိတ်အဆင့် နှင့် နောက်ဆုံးအချက်အလက်သာ ဖြင့် စစ်ထုတ်ပါ။

ဥပမာ

ဥပမာ

ဤမေးခွန်းများသည် တစ်ခုချင်းစီ ကုမ္ပဏီအလိုက် တစ်ကြိမ်လိုက်ရှာမည့်အစား tool call တစ်ခုတည်းနှင့် ဖြေရှင်းပြီးသွားသည် — SAM သည် portfolio သို့မဟုတ် watchlist ကို ရယူပြီး ကုမ္ပဏီတိုင်း၏ scorecard rating များကို ဆွဲသွင်းကာ ရလဒ်ကို အဆင့်သတ်မှတ်ပေးသည်။

Batch Processing: ကုမ္ပဏီ 3 ခုမှ 50 ခုအထိ

Underlying batch fetcher သည် query တစ်ခုလျှင် ကုမ္ပဏီ 3 ခုမှ 50 ခုအထိ ကိုင်တွယ်နိုင်အောင် ပြောင်းလဲသွားပြီး — ကုမ္ပဏီတစ်ခုလျှင် tool call တစ်ခုစီမှ batch တစ်ခုလုံးအတွက် tool call တစ်ခုတည်းသို့ ပြောင်းလဲသွားသည်။

မတိုင်မီပြီးနောက်

နောက်ဆုံးအချက်အလက်သာသမိုင်းကြောင်း အပြည့်အစုံအစား ကုမ္ပဏီတစ်ခုစီ၏ နောက်ဆုံး scorecard rating သို့သာ ရလဒ်များကို ကန့်သတ်ပါ

ယုံကြည်စိတ်ချရမှု

လက်ရှိလုပ်ဆောင်ချက်များကို ထိခိုက်စေမည့် ပြောင်းလဲမှု မရှိပါ။ Backward compatibility အပြည့်အစုံ ထိန်းသိမ်းထားသည် — လက်ရှိ portfolio နှင့် watchlist query များသည် ယခင်အတိုင်းအတိအကျ အလုပ်လုပ်နေဆဲဖြစ်သည်။ Ranking နှင့် batch စွမ်းရည် အသစ်များသည် ထပ်ပေါင်းထည့်ထားခြင်းသာဖြစ်ပြီး standard query, batch operation နှင့် edge case များပါဝင်သော test 100 ကျော်ဖြင့် အထောက်အထားပြုထားသည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

Portfolio နှင့် watchlist ဖွဲ့စည်းပုံ၊ ၎င်းတို့နှင့် ချိတ်ဆက်ထားသော scorecard rating များသည် SAM ဖောက်သည်ဒေတာဖြစ်သည် — ဒေတာလုံခြုံရေး စာမျက်နှာရှိ အခြား fund holding သို့မဟုတ် ဆုံးဖြတ်ချက် မှတ်တမ်းများနှင့် tier တူညီသည်။

Tier 3

SAM အတွင်းတွင်သာ ကျန်ရှိ

Portfolio ဖွဲ့စည်းပုံ၊ scorecard rating နှင့် ranking များကို SAM နှင့် engagement စည်းကမ်းချက်များအောက်ရှိ ခွင့်ပြုထားသော Claude workflow များအတွင်း၌သာ ပြုလုပ်သည် — အခမဲ့ ကိရိယာများ သို့မဟုတ် verification model များထဲသို့ လုံးဝ မကူးထည့်ရ။

အပြင်ဘက်တွင် ကိုးကားမီ စစ်ဆေးပါ

အနှစ်ချုပ်နှင့် ရှင်းလင်းချက်များ

SAM ပြင်ပရှိ ranking သို့မဟုတ် score ကို ရည်ညွှန်းသော ရှင်းလင်းချက်ကို ရေးဆွဲနေပါက ဖောက်သည်နှင့် ကုမ္ပဏီ identifier များကို အခြား Tier 3 အကြောင်းအရာကဲ့သို့ပင် ကိုင်တွယ်ပါ — အမည်ဝှက်ပါ သို့မဟုတ် ခွင့်ပြုထားသော workflow မှတစ်ဆင့် ဖြတ်သန်းစေပါ။
