New in SAM · Feature Guide

Shareable Chat Links

A chat session can now be shared via a public, time-limited link that renders as a read-only page with full markdown and interactive charts — no login required to view it.

What's New

  • Public share links: a cryptographic token (not a sequential ID) identifies each shared chat, preventing enumeration.
  • Expiration options: 1 hour, 24 hours, 2, 3, 7, 14, 30, or 90 days, or never — set at creation and revocable at any time.
  • Read-only rendering: shared chats render with full markdown, syntax highlighting, and interactive Highcharts, using the same sanitization pipeline as PDF export.
  • Chat list integration: share status filtering and inline revoke actions from the chat list.

Security & Hardening

Rate limiting

Configurable limits on both share creation and public view endpoints.

Message cap

Shared chats are capped at a configurable maximum number of messages.

Reliability

Extensive test coverage for share creation, link display, deletion, public access, expiration, and rate limiting, plus dedicated tests for the shared content sanitization pipeline.

Data Handling Still Applies

A shared link is public by design. Anyone with the link can view the conversation until it expires or is revoked — never share a link to a conversation containing Tier 2 or Tier 3 content. Treat share creation itself as a publication decision under Data Security, not a convenience feature.

ลิงก์แชทที่แชร์ได้

ขณะนี้เซสชันแชทสามารถแชร์ผ่านลิงก์สาธารณะที่มีเวลาจำกัดได้ ซึ่งจะแสดงผลเป็นหน้าแบบอ่านอย่างเดียว พร้อม markdown เต็มรูปแบบและกราฟแบบโต้ตอบได้ — ไม่ต้องเข้าสู่ระบบเพื่อดู

มีอะไรใหม่

  • ลิงก์แชร์สาธารณะ: โทเคนเชิงรหัส (cryptographic token) ที่ไม่ใช่ ID แบบเรียงลำดับ ใช้ระบุแชทที่แชร์แต่ละรายการ เพื่อป้องกันการไล่เดารายการ (enumeration)
  • ตัวเลือกวันหมดอายุ: 1 ชั่วโมง, 24 ชั่วโมง, 2, 3, 7, 14, 30 หรือ 90 วัน หรือไม่มีวันหมดอายุ — กำหนดได้ตั้งแต่สร้าง และสามารถเพิกถอนได้ตลอดเวลา
  • การแสดงผลแบบอ่านอย่างเดียว: แชทที่แชร์จะแสดงผลด้วย markdown เต็มรูปแบบ การเน้นไวยากรณ์ (syntax highlighting) และ Highcharts แบบโต้ตอบ โดยใช้ pipeline การทำความสะอาดข้อมูลเดียวกับการส่งออก PDF
  • การรวมเข้ากับรายการแชท: การกรองตามสถานะการแชร์และการเพิกถอนแบบอินไลน์จากรายการแชท

ความปลอดภัยและการเสริมความแข็งแกร่ง

การจำกัดอัตรา (Rate limiting)

ขีดจำกัดที่ปรับตั้งค่าได้ ทั้งสำหรับการสร้างลิงก์แชร์และ endpoint การเข้าดูแบบสาธารณะ

เพดานจำนวนข้อความ

แชทที่แชร์จะถูกจำกัดไว้ที่จำนวนข้อความสูงสุดที่ปรับตั้งค่าได้

ความน่าเชื่อถือ

มีการทดสอบครอบคลุมอย่างละเอียดสำหรับการสร้างลิงก์แชร์ การแสดงลิงก์ การลบ การเข้าถึงแบบสาธารณะ การหมดอายุ และการจำกัดอัตรา รวมถึงการทดสอบเฉพาะสำหรับ pipeline การทำความสะอาดเนื้อหาที่แชร์

หลักการจัดการข้อมูลยังคงใช้บังคับ

ลิงก์ที่แชร์ถูกออกแบบมาให้เป็นสาธารณะ ใครก็ตามที่มีลิงก์สามารถดูบทสนทนาได้จนกว่าลิงก์จะหมดอายุหรือถูกเพิกถอน — ห้ามแชร์ลิงก์ไปยังบทสนทนาที่มีเนื้อหา Tier 2 หรือ Tier 3 โดยเด็ดขาด ให้ถือว่าการสร้างลิงก์แชร์เองเป็นการตัดสินใจเผยแพร่ข้อมูลภายใต้ ความปลอดภัยข้อมูล ไม่ใช่เพียงฟีเจอร์เพื่อความสะดวก

မျှဝေနိုင်သော Chat လင့်များ

Chat session တစ်ခုကို ယခုအခါ အများပြည်သူသုံးနိုင်သော၊ အချိန်ကန့်သတ်ထားသော လင့်တစ်ခုမှတစ်ဆင့် မျှဝေနိုင်ပြီဖြစ်ပြီး ၎င်းသည် markdown အပြည့်အစုံနှင့် interactive chart များပါဝင်သော ဖတ်ရှုရုံသာ page အဖြစ် ပေါ်လာမည်ဖြစ်သည် — ကြည့်ရှုရန် login ဝင်စရာ မလိုအပ်ပါ။

အသစ်ဘာတွေရှိလဲ

  • အများပြည်သူ share လင့်များ: အစဉ်လိုက် ID အစား cryptographic token တစ်ခုက မျှဝေထားသော chat တစ်ခုစီကို ခွဲခြားသတ်မှတ်ပေးသဖြင့် enumeration ကို ကာကွယ်ပေးသည်။
  • သက်တမ်းကုန်ဆုံးမှု ရွေးချယ်စရာများ: 1 နာရီ၊ 24 နာရီ၊ 2၊ 3၊ 7၊ 14၊ 30 သို့မဟုတ် 90 ရက် သို့မဟုတ် ဘယ်တော့မှ မကုန်ဆုံးရန် — ဖန်တီးချိန်တွင် သတ်မှတ်ပြီး အချိန်မရွေး ပယ်ဖျက်နိုင်သည်။
  • ဖတ်ရှုရုံသာ ဖော်ပြခြင်း: မျှဝေထားသော chat များသည် PDF export တွင် အသုံးပြုသည့် sanitization pipeline တူညီစွာ အသုံးပြု၍ markdown အပြည့်အစုံ၊ syntax highlighting နှင့် interactive Highcharts များဖြင့် ဖော်ပြသည်။
  • Chat list ပေါင်းစပ်ခြင်း: chat list မှ share status filtering နှင့် inline revoke လုပ်ဆောင်ချက်များ။

လုံခြုံရေးနှင့် ခိုင်မာအောင်ပြုလုပ်ခြင်း

Rate Limiting

Share ဖန်တီးခြင်းနှင့် public view endpoint နှစ်ခုစလုံးတွင် ချိန်ညှိနိုင်သော ကန့်သတ်ချက်များ။

Message Cap

မျှဝေထားသော chat များကို ချိန်ညှိနိုင်သော message အများဆုံးအရေအတွက်ဖြင့် ကန့်သတ်ထားသည်။

ယုံကြည်စိတ်ချရမှု

Share ဖန်တီးခြင်း၊ လင့်ပြသခြင်း၊ ဖျက်ခြင်း၊ အများပြည်သူ ဝင်ရောက်ခြင်း၊ သက်တမ်းကုန်ဆုံးခြင်း နှင့် rate limiting အတွက် ကျယ်ပြန့်စွာ test coverage ရှိပြီး၊ မျှဝေထားသော content sanitization pipeline အတွက် သီးသန့် test များပါ ပါဝင်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

မျှဝေထားသော လင့်တစ်ခုသည် ဒီဇိုင်းအရ အများပြည်သူသုံးဖြစ်သည်။ လင့်ရှိသူ မည်သူမဆို ၎င်းသက်တမ်းကုန်ဆုံးသည် သို့မဟုတ် ပယ်ဖျက်သည်အထိ စကားဝိုင်းကို ကြည့်ရှုနိုင်သည် — Tier 2 သို့မဟုတ် Tier 3 အကြောင်းအရာပါဝင်သော စကားဝိုင်းသို့ လင့်ကို လုံးဝ မမျှဝေရ။ Share ဖန်တီးခြင်းကိုယ်တိုင်ကို အဆင်ပြေမှုအတွက် feature တစ်ခုအဖြစ်မဟုတ်ဘဲ ဒေတာလုံခြုံရေး အောက်ရှိ ထုတ်ဝေမှုဆိုင်ရာ ဆုံးဖြတ်ချက်တစ်ခုအဖြစ် သဘောထားပါ။
