ROE Formatter Bug Fix

A one-field fix: the return-on-equity metric was being displayed 100 times too high because it was run through the wrong percentage formatter.

The Problem

LayerBehavior
Data storageROE stored as an already-percentage value (15 means 15%)
Formatter usedformat_decimal_percentage, which multiplies by 100
Result15% displayed as 1500%

The Fix

Switched the ROE field to format_percentage, which correctly handles values already stored as percentages without the extra multiplication. All other percentage and decimal fields in the field registry were audited — ROE was the only mismatch.

Data Handling Still Applies

A display-formatting fix — no change to what data is available or its tier. See Data Security for the standard rules governing financial data.

ฟีเจอร์ใหม่ใน SAM · ข้อมูลอ้างอิงทางเทคนิค

การแก้ไขบั๊กตัวจัดรูปแบบ ROE

การแก้ไขในฟิลด์เดียว: ค่า ROE แสดงผลสูงเกินจริงถึง 100 เท่า เนื่องจากถูกประมวลผลผ่านตัวจัดรูปแบบเปอร์เซ็นต์ที่ไม่ถูกต้อง

ปัญหาที่พบ

ชั้นข้อมูลพฤติกรรม
การจัดเก็บข้อมูลค่า ROE ถูกจัดเก็บเป็นค่าเปอร์เซ็นต์อยู่แล้ว (15 หมายถึง 15%)
ตัวจัดรูปแบบที่ใช้format_decimal_percentage ซึ่งคูณด้วย 100
ผลลัพธ์15% แสดงผลเป็น 1500%

การแก้ไข

เปลี่ยนฟิลด์ ROE ให้ใช้ format_percentage ซึ่งจัดการค่าที่จัดเก็บเป็นเปอร์เซ็นต์อยู่แล้วได้อย่างถูกต้องโดยไม่มีการคูณซ้ำ ฟิลด์เปอร์เซ็นต์และทศนิยมอื่น ๆ ทั้งหมดใน field registry ได้รับการตรวจสอบแล้ว — ROE เป็นรายการเดียวที่ไม่ตรงกัน

หลักการจัดการข้อมูลยังคงเดิม

นี่เป็นเพียงการแก้ไขรูปแบบการแสดงผล — ไม่มีการเปลี่ยนแปลงข้อมูลที่เข้าถึงได้หรือชั้นข้อมูลของข้อมูลนั้น ดู ความปลอดภัยของข้อมูล สำหรับกฎมาตรฐานที่ใช้กับข้อมูลทางการเงิน

SAM တွင် အသစ် · နည်းပညာအညွှန်း

ROE ဖော်မတ်ချက် ချွတ်ယွင်းမှု ပြင်ဆင်ခြင်း

ကွက်တစ်ခုတည်း ပြင်ဆင်ချက် — ROE တန်ဖိုးကို မှားယွင်းသော ရာခိုင်နှုန်း ဖော်မတ်ကိရိယာဖြင့် တွက်ချက်ခဲ့သဖြင့် အမှန်ထက် ၁၀၀ ဆ မြင့်၍ ပြသနေခဲ့သည်။

ပြဿနာ

အလွှာအပြုအမူ
ဒေတာသိမ်းဆည်းမှုROE ကို ရာခိုင်နှုန်းတန်ဖိုးအဖြစ် သိမ်းဆည်းထားပြီးသားဖြစ်သည် (15 က 15% ကို ဆိုလိုသည်)
အသုံးပြုသော ဖော်မတ်ကိရိယာformat_decimal_percentage, ၎င်းသည် 100 နှင့် မြှောက်သည်
ရလဒ်15% ကို 1500% အဖြစ် ပြသသည်

ပြင်ဆင်ချက်

ROE ကွက်ကို format_percentage သို့ ပြောင်းလဲလိုက်သည်။ ၎င်းသည် ရာခိုင်နှုန်းအဖြစ် သိမ်းဆည်းပြီးသား တန်ဖိုးများကို ထပ်မံမြှောက်ခြင်းမပြုဘဲ မှန်ကန်စွာ ကိုင်တွယ်ပေးသည်။ field registry ရှိ အခြား ရာခိုင်နှုန်းနှင့် ဒဿမကွက်အားလုံးကို စစ်ဆေးပြီးဖြစ်ပြီး — ROE သာလျှင် မကိုက်ညီသော တစ်ခုတည်းသော ကွက်ဖြစ်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

ဤသည်မှာ ပြသမှုပုံစံ ပြင်ဆင်ချက်တစ်ခုသာ ဖြစ်သည် — မည်သည့်ဒေတာ ရရှိနိုင်သည် သို့မဟုတ် ၎င်း၏ tier တွင် ပြောင်းလဲမှု မရှိပါ။ ဘဏ္ဍာရေးဒေတာအတွက် စံစည်းမျဉ်းများကို ဒေတာလုံခြုံရေး တွင် ကြည့်ပါ။
