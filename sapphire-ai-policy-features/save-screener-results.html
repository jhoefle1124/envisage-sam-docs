Save Screener Results from Chat

Stock screener results generated inside an AI conversation can now be saved with one click, instead of being lost when the chat scrolls away — persisted to the database and reusable later.

What's New

  • A save button appears on any chatbot message containing stock screener results.
  • Results are stored in a new stock_screener_data JSON field on the chat message, via a dedicated save endpoint.
  • The button reflects a persistent state after a successful save, with success and error notifications.

Reliability

20 files changed, 1,896 additions. Comprehensive tests cover URL routing, tool-extraction error handling, save persistence, and the full save workflow end to end. No breaking changes to existing functionality.

Data Handling Still Applies

A saved screener result carries the same tier as the query that produced it. A generic public screen is Tier 1; a screen run against a client watchlist is Tier 2/3 — see Data Security before saving anything client-specific outside SAM.

บันทึกผลตัวคัดกรองจากแชท

ผลลัพธ์การคัดกรองหุ้นที่สร้างขึ้นภายในบทสนทนา AI สามารถบันทึกได้ด้วยคลิกเดียว แทนที่จะสูญหายไปเมื่อแชทเลื่อนผ่านไป — ข้อมูลจะถูกบันทึกลงฐานข้อมูลและสามารถนำกลับมาใช้ใหม่ได้ในภายหลัง

มีอะไรใหม่

  • ปุ่มบันทึกจะปรากฏบนข้อความของแชทบอทที่มีผลลัพธ์การคัดกรองหุ้น
  • ผลลัพธ์จะถูกจัดเก็บในฟิลด์ JSON ใหม่ชื่อ stock_screener_data บนข้อความแชท ผ่าน endpoint สำหรับการบันทึกโดยเฉพาะ
  • ปุ่มจะแสดงสถานะที่คงอยู่ถาวรหลังจากบันทึกสำเร็จ พร้อมการแจ้งเตือนทั้งกรณีสำเร็จและกรณีผิดพลาด

ความน่าเชื่อถือ

เปลี่ยนแปลง 20 ไฟล์ เพิ่มโค้ด 1,896 บรรทัด การทดสอบครอบคลุมการทำ URL routing การจัดการข้อผิดพลาดในการดึงข้อมูลเครื่องมือ (tool extraction) ความคงอยู่ของข้อมูลหลังบันทึก และ workflow การบันทึกทั้งหมดตั้งแต่ต้นจนจบ ไม่มีการเปลี่ยนแปลงที่กระทบฟังก์ชันการทำงานเดิม

หลักการจัดการข้อมูลยังคงใช้บังคับ

ผลลัพธ์ screener ที่บันทึกไว้จะมี tier เดียวกันกับ query ที่สร้างผลลัพธ์นั้น การคัดกรองสาธารณะทั่วไปคือ Tier 1 ส่วนการคัดกรองที่รันกับ watchlist ของลูกค้าคือ Tier 2/3 — ดู ความปลอดภัยข้อมูล ก่อนบันทึกข้อมูลเฉพาะลูกค้าใด ๆ ไว้นอกระบบ SAM

SAM ရှိ အသစ် · Feature Guide

စကားပြောမှတ်တမ်းမှ Screener ရလဒ် သိမ်းဆည်းခြင်း

AI စကားဝိုင်းအတွင်း ဖန်တီးထားသော stock screener ရလဒ်များကို ယခုအခါ တစ်ချက်နှိပ်ရုံဖြင့် သိမ်းဆည်းနိုင်ပြီ — chat scroll ဖြတ်သွားချိန်တွင် ပျောက်ဆုံးသွားမည့်အစား ဒေတာဘေ့စ်တွင် သိမ်းဆည်းထားပြီး နောက်ပိုင်း ပြန်လည်အသုံးပြုနိုင်သည်။

အသစ်ဘာတွေရှိလဲ

  • Chatbot message တစ်ခုတွင် stock screener ရလဒ် ပါဝင်လျှင် သိမ်းဆည်းရန် ခလုတ် ပေါ်လာသည်။
  • ရလဒ်များကို chat message ပေါ်ရှိ stock_screener_data ဟု အမည်ပေးထားသော JSON field အသစ်တွင် သီးသန့် save endpoint မှတစ်ဆင့် သိမ်းဆည်းသည်။
  • သိမ်းဆည်းမှု အောင်မြင်ပြီးနောက် ခလုတ်သည် အမြဲတမ်းရှိနေသော status ကို ပြသပြီး အောင်မြင်မှုနှင့် error အသိပေးချက်များ ပါဝင်သည်။

ယုံကြည်စိတ်ချရမှု

ဖိုင် ၂၀ ပြောင်းလဲပြီး ကုဒ်လိုင်း ၁,၈၉၆ လိုင်း ထပ်ထည့်ထားသည်။ URL routing, tool-extraction error ကိုင်တွယ်မှု, save persistence နှင့် save workflow တစ်ခုလုံးကို အစအဆုံး စစ်ဆေးသော test များ ပါဝင်သည်။ လက်ရှိလုပ်ဆောင်ချက်များကို ထိခိုက်စေမည့် ပြောင်းလဲမှု မရှိပါ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

သိမ်းဆည်းထားသော screener ရလဒ်သည် ၎င်းကို ဖန်တီးပေးသည့် query ၏ tier အတိုင်းပင် ရှိနေမည်ဖြစ်သည်။ ယေဘုယျ အများပြည်သူ screen တစ်ခုသည် Tier 1 ဖြစ်ပြီး ဖောက်သည် watchlist တစ်ခုအပေါ် run ထားသော screen သည် Tier 2/3 ဖြစ်သည် — SAM အပြင်ဘက်တွင် ဖောက်သည်နှင့်ဆိုင်သော အရာတစ်ခုခုကို မသိမ်းဆည်းမီ ဒေတာလုံခြုံရေး ကို ကြည့်ပါ။
