Prompt Studio

Prompt Studio is the custom admin interface behind the Prompt Template Library — where site admins build and organize the templates users pick from in SAM. Hierarchical categories, a proper markdown editor, and drag-and-drop ordering replace what used to be manual template management.

What's New

  • Hierarchical categories: categories can now nest under parent categories, not just sit in a flat list.
  • Rich markdown editing: template content is written in EasyMDE, a proper markdown editor with syntax highlighting and live preview.
  • Unified ordering interface: a single view lets you drag and drop both categories and templates into order, with nested categories collapsible for easier navigation.
  • Sortable, filterable lists: Tabulator-based list views for categories and templates support sorting and filtering instead of a plain table.
  • Template variable validation: the placeholder syntax used in templates ({{Company}} and similar — see Prompt Templates) is validated at save time.
  • Public/private visibility: a template can be kept private while still being drafted, or made public once it's ready for all users.

How It Relates to Prompt Templates

Prompt Studio is the authoring side; the Prompt Template Library page describes the user-facing result. If you're a site admin building or organizing templates, this page is for you. If you're looking for how to use a template that's already published, or how the {{Company}}-style placeholders work, see Prompt Templates instead.

Managing Categories

Categories now support parent-child relationships, so related templates can be grouped under a shared parent (for example, an "Investment Research" parent with sub-categories underneath). The ordering interface shows nested categories as collapsible groups, and both categories and the templates inside them can be reordered by drag and drop.

Managing Templates

Template content is written in EasyMDE rather than a plain text box — syntax highlighting and a live preview make it easier to check formatting before publishing. Two things worth knowing:

Variable validation

Placeholder syntax is checked at save time, so a malformed {{...}} tag gets caught before the template reaches users.

Public/private toggle

Draft a template privately while you refine it, then flip it public once it's ready for the whole team to see in SAM.

Reliability

Thoroughly tested. The update ships with a comprehensive test suite covering models, forms, and views, including permission checks, validation, and edge cases — 100% coverage for the new functionality across roughly 30 changed files.

Data Handling Still Applies

Templates authored in Prompt Studio are published broadly — once public, any user can select them in SAM. Keep that audience in mind while drafting.

Don't hardcode client specifics into a shared template. A template's structure and placeholders (Tier 1) are meant to be reused across engagements. Naming a real client, engagement, or case detail directly in the template text turns it into something every user with template access can see — draft with placeholders, not real names, exactly as described on Prompt Templates and Data Security.

Prompt Studio

Prompt Studio คืออินเทอร์เฟซแอดมินสำหรับสร้างและจัดระเบียบเทมเพลตที่อยู่เบื้องหลัง Prompt Template Library — จุดที่แอดมินของไซต์สร้างและจัดระเบียบเทมเพลตที่ผู้ใช้เลือกใน SAM หมวดหมู่แบบลำดับชั้น editor แบบ markdown ที่สมบูรณ์ และการจัดลำดับด้วยการลากวาง เข้ามาแทนที่การจัดการเทมเพลตด้วยมือแบบเดิม

สิ่งใหม่ในเวอร์ชันนี้

  • หมวดหมู่แบบลำดับชั้น: ตอนนี้หมวดหมู่สามารถซ้อนอยู่ภายใต้หมวดหมู่แม่ได้ ไม่ใช่แค่เรียงอยู่ในรายการแบบเรียบอีกต่อไป
  • การแก้ไข markdown แบบเต็มรูปแบบ: เนื้อหาเทมเพลตเขียนด้วย EasyMDE ซึ่งเป็น markdown editor ที่มี syntax highlighting และ live preview
  • อินเทอร์เฟซจัดลำดับแบบรวมศูนย์: มุมมองเดียวให้คุณลากวางทั้งหมวดหมู่และเทมเพลตเพื่อจัดลำดับได้ โดยหมวดหมู่ที่ซ้อนกันสามารถย่อ (collapsible) เพื่อให้นำทางง่ายขึ้น
  • รายการที่เรียงลำดับและกรองได้: มุมมองรายการหมวดหมู่และเทมเพลตที่ใช้ Tabulator รองรับการเรียงลำดับและการกรอง แทนที่ตารางธรรมดา
  • การตรวจสอบตัวแปรเทมเพลต: รูปแบบ placeholder ที่ใช้ในเทมเพลต ({{Company}} และคล้ายกัน — ดู Prompt Templates) จะถูกตรวจสอบเมื่อบันทึก
  • การมองเห็นแบบ public/private: เทมเพลตสามารถเก็บเป็น private ได้ระหว่างที่ยังร่างอยู่ หรือทำให้เป็น public เมื่อพร้อมสำหรับผู้ใช้ทุกคน

ความสัมพันธ์กับ Prompt Templates

Prompt Studio คือฝั่งการสร้างเนื้อหา (authoring) ส่วนหน้า Prompt Template Library อธิบายผลลัพธ์ที่ผู้ใช้เห็น หากคุณเป็นแอดมินของไซต์ที่กำลังสร้างหรือจัดระเบียบเทมเพลต หน้านี้คือสำหรับคุณ หากคุณกำลังมองหาวิธีใช้เทมเพลตที่เผยแพร่แล้ว หรือวิธีการทำงานของ placeholder แบบ {{Company}} ให้ดูที่ Prompt Templates แทน

การจัดการหมวดหมู่

ตอนนี้หมวดหมู่รองรับความสัมพันธ์แบบแม่-ลูก ทำให้เทมเพลตที่เกี่ยวข้องกันสามารถจัดกลุ่มอยู่ภายใต้หมวดหมู่แม่เดียวกันได้ (ตัวอย่างเช่น หมวดหมู่แม่ "Investment Research" ที่มีหมวดหมู่ย่อยอยู่ภายใต้) อินเทอร์เฟซจัดลำดับแสดงหมวดหมู่ที่ซ้อนกันเป็นกลุ่มที่ย่อได้ และทั้งหมวดหมู่และเทมเพลตภายในสามารถจัดลำดับใหม่ได้ด้วยการลากวาง

การจัดการเทมเพลต

เนื้อหาเทมเพลตเขียนด้วย EasyMDE แทนที่จะเป็นกล่องข้อความธรรมดา — syntax highlighting และ live preview ช่วยให้ตรวจสอบการจัดรูปแบบได้ง่ายขึ้นก่อนเผยแพร่ มีสองเรื่องที่ควรรู้:

การตรวจสอบตัวแปร

รูปแบบ placeholder จะถูกตรวจสอบเมื่อบันทึก ดังนั้นแท็ก {{...}} ที่ผิดรูปแบบจะถูกจับได้ก่อนที่เทมเพลตจะไปถึงผู้ใช้

สลับ public/private

ร่างเทมเพลตแบบ private ระหว่างที่ปรับแก้ จากนั้นสลับเป็น public เมื่อพร้อมให้ทั้งทีมเห็นใน SAM

ความน่าเชื่อถือ

ผ่านการทดสอบอย่างละเอียด การอัปเดตนี้มาพร้อม test suite ที่ครอบคลุม models, forms และ views รวมถึงการตรวจสอบสิทธิ์ (permission checks) การตรวจสอบความถูกต้อง (validation) และ edge case ต่าง ๆ — coverage 100% สำหรับฟังก์ชันใหม่ในไฟล์ที่เปลี่ยนแปลงประมาณ 30 ไฟล์

ข้อกำหนดการจัดการข้อมูลยังคงมีผล

เทมเพลตที่สร้างใน Prompt Studio จะถูกเผยแพร่ในวงกว้าง — เมื่อเป็น public แล้ว ผู้ใช้ทุกคนสามารถเลือกใช้ใน SAM ได้ ควรคำนึงถึงกลุ่มผู้ใช้นี้ขณะร่างเทมเพลต

อย่าใส่ข้อมูลเฉพาะของลูกค้าลงในเทมเพลตที่ใช้ร่วมกัน โครงสร้างและ placeholder ของเทมเพลต (Tier 1) มีไว้เพื่อให้นำกลับมาใช้ซ้ำได้ในหลายโครงการ การระบุชื่อลูกค้าจริง ชื่อโครงการ หรือรายละเอียดคดีลงในเนื้อหาเทมเพลตโดยตรง จะทำให้ผู้ใช้ทุกคนที่เข้าถึงเทมเพลตนั้นมองเห็นข้อมูลนี้ได้ — ให้ร่างด้วย placeholder ไม่ใช่ชื่อจริง ตามที่อธิบายไว้ใน Prompt Templates และ Data Security

Prompt Studio

Prompt Studio သည် Prompt Template Library ၏ နောက်ကွယ်တွင်ရှိသော admin interface ဖြစ်ပြီး — ဆိုက် admin များက SAM တွင် အသုံးပြုသူများ ရွေးချယ်သော template များကို တည်ဆောက်ပြီး စီစဉ်ရသည့်နေရာဖြစ်သည်။ Hierarchical categories၊ ကောင်းမွန်သော markdown editor နှင့် drag-and-drop စီစဉ်ခြင်းတို့သည် ယခင်က manual ဖြင့် template စီမံခန့်ခွဲရသည့်နည်းလမ်းကို အစားထိုးလိုက်သည်။

အသစ်ပါဝင်လာသောအရာများ

  • Hierarchical categories: categories များသည် flat list တစ်ခုတည်းတွင်မရှိတော့ဘဲ parent category အောက်တွင် nest လုပ်နိုင်ပြီဖြစ်သည်။
  • ပြည့်စုံသော markdown တည်းဖြတ်ခြင်း: template အကြောင်းအရာကို syntax highlighting နှင့် live preview ပါသော markdown editor တစ်ခုဖြစ်သည့် EasyMDE ဖြင့် ရေးသားသည်။
  • ပေါင်းစည်းထားသော စီစဉ်ခြင်း interface: view တစ်ခုတည်းဖြင့် categories နှင့် templates နှစ်ခုစလုံးကို drag and drop ဖြင့် စီစဉ်နိုင်ပြီး nest လုပ်ထားသော categories များကို navigate လွယ်ကူစေရန် collapse လုပ်နိုင်သည်။
  • Sort နှင့် filter လုပ်နိုင်သော list များ: categories နှင့် templates အတွက် Tabulator-based list view များသည် ရိုးရှင်းသော table အစား sort နှင့် filter ကို ပံ့ပိုးပေးသည်။
  • Template variable validation: template များတွင် အသုံးပြုသော placeholder syntax ({{Company}} နှင့် အလားတူများ — Prompt Templates ကိုကြည့်ပါ) ကို save လုပ်ချိန်တွင် validate လုပ်သည်။
  • Public/private မြင်နိုင်မှု: template တစ်ခုကို draft ဆွဲနေဆဲအချိန်တွင် private အဖြစ်ထားနိုင်ပြီး၊ အသုံးပြုသူအားလုံးအတွက် အသင့်ဖြစ်သောအခါ public ပြုလုပ်နိုင်သည်။

Prompt Templates နှင့် ဆက်စပ်ပုံ

Prompt Studio သည် ရေးသားဖန်တီးသည့်ဘက် (authoring side) ဖြစ်ပြီး Prompt Template Library စာမျက်နှာက အသုံးပြုသူမြင်ရသော ရလဒ်ကို ဖော်ပြသည်။ သင်သည် template များကို တည်ဆောက်နေသော သို့မဟုတ် စီစဉ်နေသော ဆိုက် admin ဖြစ်ပါက ဤစာမျက်နှာသည် သင့်အတွက်ဖြစ်သည်။ တင်ပြီးသား template ကို မည်သို့အသုံးပြုရမည်၊ သို့မဟုတ် {{Company}} ပုံစံ placeholder များ မည်သို့အလုပ်လုပ်သည်ကို လိုချင်ပါက Prompt Templates ကို ကြည့်ပါ။

Categories စီမံခန့်ခွဲခြင်း

ယခုအခါ categories များသည် parent-child ဆက်ဆံရေးကို ပံ့ပိုးပေးသဖြင့် ဆက်စပ်သော template များကို parent category တစ်ခုတည်း အောက်တွင် အုပ်စုဖွဲ့နိုင်သည် (ဥပမာ — "Investment Research" ဟူသော parent category အောက်တွင် sub-category များ)။ စီစဉ်ခြင်း interface သည် nest လုပ်ထားသော categories များကို collapse လုပ်နိုင်သော အုပ်စုများအဖြစ် ပြသပြီး categories နှင့် အတွင်းရှိ template များ နှစ်ခုစလုံးကို drag and drop ဖြင့် ပြန်လည်စီစဉ်နိုင်သည်။

Templates စီမံခန့်ခွဲခြင်း

Template အကြောင်းအရာကို ရိုးရှင်းသော text box အစား EasyMDE ဖြင့် ရေးသားသည် — syntax highlighting နှင့် live preview က publish မလုပ်မီ ပုံစံချမှုကို စစ်ဆေးရာတွင် ပိုမိုလွယ်ကူစေသည်။ သိထားသင့်သော အချက်နှစ်ချက်ရှိသည် —

Variable validation

Placeholder syntax ကို save လုပ်ချိန်တွင် စစ်ဆေးသောကြောင့် ပုံစံမှားနေသော {{...}} tag ကို template သည် အသုံးပြုသူများထံ မရောက်မီ ဖမ်းမိနိုင်သည်။

Public/private toggle

Template ကို ပြင်ဆင်နေစဉ် private အဖြစ် draft ရေးပြီး၊ SAM တွင် team တစ်ခုလုံး မြင်ရန် အသင့်ဖြစ်သောအခါ public သို့ ပြောင်းပါ။

ယုံကြည်စိတ်ချရမှု

အသေးစိတ် စမ်းသပ်ပြီးဖြစ်သည်။ ဤ update သည် permission checks၊ validation နှင့် edge case များအပါအဝင် models၊ forms နှင့် views များကို ဖုံးလွှမ်းသော ပြည့်စုံသည့် test suite နှင့်အတူ ထွက်ရှိလာသည် — ပြောင်းလဲခဲ့သော file ခန့်မှန်းခြေ 30 ခုတွင် feature အသစ်အတွက် coverage 100% ရှိသည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေဆဲ

Prompt Studio တွင် ရေးသားထားသော template များကို ကျယ်ပြန့်စွာ publish လုပ်သည် — public ဖြစ်သွားပြီးနောက် အသုံးပြုသူတိုင်းက SAM တွင် ၎င်းတို့ကို ရွေးချယ်နိုင်သည်။ Template ရေးဆွဲနေစဉ် ဤ audience ကို သတိထားပါ။

ဖောက်သည်၏ တိကျသောအချက်အလက်များကို မျှဝေထားသော template ထဲသို့ တိုက်ရိုက်ထည့်သွင်းခြင်း မပြုပါနှင့်။ Template ၏ ဖွဲ့စည်းပုံနှင့် placeholder များ (Tier 1) သည် engagement များစွာတွင် ပြန်လည်အသုံးပြုနိုင်ရန် ရည်ရွယ်သည်။ Template စာသားထဲတွင် တကယ့်ဖောက်သည်၊ engagement သို့မဟုတ် case အသေးစိတ်ကို တိုက်ရိုက်ဖော်ပြခြင်းသည် template ကို ဝင်ရောက်ခွင့်ရှိသူ အားလုံး မြင်နိုင်သောအရာအဖြစ် ပြောင်းလဲသွားစေသည် — Prompt Templates နှင့် Data Security တွင် ဖော်ပြထားသည့်အတိုင်း တကယ့်နာမည်များအစား placeholder များဖြင့် draft ရေးပါ။
