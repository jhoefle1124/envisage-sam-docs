AI Core Runtime Extraction

A structural refactor: shared AI runtime infrastructure moved out of the chatbots app into a new apps.ai_core, and tool providers relocated to the domain apps that actually own their data — no functional behavior changes.

Why

The chatbots app had grown into a general-purpose AI platform module that other apps (AI screener, portfolios, scorecards) had to reach into for tool contracts, the LLM factory, and stock-screener query types. That coupling made chat-only changes risky and blocked reuse of the tool runtime outside the chat surface.

What Moved Where

New homeTool providers
apps.ai_core.tool_providersGeneric external integrations: Tavily search, Jina reader, Brave News search, broker notes MCP, Firecrawl
apps.stock_data.tool_providerscompany_financials, daily_price_metrics, price_equity
apps.stock_screener.tool_providersstock_screener tool and its complexity limiter
apps.scorecards.tool_providersscorecard
apps.portfolios.tool_providersportfolio_analyst
apps.chatbots.tool_providersStays: chat-only tools (document tools, chart_generator) tied to chat session state

A new apps.ai_core app now hosts the tool registry, context, responses, base classes, categories, LLM factory, exceptions, URL helpers, and financial formatter. Stock-screener query contracts (filter schemas, field registry, query builder) moved to apps.stock_screener.query.

Import-Boundary Enforcement

New AST-based regression tests guard the new architecture: runtime modules in ai_core and stock-screener query contracts may not import from apps.chatbots, each domain's tool provider package may not import from apps.chatbots, and apps.chatbots.tool_providers is guarded against new modules for already-migrated providers reappearing there.

Reliability

41 commits across four migration waves, with temporary compatibility shims removed once every runtime caller and test was repointed. The full test suite runs clean; the only behavioral change is a targeted performance fix in the stock screener's allowed-fields lookup, covered by its own dedicated test.

Data Handling Still Applies

A structural code refactor with no functional changes — every tool still respects the same tiering rules it did before the move. See Data Security for the underlying policy this codebase enforces.
