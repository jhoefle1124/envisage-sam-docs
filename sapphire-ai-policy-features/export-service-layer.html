Export Service Layer & Error Handling

Exporting AI screener results to the manual screener got a shared service layer, comprehensive error handling, and configurable export limits — making the export path more robust and easier to extend.

What's New

  • Shared export service: a new service layer used by both the AI screener and manual screener, removing duplicated export logic.
  • Enhanced error handling: validation for screener name length (60-character limit) and user-friendly messages for every failure scenario.
  • Dynamic export limits: the export limit message now reads from a SCREENER_EXPORT_LIMIT environment variable instead of being hardcoded.

Code Quality

Removed redundant @transaction.atomic decorators where not needed, improved view organization, and consolidated duplicate export logic that previously existed separately for the AI screener and manual screener.

Reliability

345 lines of tests for export helper functions and 539 lines for AI screener forms integration, plus fixes for previously flaky tests. No breaking changes. See also Enhanced Export Functionality for the user-facing view.

Data Handling Still Applies

Exporting through the shared service doesn't change tier rules — a generic public screen export stays close to Tier 1, while a named or client-tied export follows Tier 2/3 rules per Data Security.

Export Service Layer และการจัดการข้อผิดพลาด

การส่งออกผลลัพธ์จาก AI screener ไปยัง manual screener ได้รับ shared service layer การจัดการข้อผิดพลาดที่ครอบคลุม และขีดจำกัดการส่งออกที่กำหนดค่าได้ — ทำให้เส้นทางการส่งออกแข็งแกร่งขึ้นและขยายต่อได้ง่ายขึ้น

มีอะไรใหม่

  • Shared export service: service layer ใหม่ที่ใช้ร่วมกันทั้ง AI screener และ manual screener ช่วยลดความซ้ำซ้อนของ export logic
  • Enhanced error handling: การตรวจสอบความยาวชื่อ screener (จำกัดไม่เกิน 60 ตัวอักษร) และข้อความที่เป็นมิตรกับผู้ใช้สำหรับทุกกรณีที่เกิดข้อผิดพลาด
  • Dynamic export limits: ข้อความแจ้งขีดจำกัดการส่งออกอ่านค่าจากตัวแปรสภาพแวดล้อม SCREENER_EXPORT_LIMIT แทนการกำหนดค่าตายตัวไว้ในโค้ด

คุณภาพของโค้ด

ลบ decorator @transaction.atomic ที่ซ้ำซ้อนออกในจุดที่ไม่จำเป็น ปรับปรุงการจัดระเบียบ view ให้ดีขึ้น และรวม export logic ที่ซ้ำซ้อนซึ่งก่อนหน้านี้แยกกันอยู่ระหว่าง AI screener และ manual screener เข้าด้วยกัน

ความน่าเชื่อถือ

การทดสอบ 345 บรรทัด สำหรับ export helper functions และ 539 บรรทัด สำหรับการรวม AI screener forms พร้อมทั้งแก้ไข test ที่เคยไม่เสถียร (flaky) ไม่มีการเปลี่ยนแปลงที่กระทบการทำงานเดิม ดูเพิ่มเติมที่ ฟังก์ชันการส่งออกที่ปรับปรุงแล้ว สำหรับมุมมองฝั่งผู้ใช้

กฎการจัดการข้อมูลยังคงมีผลใช้บังคับ

การส่งออกผ่าน shared service ไม่ได้เปลี่ยนแปลงกฎเรื่อง tier — การส่งออกหน้าจอสาธารณะทั่วไปยังคงใกล้เคียง Tier 1 ในขณะที่การส่งออกที่ระบุชื่อหรือผูกกับลูกค้าจะต้องปฏิบัติตามกฎ Tier 2/3 ตาม ความปลอดภัยของข้อมูล

Export Service Layer နှင့် အမှားများ ကိုင်တွယ်ခြင်း

AI screener ရလဒ်များကို manual screener သို့ export လုပ်ခြင်းသည် shared service layer၊ ပြည့်စုံသော error handling နှင့် ချိန်ညှိနိုင်သော export ကန့်သတ်ချက်များ ရရှိလာသည် — export လမ်းကြောင်းကို ပိုမိုခိုင်မာစေပြီး တိုးချဲ့ရလွယ်ကူစေသည်။

အသစ်ပါဝင်မှုများ

  • Shared export service: AI screener နှင့် manual screener နှစ်ခုစလုံးတွင် အသုံးပြုသော service layer အသစ်တစ်ခု — export logic ထပ်နေမှုများကို ဖယ်ရှားပေးသည်
  • Enhanced error handling: screener အမည်၏ အရှည် (စာလုံး 60 လုံး ကန့်သတ်ချက်) စစ်ဆေးခြင်းနှင့် error scenario တိုင်းအတွက် သုံးရလွယ်ကူသော message များ
  • Dynamic export limits: export ကန့်သတ်ချက် message သည် ယခုအခါ ကုဒ်တွင် hardcode မလုပ်တော့ဘဲ SCREENER_EXPORT_LIMIT environment variable မှ တန်ဖိုးကို ဖတ်ယူသည်

ကုဒ်အရည်အသွေး

မလိုအပ်သည့်နေရာများတွင် ထပ်နေသော @transaction.atomic decorator များကို ဖယ်ရှားပြီး view ဖွဲ့စည်းပုံကို ပိုမိုကောင်းမွန်အောင်ပြုလုပ်ကာ AI screener နှင့် manual screener အတွက် သီးခြားစီ ရှိနေခဲ့သော export logic ထပ်နေမှုများကို ပေါင်းစည်းလိုက်သည်။

ယုံကြည်စိတ်ချရမှု

စာကြောင်း 345 ကြောင်း export helper function များအတွက် test များနှင့် စာကြောင်း 539 ကြောင်း AI screener forms ပေါင်းစည်းမှုအတွက် test များ အပြင် ယခင်က မတည်ငြိမ်ခဲ့သော (flaky) test များကိုပါ ပြင်ဆင်ထားသည်။ Breaking change များ မရှိပါ။ ဝန်ဆောင်မှုအသုံးပြုသူဘက်မှ မြင်ကွင်းအတွက် မြှင့်တင်ထားသော Export လုပ်ဆောင်ချက် ကိုပါ ကြည့်ပါ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေသည်

shared service မှတစ်ဆင့် export လုပ်ခြင်းသည် tier စည်းမျဉ်းများကို ပြောင်းလဲမသွားစေပါ — အထွေထွေ public screen export သည် Tier 1 နှင့် နီးစပ်နေဆဲဖြစ်ပြီး အမည်ဖော်ထားသော သို့မဟုတ် ဖောက်သည်နှင့်ဆက်စပ်သော export သည် ဒေတာလုံခြုံရေး အရ Tier 2/3 စည်းမျဉ်းများကို လိုက်နာရမည်။
