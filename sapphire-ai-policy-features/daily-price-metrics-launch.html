Daily Price Metrics Tool Launch

The original launch of the daily price metrics tool — querying historical time-series financial data across 40+ metrics in 8 categories, with smart aggregation and flexible input handling built in from day one.

Metric Categories

PE Ratios

Historical P/E metrics and EPS data.

Price to Book

P/B ratios and book value metrics.

EV Multiples

EV/EBITDA and EV/Sales ratios.

Market Data

Prices, market cap, shares outstanding, dividends.

Debt Metrics

Debt levels and cash positions.

Financial Fundamentals

Revenue, EBITDA, profits.

Equity Metrics

ROE, payout ratios, equity values.

Fiscal Data

Reporting dates and quarters.

Smart Input Handling

Query by company name, ticker, or ID, over any date range, with individual field selection so a request for "just the close price" doesn't pull an entire metric category. Parameter validation tolerates the various formats an LLM might generate — stringified lists, single strings — and aggregation suggestions adjust automatically based on the requested date range. Results are cached for 24 hours.

Reliability

32 comprehensive tests covering input validation, formatting, caching, and edge cases — plus a dedicated test-prompts document written for LLM testing.

Data Handling Still Applies

Standalone public price and fundamentals data is generally Tier 1. Once a query is tied to a specific portfolio or engagement, treat it as Tier 2/3 per Data Security.

เปิดตัวเครื่องมือ Daily Price Metrics

การเปิดตัวครั้งแรกของเครื่องมือ daily price metrics — สืบค้นข้อมูล time-series ทางการเงินย้อนหลังครอบคลุมมากกว่า 40 ตัวชี้วัดใน 8 หมวดหมู่ พร้อมการรวมข้อมูล (aggregation) อัจฉริยะและการรับ input ที่ยืดหยุ่นตั้งแต่วันแรก

หมวดหมู่ตัวชี้วัด

PE Ratios

ตัวชี้วัด P/E ย้อนหลังและข้อมูล EPS

Price to Book

อัตราส่วน P/B และตัวชี้วัดมูลค่าทางบัญชี

EV Multiples

อัตราส่วน EV/EBITDA และ EV/Sales

Market Data

ราคา มูลค่าตลาด จำนวนหุ้นที่ออก และเงินปันผล

Debt Metrics

ระดับหนี้สินและสถานะเงินสด

Financial Fundamentals

รายได้ EBITDA และกำไร

Equity Metrics

ROE อัตราการจ่ายเงินปันผล และมูลค่าส่วนของผู้ถือหุ้น

Fiscal Data

วันที่รายงานและไตรมาส

การรับ Input อัจฉริยะ

สืบค้นได้ตามชื่อบริษัท ticker หรือ ID ในช่วงวันที่ใดก็ได้ พร้อมเลือกฟิลด์แต่ละรายการได้ ทำให้คำขอ "ขอแค่ราคาปิด" ไม่ต้องดึงข้อมูลทั้งหมวดหมู่มา การตรวจสอบพารามิเตอร์รองรับรูปแบบหลากหลายที่ LLM อาจสร้างขึ้น — รายการที่แปลงเป็น string, string เดี่ยว — และคำแนะนำการรวมข้อมูลจะปรับโดยอัตโนมัติตามช่วงวันที่ที่ร้องขอ ผลลัพธ์ถูกแคชไว้เป็นเวลา 24 ชั่วโมง

ความน่าเชื่อถือ

การทดสอบครอบคลุม 32 รายการ ครอบคลุมการตรวจสอบ input การจัดรูปแบบ การแคช และกรณีขอบเขตพิเศษ (edge case) พร้อมทั้งเอกสาร test-prompts เฉพาะที่เขียนขึ้นสำหรับการทดสอบกับ LLM

กฎการจัดการข้อมูลยังคงบังคับใช้เหมือนเดิม

ข้อมูลราคาและปัจจัยพื้นฐานสาธารณะแบบเดี่ยว ๆ โดยทั่วไปจัดเป็น Tier 1 เมื่อการสืบค้นถูกผูกเข้ากับพอร์ตหรือ engagement ใดโดยเฉพาะ ให้ถือว่าเป็น Tier 2/3 ตาม Data Security

Daily Price Metrics ကိရိယာ မိတ်ဆက်

daily price metrics ကိရိယာ၏ မူလစတင်မိတ်ဆက်မှု — အမျိုးအစား ၈ ခုအတွင်း ဘဏ္ဍာရေးဆိုင်ရာ metric ၄၀ ကျော်ကို ခြုံငုံသော historical time-series ဒေတာကို စုံစမ်းမေးမြန်းနိုင်ပြီး၊ ပထမနေ့မှစတင်၍ ထည့်သွင်းထားသော ဉာဏ်ရည်ရှိသော aggregation နှင့် ပြောင်းလွယ်ပြင်လွယ် input ကိုင်တွယ်မှုပါဝင်သည်။

Metric အမျိုးအစားများ

PE Ratios

Historical P/E metric များနှင့် EPS ဒေတာ

Price to Book

P/B အချိုးများနှင့် စာရင်းတန်ဖိုး metric များ

EV Multiples

EV/EBITDA နှင့် EV/Sales အချိုးများ

Market Data

ဈေးနှုန်း၊ market cap၊ ရှယ်ယာအရေအတွက်နှင့် ငွေခွဲဝေမှု

Debt Metrics

အကြွေးအဆင့်နှင့် ငွေသားအခြေအနေ

Financial Fundamentals

အသုံးအဆောင်ငွေ၊ EBITDA၊ အမြတ်

Equity Metrics

ROE၊ ငွေခွဲဝေမှုအချိုးနှင့် equity တန်ဖိုးများ

Fiscal Data

အစီရင်ခံရက်စွဲနှင့် သုံးလပတ်များ

ဉာဏ်ရည်ရှိသော Input ကိုင်တွယ်မှု

ကုမ္ပဏီအမည်၊ ticker သို့မဟုတ် ID ဖြင့် မည်သည့်ရက်စွဲအပိုင်းအခြားမဆို စုံစမ်းမေးမြန်းနိုင်ပြီး၊ field တစ်ခုချင်းစီရွေးချယ်နိုင်သဖြင့် "ပိတ်ဈေးတစ်ခုတည်း" ကို တောင်းဆိုချက်သည် metric အမျိုးအစားတစ်ခုလုံးကို ဆွဲယူစရာ မလိုတော့ပါ။ Parameter validation သည် LLM တစ်ခုက ဖန်တီးနိုင်သော ပုံစံအမျိုးမျိုးကို လက်ခံသည် — string အဖြစ်ပြောင်းထားသော list များ၊ string တစ်ခုတည်း — ပြီးတော့ aggregation အကြံပြုချက်များသည် တောင်းဆိုထားသော ရက်စွဲအပိုင်းအခြားပေါ် အခြေခံ၍ အလိုအလျောက် ချိန်ညှိပေးသည်။ ရလဒ်များကို ၂၄ နာရီကြာ cache ထားသည်

ယုံကြည်စိတ်ချရမှု

ကျယ်ကျယ်ပြန့်ပြန့် test ၃၂ ခု ဖြင့် input validation၊ ပုံစံချခြင်း၊ caching နှင့် edge case များကို ခြုံငုံစစ်ဆေးထားပြီး LLM စမ်းသပ်ခြင်းအတွက် အထူးရေးသားထားသော test-prompts စာတမ်းတစ်စောင်ပါ ပါဝင်သည်

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေသည်

အများပြည်သူ ဈေးနှုန်းနှင့် အခြေခံအချက်အလက် ဒေတာ တစ်ခုတည်းအနေနဲ့ ယေဘုယျအားဖြင့် Tier 1 ဖြစ်သည်။ query တစ်ခုသည် သီးခြား portfolio သို့မဟုတ် engagement တစ်ခုနှင့် ချိတ်ဆက်လိုက်သည်နှင့် Data Security အရ Tier 2/3 အဖြစ် ကိုင်တွယ်ပါ
