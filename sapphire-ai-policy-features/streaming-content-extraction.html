New in SAM · Feature Guide

Streaming Recovery & Session Handling

Refresh the page mid-response and SAM now picks the stream back up automatically, individual messages can be exported as their own PDF, and document uploads get proper session isolation.

Response Streaming Recovery

A new tracker model persists streaming state across page refreshes, with automatic recovery detection and stream continuation on reconnection. Stale trackers are cleaned up by a scheduled task, and a state machine validates the tracker's lifecycle to prevent inconsistent states.

Single Message PDF Export

Individual chat messages can now be exported as their own professionally formatted PDF, complete with citation formatting, using the same underlying sanitization pipeline as full-session export.

Document Session Handling

  • Session-based filtering for document progress WebSocket updates, so progress notifications don't leak across sessions.
  • Atomic transactions for document association, with proper error notification if association fails.
  • Deferred tracker persistence with Celery retry for resilience against transient failures.

Reliability

72 files changed, 8,666 insertions. Comprehensive unit tests for streaming recovery, citation utilities, PDF export, and document processing, plus frontend TypeScript tests for the new handlers and stores.

Data Handling Still Applies

Recovery and export mechanics don't change tier — an exported single message carries the same sensitivity as the conversation it came from. See Data Security.

ฟีเจอร์ใหม่ใน SAM · คู่มือฟีเจอร์

การกู้คืนการสตรีมมิ่งและการจัดการเซสชัน

หากรีเฟรชหน้าเว็บระหว่างที่ SAM กำลังตอบกลับ ระบบจะดึงสตรีมกลับมาต่อโดยอัตโนมัติ ข้อความแต่ละรายการสามารถส่งออกเป็น PDF ของตัวเองได้ และการอัปโหลดเอกสารก็มีการแยกเซสชันอย่างถูกต้อง

การกู้คืนการสตรีมมิ่งคำตอบ

โมเดล tracker ตัวใหม่จะเก็บสถานะการสตรีมมิ่งไว้แม้จะมีการรีเฟรชหน้าเว็บ พร้อมระบบตรวจจับการกู้คืนอัตโนมัติและดำเนินการสตรีมต่อเมื่อเชื่อมต่อใหม่ tracker ที่ค้างอยู่จะถูกล้างโดย scheduled task และ state machine จะตรวจสอบวงจรชีวิตของ tracker เพื่อป้องกันสถานะที่ไม่สอดคล้องกัน

การส่งออกข้อความเดี่ยวเป็น PDF

ขณะนี้ข้อความแชทแต่ละรายการสามารถส่งออกเป็นไฟล์ PDF ที่จัดรูปแบบอย่างมืออาชีพในตัวเองได้ พร้อมการจัดรูปแบบการอ้างอิง (citation) โดยใช้ sanitization pipeline เดียวกันกับที่ใช้ในการส่งออกทั้งเซสชัน

การจัดการเซสชันเอกสาร

  • การกรองตามเซสชันสำหรับการอัปเดต WebSocket ของความคืบหน้าเอกสาร เพื่อไม่ให้การแจ้งเตือนความคืบหน้ารั่วไหลข้ามเซสชัน
  • ธุรกรรมแบบ atomic สำหรับการเชื่อมโยงเอกสาร พร้อมการแจ้งเตือนข้อผิดพลาดที่เหมาะสมหากการเชื่อมโยงล้มเหลว
  • การบันทึก tracker แบบ deferred พร้อมการ retry ผ่าน Celery เพื่อความทนทานต่อความล้มเหลวชั่วคราว

ความน่าเชื่อถือ

เปลี่ยนแปลง 72 ไฟล์ เพิ่มโค้ด 8,666 บรรทัด มี unit test ที่ครอบคลุมสำหรับการกู้คืนการสตรีมมิ่ง เครื่องมือการอ้างอิง การส่งออก PDF และการประมวลผลเอกสาร รวมถึง TypeScript test ฝั่ง frontend สำหรับ handler และ store ใหม่

หลักการจัดการข้อมูลยังคงใช้เหมือนเดิม

กลไกการกู้คืนและการส่งออกไม่ได้เปลี่ยนแปลงระดับชั้นข้อมูล (tier) — ข้อความเดี่ยวที่ส่งออกมายังคงมีระดับความอ่อนไหวเท่ากับบทสนทนาต้นทาง ดูเพิ่มเติมที่ ความปลอดภัยของข้อมูล

SAM တွင် အသစ် · ဝန်ဆောင်မှုလမ်းညွှန်

စတရိမ်းမင်း ပြန်လည်ရယူခြင်း နှင့် Session စီမံခြင်း

SAM မှ ဖြေကြားနေစဉ် စာမျက်နှာကို refresh လုပ်လိုက်လျှင် stream ကို အလိုအလျောက် ပြန်ဆက်ယူနိုင်ပါပြီ၊ စာတိုတစ်ခုချင်းစီကို ၎င်း၏ PDF အနေဖြင့် export လုပ်နိုင်ပြီး document upload များအတွက်လည်း session ခွဲခြားမှု မှန်ကန်စွာ ရရှိပါသည်။

အဖြေစတရိမ်းမင်း ပြန်လည်ရယူခြင်း

tracker မော်ဒယ်အသစ်သည် စာမျက်နှာ refresh လုပ်သည့်တိုင်း streaming အခြေအနေကို ထိန်းသိမ်းထားပေးပြီး၊ ပြန်လည်ဆက်သွယ်သောအခါ အလိုအလျောက် ပြန်လည်ရှာဖွေမှု detection နှင့် stream ဆက်လက်လုပ်ဆောင်မှု ပါဝင်သည်။ အသုံးမကျတော့သော tracker များကို scheduled task မှ ရှင်းလင်းပေးပြီး၊ state machine တစ်ခုက tracker ၏ lifecycle ကို စစ်ဆေးအတည်ပြုကာ မကိုက်ညီသော အခြေအနေများ မဖြစ်ပေါ်စေရန် ကာကွယ်ပေးသည်။

စာတိုတစ်ခုချင်း PDF Export

ယခုအခါ chat စာတို တစ်ခုချင်းစီကို ကိုးကားချက် (citation) ပုံစံနှင့်တကွ ပရော်ဖက်ရှင်နယ်ပုံစံချထားသော ၎င်း၏ PDF တစ်ခုအနေဖြင့် export လုပ်နိုင်ပြီး၊ session တစ်ခုလုံး export လုပ်ရာတွင် အသုံးပြုသည့် sanitization pipeline အတိုင်းပင် အသုံးပြုသည်။

Document Session စီမံခြင်း

  • document တိုးတက်မှု WebSocket update များအတွက် session-based filtering ထားရှိသဖြင့် တိုးတက်မှုအသိပေးချက်များသည် session များကြား ယိုစိမ့်မှု မရှိစေပါ။
  • document ချိတ်ဆက်မှုအတွက် atomic transaction များ ပြုလုပ်ထားပြီး၊ ချိတ်ဆက်မှု မအောင်မြင်ပါက သင့်လျော်သော error အသိပေးချက် ပါဝင်သည်။
  • ယာယီ ချို့ယွင်းမှုများကို ခံနိုင်ရည်ရှိစေရန် Celery retry ဖြင့် deferred tracker persistence ကို အသုံးပြုထားသည်။

ယုံကြည်စိတ်ချရမှု

ဖိုင် 72 ဖိုင် ပြောင်းလဲပြီး ကုဒ်လိုင်း 8,666 လိုင်း ထပ်ထည့်ထားသည်။ streaming recovery၊ citation utility များ၊ PDF export နှင့် document processing အတွက် ကျယ်ပြန့်သော unit test များအပြင် handler နှင့် store အသစ်များအတွက် frontend TypeScript test များပါ ပါဝင်သည်။

ဒေတာစီမံမှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်နေဆဲ

ပြန်လည်ရယူခြင်းနှင့် export လုပ်ငန်းစဉ်များသည် tier ကို ပြောင်းလဲမပေးပါ — export လုပ်ထားသော စာတိုတစ်ခုသည် မူရင်း conversation ၏ အရေးကြီးမှုအဆင့်အတိုင်းပင် ရှိနေဆဲ ဖြစ်သည်။ ဒေတာလုံခြုံရေး တွင် ဆက်လက်ကြည့်ရှုပါ။
