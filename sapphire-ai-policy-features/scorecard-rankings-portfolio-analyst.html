Integrate Scorecards into SAMAI Chatbot

A new scorecard tool provider lets the chatbot pull team-scoped Scorecard and ValuationScorecard data straight into Investment Thesis and company-analysis conversations — so scorecard assessments show up alongside financials instead of requiring a separate lookup.

Overview

Scorecard data — both regular Scorecard and ValuationScorecard records — is now available to the chatbot as a first-class tool. When a user asks for an Investment Thesis or general company analysis, the chatbot can pull in the team's own scorecard assessments for that company. Scorecards are team-scoped: each team sees only the scorecards it created.

Implementation Plan

1. Scorecard Tool Provider

apps/chatbots/tool_providers/scorecard_tool.py

  • New ScorecardProvider class inheriting from ToolProvider
  • Queries scorecards by company (name, ticker, or ID)
  • Filters by scorecard type (factor types)
  • Includes both regular scorecards and valuation scorecards
  • Returns overall scores, factor scores, and comments
  • Formats results with analyst information
  • Uses get_context_value("team_id") to enforce team-scoped queries
  • Handles the case where no scorecards exist for a company

2. Tool Input Schema

apps/chatbots/tool_providers/scorecard_tool.py

FieldPurpose
queryCompany name, ticker, or ID
scorecard_typesList of types to include, or "all"
include_valuationBoolean — include valuation scorecards
include_detailsBoolean — detailed factor/subfactor scores
limitMax results per company

3. Tool Logic

Team scoping

Every query is scoped to team_id pulled from context.

Company matching

Joins with the Company model for name/ticker search.

Score breakdowns

Includes weighted scores and factor breakdowns.

Valuation formatting

Formats financial metrics (IRR, target prices) from valuation scorecards.

Attribution

Includes analyst names and creation dates.

LLM-ready output

Returns structured data suitable for LLM interpretation.

4. Tool Registration

apps/chatbots/apps.py — import ScorecardProvider and register it with tool_registry.register(ScorecardProvider()).

5. Tests

apps/chatbots/tests/test_scorecard_tool.py covers:

  • Team-scoped queries
  • Company lookup by name, ticker, or ID
  • Filtering by scorecard types
  • Valuation scorecard integration
  • Empty-results handling
  • Behavior with and without team context

6. Tool Priority

apps/chatbots/registry.py — "scorecard" is added to the priority_order list right after "company_financials", ensuring scorecard data is available early in tool selection.

7. Optional: Investment Thesis Generator Tool

apps/chatbots/tool_providers/investment_thesis_generator.py — a specialized tool that combines company financials, scorecards, and notes into a structured investment thesis, can save to the InvestmentThesis model on request, and formats output professionally.

Key Design Decisions

1

Team scoping

All scorecard queries are automatically filtered by team_id from context.

2

Tool separation

The scorecard tool stays separate from company_financials for modularity.

3

Data aggregation

Both scorecard types return in a single, unified format.

4

Citation support

Scorecard data carries proper citations back to its source — see Citation Management in SAM.

5

Performance

select_related/prefetch_related keep the underlying queries efficient.

Example Tool Usage

Example

"What's the investment thesis for Apple?"

For a request like this, the chatbot will:

  • Use the company_financials tool to get financial data
  • Use the scorecard tool to get the team's scorecard assessments
  • Potentially use document tools for relevant documents
  • Synthesize everything into a comprehensive investment thesis

How It Relates to Scorecard Rankings

This is a different surface for scorecard data than Portfolio Analyst: Scorecard Rankings. That feature pulls scorecard ratings into portfolio/watchlist batch queries inside the portfolio analyst tool. This feature makes scorecard data available to the general SAM chatbot for single-company Investment Thesis and analysis conversations. Both are team-scoped and both stay inside SAM — they simply serve different query patterns.

Benefits

Team-specific insights

Each team sees only its own scorecard assessments.

Comprehensive analysis

Combines quantitative (financials) and qualitative (scorecard) data in one answer.

Investment decision support

Provides structured evaluation data to support investment decisions.

Audit trail

Maintains clear attribution to analysts and dates.

Testing & Validation

LayerCoverage
Unit testsEach component tested in isolation
Integration testsTool tested with real database queries
End-to-end testsTested through the chatbot interface with sample queries
Team isolation testsVerifies teams only ever see their own scorecard data

Target: >90% coverage. Acceptance criteria require the tool to be registered and appear in the chatbot's available tools, correctly filter by team, search companies by name/ticker/ID, include both scorecard types, and surface in Investment Thesis queries — with team isolation verified before rollout.

Data Handling Still Applies

Scorecard assessments are SAM client data tied to a specific team's portfolio work — the same tier as any other fund holding or decision record on the Data Security page. Team scoping is an access control inside SAM, not a declassification.

Tier 3

Stays inside SAM

Scorecard scores, factor breakdowns, analyst comments, and any thesis generated from them are processed only inside SAM and approved Claude workflows — never pasted into free tools or verification models.

Check before quoting externally

Thesis summaries and commentary

If you're drafting commentary that references a scorecard-derived thesis outside SAM, treat client and company identifiers the same way you would any other Tier 3 content — anonymize or route through the approved workflow.

Scorecard များကို SAMAI Chatbot သို့ ပေါင်းစည်းခြင်း

Scorecard tool provider အသစ်တစ်ခုသည် chatbot အား team scope ပြုလုပ်ထားသော Scorecard နှင့် ValuationScorecard ဒေတာများကို Investment Thesis နှင့် ကုမ္ပဏီခွဲခြမ်းစိတ်ဖြာမှု စကားဝိုင်းများထဲသို့ တိုက်ရိုက် ထည့်သွင်းနိုင်စေသည် — scorecard အကဲဖြတ်ချက်များကို ငွေကြေးအချက်အလက်များနှင့်အတူ ပြသနိုင်ပြီး သီးခြားရှာဖွေရန် မလိုတော့ပါ။

ခြုံငုံသုံးသပ်ချက်

Scorecard ဒေတာ — ပုံမှန် Scorecard နှင့် ValuationScorecard မှတ်တမ်းနှစ်မျိုးလုံး — ယခု chatbot အတွက် first-class tool အဖြစ် ရရှိနိုင်ပြီဖြစ်သည်။ အသုံးပြုသူက Investment Thesis သို့မဟုတ် ယေဘုယျ ကုမ္ပဏီခွဲခြမ်းစိတ်ဖြာမှု တောင်းဆိုသောအခါ chatbot သည် ထိုကုမ္ပဏီအတွက် ၎င်း၏ team ကိုယ်တိုင် ပြုလုပ်ထားသော scorecard အကဲဖြတ်ချက်များကို ထည့်သွင်းနိုင်သည်။ Scorecard များသည် team scope ဖြစ်သည် — team တစ်ခုစီသည် မိမိတို့ ဖန်တီးထားသော scorecard များကိုသာ မြင်ရသည်။

1. Scorecard Tool Provider

  • ကုမ္ပဏီအလိုက် (အမည်၊ ticker သို့မဟုတ် ID) scorecard များကို ရှာဖွေသည်
  • Scorecard အမျိုးအစား (factor types) အလိုက် စစ်ထုတ်သည်
  • ပုံမှန် scorecard နှင့် valuation scorecard နှစ်မျိုးလုံး ပါဝင်သည်
  • စုစုပေါင်းရမှတ်၊ factor ရမှတ်များနှင့် မှတ်ချက်များကို ပြန်ပေးသည်
  • ရလဒ်များကို analyst အချက်အလက်နှင့်အတူ ပုံစံချသည်
  • team scope ဖြင့် ရှာဖွေမှုများကို အတည်ပြုစေရန် get_context_value("team_id") ကို အသုံးပြုသည်
  • ကုမ္ပဏီတစ်ခုအတွက် scorecard မရှိသည့်အခြေအနေကို ကိုင်တွယ်သည်

LLM-ready output

Scorecard Rankings နှင့် ဆက်စပ်ပုံ

ဤသည်မှာ Portfolio Analyst: Scorecard Rankings နှင့် မတူညီသော scorecard ဒေတာ ပြသနည်းတစ်ခု ဖြစ်သည်။ ထိုဖီချာသည် scorecard ရမှတ်များကို portfolio analyst tool အတွင်း portfolio/watchlist batch ရှာဖွေမှုများထဲသို့ ထည့်သွင်းသည်။ ဤဖီချာမူ scorecard ဒေတာကို ယေဘုယျ SAM chatbot တွင် ကုမ္ပဏီတစ်ခုချင်း Investment Thesis နှင့် ခွဲခြမ်းစိတ်ဖြာမှု စကားဝိုင်းများအတွက် ရရှိနိုင်စေသည်။ နှစ်ခုစလုံးသည် team scope ဖြစ်ပြီး SAM အတွင်း၌သာ ရှိနေသည် — ရှာဖွေမှု ပုံစံများ ကွဲပြားခြင်းသာ ကွာခြားသည်။

အကျိုးကျေးဇူးများ

Team အလိုက် ထိုးထွင်းသိမြင်မှု

Team တစ်ခုစီသည် မိမိတို့၏ scorecard အကဲဖြတ်ချက်များကိုသာ မြင်ရသည်။

ပြည့်စုံသော ခွဲခြမ်းစိတ်ဖြာမှု

အရေအတွက်ဆိုင်ရာ (ငွေကြေး) နှင့် အရည်အသွေးဆိုင်ရာ (scorecard) ဒေတာကို အဖြေတစ်ခုတည်းတွင် ပေါင်းစပ်ပေးသည်။

ရင်းနှီးမြှုပ်နှံမှု ဆုံးဖြတ်ချက် ပံ့ပိုးမှု

ရင်းနှီးမြှုပ်နှံမှု ဆုံးဖြတ်ချက်များကို ပံ့ပိုးရန် ဖွဲ့စည်းပုံရှိသော အကဲဖြတ်ဒေတာကို ပေးအပ်သည်။

Audit trail

Analyst များနှင့် ရက်စွဲများကို ရှင်းလင်းစွာ ဆက်စပ်ညွှန်းဆိုမှု ထိန်းသိမ်းသည်။

စမ်းသပ်မှုနှင့် အတည်ပြုခြင်း

အလွှာလွှမ်းခြုံမှု
Unit testsComponent တစ်ခုချင်းစီကို သီးခြားစီ စမ်းသပ်သည်
Integration testsTool ကို ဒေတာဘေ့စ် query အစစ်များဖြင့် စမ်းသပ်သည်
End-to-end testsနမူနာမေးခွန်းများဖြင့် chatbot interface မှတစ်ဆင့် စမ်းသပ်သည်
Team isolation testsTeam များသည် မိမိတို့၏ scorecard ဒေတာကိုသာ အမြဲမြင်ကြောင်း အတည်ပြုသည်

ပစ်မှတ်: coverage 90% ကျော်။ လက်ခံနိုင်သော စံသတ်မှတ်ချက်များအရ tool သည် register ဖြစ်ရမည်၊ chatbot ၏ tool စာရင်းတွင် ပေါ်လာရမည်၊ team အလိုက် မှန်ကန်စွာ စစ်ထုတ်ရမည်၊ ကုမ္ပဏီများကို အမည်/ticker/ID ဖြင့် ရှာဖွေနိုင်ရမည်၊ scorecard အမျိုးအစားနှစ်မျိုးလုံး ပါဝင်ရမည် နှင့် Investment Thesis မေးခွန်းများတွင် ပေါ်လာရမည် — rollout မတိုင်မီ team isolation ကို အတည်ပြုပြီး ဖြစ်ရမည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်ဆဲ

Scorecard အကဲဖြတ်ချက်များသည် သီးခြား team တစ်ခု၏ portfolio လုပ်ငန်းနှင့် ချိတ်ဆက်နေသော SAM client ဒေတာဖြစ်ပြီး Data Security စာမျက်နှာပေါ်ရှိ အခြားရန်ပုံငွေ ထိန်းချုပ်မှု သို့မဟုတ် ဆုံးဖြတ်ချက်မှတ်တမ်းများနည်းတူ tier အဆင့်တူညီသည်။ Team scoping သည် SAM အတွင်း access control တစ်ခုသာဖြစ်ပြီး declassification မဟုတ်ပါ။

Tier 3

SAM အတွင်း၌သာ ရှိနေသည်

Scorecard ရမှတ်များ၊ factor အသေးစိတ် ခွဲခြမ်းချက်များ၊ analyst မှတ်ချက်များ နှင့် ၎င်းတို့မှ ဖန်တီးထားသော thesis မည်သည်ကိုမဆို SAM နှင့် ခွင့်ပြုထားသော Claude workflow များအတွင်းသာ ပြုလုပ်ရမည် — အခမဲ့ tools သို့မဟုတ် စစ်ဆေးရေး မော်ဒယ်များထဲ ဘယ်တော့မှ ကူးထည့်ခြင်း မပြုရ။

ပြင်ပတွင် ကိုးကားမီ စစ်ဆေးပါ

Thesis အကျဉ်းချုပ်နှင့် ရှင်းလင်းချက်များ

SAM အပြင်ဘက်တွင် scorecard မှ ရရှိသော thesis ကို ကိုးကားသည့် ရှင်းလင်းချက်ကို ရေးနေပါက ဖောက်သည်နှင့် ကုမ္ပဏီ identifier များကို အခြား Tier 3 အကြောင်းအရာများကဲ့သို့ပင် ဆက်ဆံပါ — anonymize လုပ်ပါ သို့မဟုတ် ခွင့်ပြုထားသော workflow မှတစ်ဆင့် ဖြတ်သန်းပါ။
