New in SAM · Feature Guide

Partial Match Exclusions

Excluding companies from a screen now matches on partial company names and tickers instead of requiring an exact match — so \"exclude Pineapple\" correctly excludes every company with Pineapple in the name.

What Changed

  • Name exclusions now use partial matching (NOT LIKE) instead of exact matching.
  • Ticker exclusions also support partial matching — excluding AAPL excludes XNGS:AAPL too.
  • Improved handling for complex queries that combine inclusions and exclusions in the same request.

What You Can Ask

Example

"Show me Apple stocks but exclude anything with Pineapple."

Example

"Tech companies excluding MSFT."

Example

"All banks except those with 'Community' in the name."

Under the Hood

Inclusion filters use OR logic (match any of these); exclusion filters use AND logic (must not match any of these) — the separation that lets one query mix "show me X" with "but not Y" correctly. The NOT LIKE operator includes NULL handling so companies with missing name or ticker data aren't accidentally dropped.

Data Handling Still Applies

Exclusion criteria based on public names or tickers are generally Tier 1. If the exclusion list reflects a client's specific mandate, treat that framing as Tier 2/3 per Data Security — the same rule covered on Exclusion List Capabilities.

การยกเว้นด้วยการจับคู่บางส่วน

การยกเว้นบริษัทออกจากการสกรีนตอนนี้จับคู่กับชื่อบริษัทและ ticker แบบบางส่วนแล้ว แทนที่จะต้องจับคู่แบบตรงทั้งหมด — ทำให้ "exclude Pineapple" สามารถยกเว้นทุกบริษัทที่มีคำว่า Pineapple อยู่ในชื่อได้อย่างถูกต้อง

มีอะไรเปลี่ยนแปลงบ้าง

  • การยกเว้นตามชื่อตอนนี้ใช้การจับคู่บางส่วน (NOT LIKE) แทนการจับคู่แบบตรงทั้งหมด
  • การยกเว้นตาม ticker ก็รองรับการจับคู่บางส่วนเช่นกัน — การยกเว้น AAPL จะยกเว้น XNGS:AAPL ด้วย
  • ปรับปรุงการจัดการคำสั่งค้นหาที่ซับซ้อนซึ่งรวมทั้งการรวมและการยกเว้นไว้ในคำขอเดียวกัน

สิ่งที่คุณสามารถถามได้

ตัวอย่าง

"แสดงหุ้น Apple ให้ฉันดู แต่ยกเว้นอะไรก็ตามที่มีคำว่า Pineapple"

ตัวอย่าง

"บริษัทเทคโนโลยี ยกเว้น MSFT"

ตัวอย่าง

"ธนาคารทั้งหมด ยกเว้นที่มีคำว่า 'Community' อยู่ในชื่อ"

เบื้องหลังการทำงาน

ตัวกรองการรวมใช้ตรรกะ OR (จับคู่กับรายการใดรายการหนึ่ง); ตัวกรองการยกเว้นใช้ตรรกะ AND (ต้องไม่จับคู่กับรายการใดเลย) — การแยกกันนี้ทำให้คำสั่งค้นหาเดียวสามารถผสม "แสดง X" กับ "แต่ไม่รวม Y" ได้อย่างถูกต้อง ตัวดำเนินการ NOT LIKE มีการจัดการค่า NULL ด้วย เพื่อไม่ให้บริษัทที่ไม่มีข้อมูลชื่อหรือ ticker ถูกตัดออกไปโดยไม่ตั้งใจ

หลักการจัดการข้อมูลยังคงบังคับใช้

เกณฑ์การยกเว้นที่อิงตามชื่อหรือ ticker สาธารณะโดยทั่วไปจัดเป็น Tier 1 หากรายชื่อยกเว้นสะท้อนถึงข้อกำหนด (mandate) เฉพาะของลูกค้ารายใดรายหนึ่ง ให้ถือว่า framing นั้นเป็น Tier 2/3 ตามที่ระบุใน Data Security — เป็นกฎเดียวกับที่กล่าวถึงใน Exclusion List Capabilities

SAM တွင် အသစ်ထွက်ရှိ · ဖီချာလမ်းညွှန်

တစ်စိတ်တစ်ပိုင်း ကိုက်ညီမှုဖြင့် ချန်လှပ်ခြင်း

Screen တစ်ခုမှ ကုမ္ပဏီများကို ချန်လှပ်ခြင်းသည် ယခုအခါ အတိအကျ ကိုက်ညီမှု လိုအပ်မည့်အစား ကုမ္ပဏီအမည်နှင့် ticker များကို တစ်စိတ်တစ်ပိုင်းသာ ကိုက်ညီစေခြင်းဖြင့် လုပ်ဆောင်နိုင်ပါပြီ — ထို့ကြောင့် "exclude Pineapple" ဆိုလျှင် Pineapple ဟူသော စကားလုံးပါသော ကုမ္ပဏီအားလုံးကို မှန်ကန်စွာ ချန်လှပ်ပေးနိုင်သည်။

ပြောင်းလဲသွားသည်များ

  • အမည်ဖြင့် ချန်လှပ်ခြင်းသည် ယခုအခါ အတိအကျ ကိုက်ညီမှုအစား တစ်စိတ်တစ်ပိုင်း ကိုက်ညီမှု (NOT LIKE) ကို အသုံးပြုသည်
  • Ticker ဖြင့် ချန်လှပ်ခြင်းသည်လည်း တစ်စိတ်တစ်ပိုင်း ကိုက်ညီမှုကို ပံ့ပိုးသည် — AAPL ကို ချန်လှပ်လိုက်လျှင် XNGS:AAPL ကိုပါ ချန်လှပ်ပေးသည်
  • ထည့်သွင်းမှုနှင့် ချန်လှပ်မှုကို တောင်းဆိုမှုတစ်ခုတည်းတွင် ပေါင်းစပ်ထားသော ရှုပ်ထွေးသည့် query များအတွက် ကိုင်တွယ်မှုကို တိုးတက်စေခဲ့သည်

သင်မေးနိုင်သည့်ဥပမာများ

ဥပမာ

"Apple ရှယ်ယာများကို ပြပါ၊ ဒါပေမယ့် Pineapple ပါတဲ့ ဘာမဆို ချန်လှပ်ပါ။"

ဥပမာ

"MSFT ကို ချန်လှပ်ထားသော နည်းပညာကုမ္ပဏီများ။"

ဥပမာ

"အမည်တွင် 'Community' ပါသော ဘဏ်များမှလွဲ၍ ဘဏ်အားလုံး။"

နောက်ကွယ်တွင် အလုပ်လုပ်ပုံ

ထည့်သွင်းမှု filter များသည် OR logic (ဤစာရင်းများထဲမှ တစ်ခုခုနှင့် ကိုက်ညီရုံဖြင့် ရ) ကို အသုံးပြုပြီး ချန်လှပ်မှု filter များသည် AND logic (ဤစာရင်းများထဲမှ မည်သည့်တစ်ခုနှင့်မျှ မကိုက်ညီရ) ကို အသုံးပြုသည် — ဤသို့ ခွဲခြားထားခြင်းက query တစ်ခုတည်းအတွင်း "X ကို ပြပါ" နှင့် "Y မပါစေရ" ကို မှန်ကန်စွာ ရောနှောနိုင်စေသည်။ NOT LIKE operator တွင် NULL ကိုင်တွယ်မှု ပါဝင်သဖြင့် အမည် သို့မဟုတ် ticker အချက်အလက် မပြည့်စုံသော ကုမ္ပဏီများကို မတော်တဆ ချန်လှပ်မိခြင်း မဖြစ်စေပါ။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်

အများပြည်သူ အမည် သို့မဟုတ် ticker များအပေါ် အခြေခံသော ချန်လှပ်ရေး criteria များသည် ယေဘုယျအားဖြင့် Tier 1 ဖြစ်သည်။ ချန်လှပ်ရန် စာရင်းသည် ဖောက်သည်တစ်ဦး၏ တိကျသော mandate ကို ထင်ဟပ်ပါက ထို framing ကို Data Security အရ Tier 2/3 အဖြစ် သတ်မှတ်ပါ — Exclusion List Capabilities တွင် ဖော်ပြထားသည့် စည်းမျဉ်းတူညီစွာပင်။
