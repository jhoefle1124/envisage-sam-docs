System-wide Filtering Tools

The AI Screener and Chatbot Screener now share the same expanded filtering vocabulary: region and sub-region, international revenue exposure, and predefined company lists — so the same screen works whether you're driving it from the screener UI or asking SAM directly.

What's New

  • Regional filtering: filter companies by region and sub-region — North America, Europe, Asia-Pacific, and more, with full sub-region coverage.
  • International revenue screening: filter companies by the percentage of revenue earned from international markets.
  • Company list screening: screen against predefined company lists and watchlists, not just open-ended criteria.
  • Exchange-prefixed ticker support: tickers with an exchange prefix (e.g. NYSE:AAPL) are now handled correctly in list-based screens.

No breaking changes. All of this is additive and backward compatible — existing screens and chatbot queries behave exactly as before.

International Revenue Screening

Screens can now filter on the share of a company's revenue that comes from international markets — useful for isolating globally exposed names or, conversely, companies whose revenue is concentrated domestically.

Illustrative query

"Companies headquartered in Western Europe with more than 40% of revenue from international markets."

Company List & Watchlist Screening

Screens can now run against a predefined company list or watchlist instead of (or in addition to) open criteria — useful for checking which names on an existing list pass a new set of filters. Tickers with an exchange prefix, such as NYSE:AAPL or LSE:VOD, are now parsed correctly when included in a list.

Illustrative query

"From my watchlist, which companies are based in Southeast Asia with international revenue above 30%?"

Reliability

Thoroughly tested. The update ships with an expanded test suite (400+ lines of new tests) covering regional filtering, company list screening, and exchange-prefixed ticker edge cases, alongside updated coverage for the refactored query builder and services.

Data Handling Still Applies

Screening on public company fundamentals — region, revenue mix, exchange listing — is generally Tier 1. The tier shifts once a screen is tied to ENVISAGE or client context.

Usually fine

Open-market screening

Filtering public companies by region or international revenue on its own, with no client framing, stays close to Tier 1.

Check first

Screening against a client watchlist

Running a screen against a specific client's watchlist or portfolio list ties the query to that engagement — treat it as Tier 2/3 and keep it inside SAM and approved workflows, per Data Security.

เครื่องมือกรองข้อมูลทั่วทั้งระบบ

AI Screener และ Chatbot Screener ตอนนี้ใช้คำศัพท์การกรองข้อมูลชุดเดียวกันที่ขยายเพิ่มแล้ว ได้แก่ ภูมิภาคและภูมิภาคย่อย สัดส่วนรายได้จากต่างประเทศ และรายชื่อบริษัทที่กำหนดไว้ล่วงหน้า — ทำให้การสกรีนแบบเดียวกันใช้งานได้ไม่ว่าจะสั่งผ่านหน้า screener หรือถาม SAM โดยตรง

มีอะไรใหม่บ้าง

  • การกรองตามภูมิภาค: กรองบริษัทตามภูมิภาคและภูมิภาคย่อย — อเมริกาเหนือ ยุโรป เอเชียแปซิฟิก และอื่น ๆ ครอบคลุมภูมิภาคย่อยทั้งหมด
  • การสกรีนรายได้จากต่างประเทศ: กรองบริษัทตามสัดส่วนรายได้ที่มาจากตลาดต่างประเทศ
  • การสกรีนตามรายชื่อบริษัท: สกรีนเทียบกับรายชื่อบริษัทหรือ watchlist ที่กำหนดไว้ล่วงหน้า ไม่ใช่แค่เกณฑ์แบบเปิดกว้างเท่านั้น
  • รองรับ ticker ที่มี exchange prefix: ticker ที่มี exchange prefix (เช่น NYSE:AAPL) ตอนนี้ถูกจัดการได้อย่างถูกต้องในการสกรีนแบบใช้รายชื่อ

ไม่มีการเปลี่ยนแปลงที่กระทบระบบเดิม ทั้งหมดนี้เป็นการเพิ่มเติมและเข้ากันได้กับของเดิม (backward compatible) — หน้าจอสกรีนและคำถามผ่านแชทบอทที่มีอยู่เดิมยังทำงานเหมือนเดิมทุกประการ

การสกรีนรายได้จากต่างประเทศ

การสกรีนตอนนี้สามารถกรองตามสัดส่วนรายได้ของบริษัทที่มาจากตลาดต่างประเทศได้แล้ว — มีประโยชน์สำหรับการคัดกรองบริษัทที่มีการดำเนินธุรกิจทั่วโลก หรือในทางกลับกัน บริษัทที่มีรายได้กระจุกตัวอยู่ในประเทศ

ตัวอย่างคำถาม

"บริษัทที่มีสำนักงานใหญ่ในยุโรปตะวันตกและมีรายได้จากต่างประเทศมากกว่า 40%"

การสกรีนตามรายชื่อบริษัทและ Watchlist

การสกรีนตอนนี้สามารถทำงานเทียบกับรายชื่อบริษัทหรือ watchlist ที่กำหนดไว้ล่วงหน้าแทน (หรือเพิ่มเติมจาก) เกณฑ์แบบเปิดกว้างได้แล้ว — มีประโยชน์สำหรับตรวจสอบว่าบริษัทในรายชื่อที่มีอยู่ผ่านเกณฑ์กรองชุดใหม่หรือไม่ ticker ที่มี exchange prefix เช่น NYSE:AAPL หรือ LSE:VOD ตอนนี้ถูกแยกวิเคราะห์ได้อย่างถูกต้องเมื่อรวมอยู่ในรายชื่อ

ตัวอย่างคำถาม

"จากรายชื่อ watchlist ของฉัน บริษัทใดบ้างที่ตั้งอยู่ในเอเชียตะวันออกเฉียงใต้และมีรายได้จากต่างประเทศมากกว่า 30%?"

การเปลี่ยนแปลงในผลลัพธ์

ส่วนการเปลี่ยนแปลง
ตารางผลลัพธ์ตอนนี้แสดงเปอร์เซ็นต์รายได้จากต่างประเทศควบคู่กับฟิลด์เดิมที่มีอยู่
การแสดงผลลัพธ์บริษัทตอนนี้แสดงข้อมูลภูมิภาคสำหรับบริษัทที่ตรงกันแต่ละแห่ง
การประมวลผลคำสั่งค้นหาปรับโครงสร้างใหม่เป็นโมดูลที่เจาะจง (การจัดรูปแบบ การตรวจสอบความถูกต้อง การสร้างคำสั่งค้นหา) เพื่อผลลัพธ์ที่สม่ำเสมอและดูแลรักษาง่ายขึ้น

ความน่าเชื่อถือ

ผ่านการทดสอบอย่างละเอียด อัปเดตนี้มาพร้อมชุดทดสอบที่ขยายเพิ่ม (โค้ดทดสอบใหม่กว่า 400 บรรทัด) ครอบคลุมการกรองตามภูมิภาค การสกรีนตามรายชื่อบริษัท และกรณี edge case ของ ticker ที่มี exchange prefix พร้อมทั้งอัปเดตการทดสอบสำหรับ query builder และ services ที่ปรับโครงสร้างใหม่

หลักการจัดการข้อมูลยังคงบังคับใช้

การสกรีนตามข้อมูลพื้นฐานสาธารณะของบริษัท — ภูมิภาค สัดส่วนรายได้ การจดทะเบียนในตลาดหลักทรัพย์ — โดยทั่วไปจัดเป็น Tier 1 ระดับชั้นข้อมูลจะเปลี่ยนไปเมื่อการสกรีนถูกผูกกับบริบทของ ENVISAGE หรือลูกค้า

โดยทั่วไปไม่มีปัญหา

การสกรีนตลาดแบบเปิด

การกรองบริษัทสาธารณะตามภูมิภาคหรือรายได้จากต่างประเทศเพียงอย่างเดียว โดยไม่มีการผูกบริบทลูกค้า ยังคงใกล้เคียงกับ Tier 1

ตรวจสอบก่อน

การสกรีนเทียบกับ Watchlist ของลูกค้า

การรันการสกรีนเทียบกับ watchlist หรือรายชื่อพอร์ตของลูกค้ารายใดรายหนึ่งโดยเฉพาะ ทำให้คำสั่งค้นหาผูกกับงานนั้น — ควรถือว่าเป็น Tier 2/3 และเก็บไว้ภายใน SAM และ workflow ที่ได้รับอนุมัติ ตามที่ระบุใน Data Security

SAM တွင် အသစ်ထွက်ရှိ · ဖီချာလမ်းညွှန်

စနစ်တစ်ခုလုံးအတွက် စစ်ထုတ်ခြင်း ကိရိယာများ

AI Screener နှင့် Chatbot Screener တို့သည် ယခုအခါ ကျယ်ပြန့်စွာ တိုးချဲ့ထားသော filtering ဝေါဟာရများကို အတူတကွ အသုံးပြုကြသည် — ဒေသနှင့် ဒေသခွဲ၊ နိုင်ငံတကာ ဝင်ငွေရလဒ်နှင့် ကြိုတင်သတ်မှတ်ထားသော ကုမ္ပဏီစာရင်းများ — ထို့ကြောင့် screener UI မှတစ်ဆင့် အသုံးပြုသည်ဖြစ်စေ SAM ကို တိုက်ရိုက်မေးသည်ဖြစ်စေ တူညီသော screen တစ်ခုတည်း အလုပ်လုပ်နိုင်သည်။

အသစ်ပါဝင်လာသည်များ

  • ဒေသအလိုက် filtering: ဒေသနှင့် ဒေသခွဲအလိုက် ကုမ္ပဏီများကို filter လုပ်ခြင်း — မြောက်အမေရိက၊ ဥရောပ၊ အာရှပစိဖိတ် စသည်တို့အပါအဝင် ဒေသခွဲအားလုံးကို အပြည့်အစုံ လွှမ်းခြုံသည်
  • နိုင်ငံတကာ ဝင်ငွေ စစ်ထုတ်ခြင်း: ကုမ္ပဏီများကို နိုင်ငံတကာစျေးကွက်များမှ ရရှိသော ဝင်ငွေ ရာခိုင်နှုန်းအလိုက် filter လုပ်ခြင်း
  • ကုမ္ပဏီစာရင်းအလိုက် စစ်ထုတ်ခြင်း: ကြိုတင်သတ်မှတ်ထားသော ကုမ္ပဏီစာရင်းများ သို့မဟုတ် watchlist များအပေါ် အခြေခံ၍ စစ်ထုတ်ခြင်း — ဖွင့်ထားသော စည်းကမ်းချက်များသာမက
  • Exchange-prefix ပါ ticker ပံ့ပိုးမှု: exchange prefix ပါသော ticker များ (ဥပမာ NYSE:AAPL) ကို ယခုအခါ စာရင်းအခြေပြု screen များတွင် မှန်ကန်စွာ ကိုင်တွယ်နိုင်ပြီ

ပြောင်းလဲမှု ပျက်စီးစေခြင်း မရှိပါ။ ဤအားလုံးသည် ထပ်ဆောင်းသာဖြစ်ပြီး backward compatible ဖြစ်သည် — ရှိပြီးသား screen များနှင့် chatbot query များသည် ယခင်အတိုင်း အတိအကျ အလုပ်လုပ်ဆဲဖြစ်သည်။

နိုင်ငံတကာ ဝင်ငွေ စစ်ထုတ်ခြင်း

Screen များသည် ယခုအခါ ကုမ္ပဏီတစ်ခု၏ ဝင်ငွေအနက် နိုင်ငံတကာစျေးကွက်များမှ ရရှိသော ဝေစုကို filter လုပ်နိုင်ပြီ — ကမ္ဘာလုံးဆိုင်ရာ လုပ်ငန်းလည်ပတ်မှုရှိသော ကုမ္ပဏီများကို ခွဲထုတ်ရန် သို့မဟုတ် ပြောင်းပြန်အားဖြင့် ဝင်ငွေအများစုကို ပြည်တွင်း၌သာ ရရှိသော ကုမ္ပဏီများကို ခွဲထုတ်ရန် အသုံးဝင်သည်။

ဥပမာ query

"အနောက်ဥရောပတွင် အဓိကရုံးစိုက်ပြီး ဝင်ငွေ၏ 40% ထက်ပိုသော ပမာဏကို နိုင်ငံတကာစျေးကွက်များမှ ရရှိသော ကုမ္ပဏီများ။"

ကုမ္ပဏီစာရင်းနှင့် Watchlist စစ်ထုတ်ခြင်း

Screen များသည် ယခုအခါ ဖွင့်ထားသော စည်းကမ်းချက်အစား (သို့မဟုတ် ထပ်ဆောင်း၍) ကြိုတင်သတ်မှတ်ထားသော ကုမ္ပဏီစာရင်း သို့မဟုတ် watchlist အပေါ် အခြေခံ၍ လည်ပတ်နိုင်ပြီ — ရှိပြီးသား စာရင်းရှိ ကုမ္ပဏီများ filter အသစ်တစ်ခုကို ဖြတ်ကျော်နိုင်မှု ရှိမရှိ စစ်ဆေးရန် အသုံးဝင်သည်။ NYSE:AAPL သို့မဟုတ် LSE:VOD ကဲ့သို့ exchange prefix ပါသော ticker များကို စာရင်းတွင် ထည့်သွင်းသောအခါ ယခုအခါ မှန်ကန်စွာ parse လုပ်နိုင်ပြီ။

ဥပမာ query

"ကျွန်ုပ်၏ watchlist မှ မည်သည့်ကုမ္ပဏီများသည် အရှေ့တောင်အာရှတွင် အခြေစိုက်ပြီး နိုင်ငံတကာဝင်ငွေ 30% ထက်ပိုသနည်း?"

ရလဒ်များတွင် ပြောင်းလဲသွားသည်များ

အပိုင်းပြောင်းလဲမှု
ရလဒ်ဇယားရှိပြီးသား field များနှင့်အတူ နိုင်ငံတကာဝင်ငွေ ရာခိုင်နှုန်းကို ယခုပြသသည်
ကုမ္ပဏီရလဒ် ပြသမှုကိုက်ညီသော ကုမ္ပဏီတစ်ခုစီအတွက် ဒေသအချက်အလက်ကို ယခုပြသသည်
Query ပြုပြင်ခြင်းပိုမို တသမတ်တည်းရှိပြီး ထိန်းသိမ်းရလွယ်ကူသော ရလဒ်များအတွက် အာရုံစိုက်သော module များ (ပုံစံချခြင်း၊ စစ်ဆေးခြင်း၊ query တည်ဆောက်ခြင်း) အဖြစ် ပြန်လည်ဖွဲ့စည်းထားသည်

ယုံကြည်စိတ်ချရမှု

အသေးစိတ် စမ်းသပ်ပြီးဖြစ်သည်။ ဤအပ်ဒိတ်တွင် ဒေသအလိုက် filtering၊ ကုမ္ပဏီစာရင်း စစ်ထုတ်ခြင်း နှင့် exchange-prefix ပါ ticker edge case များကို လွှမ်းခြုံသော ကျယ်ပြန့်သည့် test suite အသစ် (line 400+) ပါဝင်ပြီး ပြန်လည်ဖွဲ့စည်းထားသော query builder နှင့် services များအတွက် test coverage ကိုလည်း အပ်ဒိတ်လုပ်ထားသည်။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်

အများပြည်သူ ကုမ္ပဏီ အခြေခံအချက်အလက် — ဒေသ၊ ဝင်ငွေအချိုး၊ ရှယ်ယာဈေးကွက် စာရင်းသွင်းမှု — အပေါ် အခြေခံ၍ စစ်ထုတ်ခြင်းသည် ယေဘုယျအားဖြင့် Tier 1 ဖြစ်သည်။ screen တစ်ခုသည် ENVISAGE သို့မဟုတ် ဖောက်သည် context နှင့် ချိတ်ဆက်လိုက်သည်နှင့် tier ပြောင်းသွားသည်။

များသောအားဖြင့် ပြဿနာမရှိ

ပွင့်လင်းစျေးကွက် Screening

အများပြည်သူ ကုမ္ပဏီများကို ဖောက်သည် framing မပါဘဲ ဒေသ သို့မဟုတ် နိုင်ငံတကာဝင်ငွေအလိုက်သာ filter လုပ်ခြင်းသည် Tier 1 နှင့် နီးစပ်နေဆဲဖြစ်သည်

ဦးစွာ စစ်ဆေးပါ

ဖောက်သည် Watchlist အပေါ် Screening

ဖောက်သည်တစ်ဦးချင်းစီ၏ watchlist သို့မဟုတ် portfolio list အပေါ် screen တစ်ခုကို လည်ပတ်ခြင်းသည် query ကို ထို engagement နှင့် ချည်နှောင်လိုက်ခြင်းဖြစ်သည် — ၎င်းကို Tier 2/3 အဖြစ် သတ်မှတ်ပြီး Data Security အရ SAM နှင့် ခွင့်ပြုထားသော workflow များအတွင်းသာ ထားရှိပါ
