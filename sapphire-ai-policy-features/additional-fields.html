New in SAM · Feature Guide

Adding up to Sixty Additional Fields on Screens

Portfolio analyst now returns 60+ additional financial and company fields, bringing it to full parity with our database schema — and a new include_fields option lets you request only the fields you actually need.

What's New

  • 60+ new fields across financial metrics, valuation metrics, company fundamentals, and performance indicators — closing the gap between the portfolio analyst tool and the underlying database schema.
  • Selective field inclusion via a new include_fields parameter, so a query can request a specific subset of data instead of everything.
  • Backward compatible: if include_fields isn't specified, the tool still returns the full field set as before.

Field Categories

Financial Metrics

ROA, ROE, Revenue Growth Rate, and related profitability and growth measures.

Valuation Metrics

EV/EBITDA, Price-to-Sales, FCF Yield, and other valuation multiples.

Company Fundamentals

Market Cap, Industry, Country, and other core company identifiers.

Performance Indicators

Quarterly revenue growth, quarterly earnings growth, and related period-over-period measures.

Selective Field Inclusion

The new include_fields parameter lets a query ask for exactly the data it needs rather than the full field set every time. This reduces response size and improves performance, which matters most on large portfolios or batch queries.

1

Omitted

Not specifying include_fields returns the full field set, exactly as the tool behaved before this change.

2

Specified

Naming specific fields returns only those — smaller, faster responses when you don't need everything.

What You Can Ask

Example

"Show me ROE and FCF yield for my portfolio companies."

Example

"Just give me market cap and industry for these tickers — I don't need the full profile."

Behind the scenes, SAM maps a request like this to the relevant fields via include_fields, so you get a focused answer instead of a full data dump you then have to sift through.

Reliability

Backward compatible. Existing queries that don't use include_fields are unaffected. Testing covered correct data return for all new fields, various include_fields combinations, and formatting for percentages, currency, and dates. Field names follow existing database schema conventions.

Data Handling Still Applies

Public financial metrics and company fundamentals on their own are generally Tier 1. As with scorecards and screening, the tier shifts once results are tied to a specific ENVISAGE portfolio or client context.

Usually fine

Public metrics, standalone

Pulling ROE, EV/EBITDA, or market cap for a publicly listed company on its own stays close to Tier 1.

Check first

Tied to a portfolio or watchlist

Returning these fields for a specific client's portfolio or watchlist ties the data to that engagement — treat it as Tier 2/3 and keep it inside SAM and approved workflows, per Data Security.

ใหม่ใน SAM · คู่มือฟีเจอร์

การเพิ่มฟิลด์เสริมสูงสุด 60 ฟิลด์ในหน้าจอคัดกรอง

ตอนนี้ Portfolio Analyst ส่งคืนฟิลด์ทางการเงินและข้อมูลบริษัทเพิ่มเติมกว่า 60 ฟิลด์ ทำให้เทียบเท่ากับ schema ฐานข้อมูลของเราอย่างสมบูรณ์ — และตัวเลือกใหม่ include_fields ช่วยให้คุณขอเฉพาะฟิลด์ที่ต้องการจริง ๆ

มีอะไรใหม่

  • ฟิลด์ใหม่กว่า 60 ฟิลด์ ครอบคลุมตัวชี้วัดทางการเงิน ตัวชี้วัดการประเมินมูลค่า ปัจจัยพื้นฐานของบริษัท และตัวชี้วัดผลการดำเนินงาน — ปิดช่องว่างระหว่างเครื่องมือ portfolio analyst กับ schema ฐานข้อมูลที่รองรับอยู่
  • การเลือกฟิลด์แบบเจาะจง ผ่านพารามิเตอร์ใหม่ include_fields ทำให้ query สามารถขอข้อมูลเฉพาะบางส่วนแทนที่จะขอทั้งหมด
  • ใช้งานร่วมกับของเดิมได้: หากไม่ระบุ include_fields เครื่องมือจะยังคงส่งคืนชุดฟิลด์เต็มเหมือนเดิม

หมวดหมู่ฟิลด์

ตัวชี้วัดทางการเงิน

ROA, ROE, Revenue Growth Rate และตัวชี้วัดความสามารถในการทำกำไรและการเติบโตที่เกี่ยวข้อง

ตัวชี้วัดการประเมินมูลค่า

EV/EBITDA, Price-to-Sales, FCF Yield และตัวคูณการประเมินมูลค่าอื่น ๆ

ปัจจัยพื้นฐานของบริษัท

Market Cap, Industry, Country และตัวระบุตัวตนหลักอื่น ๆ ของบริษัท

ตัวชี้วัดผลการดำเนินงาน

การเติบโตของรายได้รายไตรมาส การเติบโตของกำไรรายไตรมาส และตัวชี้วัดเปรียบเทียบช่วงเวลาที่เกี่ยวข้อง

การเลือกฟิลด์แบบเจาะจง

พารามิเตอร์ใหม่ include_fields ช่วยให้ query สามารถขอข้อมูลที่ต้องการจริง ๆ แทนที่จะขอชุดฟิลด์เต็มทุกครั้ง วิธีนี้ช่วยลดขนาดการตอบกลับและเพิ่มประสิทธิภาพ ซึ่งสำคัญที่สุดสำหรับพอร์ตขนาดใหญ่หรือ query แบบ batch

1

ไม่ระบุ

การไม่ระบุ include_fields จะส่งคืนชุดฟิลด์เต็ม เหมือนกับพฤติกรรมเดิมของเครื่องมือก่อนการเปลี่ยนแปลงนี้

2

ระบุ

การระบุชื่อฟิลด์เฉพาะจะส่งคืนเฉพาะฟิลด์เหล่านั้น — การตอบกลับที่เล็กลงและเร็วขึ้นเมื่อคุณไม่ต้องการข้อมูลทั้งหมด

สิ่งที่คุณสามารถถามได้

ตัวอย่าง

"แสดง ROE และ FCF yield สำหรับบริษัทในพอร์ตของฉัน"

ตัวอย่าง

"ขอแค่ market cap และ industry สำหรับ ticker เหล่านี้ — ไม่ต้องการโปรไฟล์เต็ม"

เบื้องหลัง SAM จะจับคู่คำขอลักษณะนี้กับฟิลด์ที่เกี่ยวข้องผ่าน include_fields ทำให้คุณได้คำตอบที่ตรงจุดแทนที่จะได้ข้อมูลดิบจำนวนมากที่ต้องมาคัดกรองเอง

ความน่าเชื่อถือ

ใช้งานร่วมกับของเดิมได้ query เดิมที่ไม่ได้ใช้ include_fields จะไม่ได้รับผลกระทบ การทดสอบครอบคลุมการส่งคืนข้อมูลที่ถูกต้องสำหรับฟิลด์ใหม่ทั้งหมด การผสมผสาน include_fields ต่าง ๆ และการจัดรูปแบบเปอร์เซ็นต์ สกุลเงิน และวันที่ ชื่อฟิลด์เป็นไปตามข้อตกลง schema ฐานข้อมูลที่มีอยู่เดิม

หลักการจัดการข้อมูลยังคงใช้บังคับ

ตัวชี้วัดทางการเงินสาธารณะและปัจจัยพื้นฐานของบริษัทโดยลำพัง โดยทั่วไปจัดเป็น Tier 1 เช่นเดียวกับ scorecard และการคัดกรอง ระดับชั้นข้อมูลจะเปลี่ยนไปเมื่อผลลัพธ์ถูกผูกกับพอร์ตหรือบริบทลูกค้าของ ENVISAGE โดยเฉพาะ

โดยทั่วไปไม่มีปัญหา

ตัวชี้วัดสาธารณะ แบบเดี่ยว

การดึง ROE, EV/EBITDA หรือ market cap สำหรับบริษัทมหาชนโดยลำพัง ยังคงใกล้เคียงกับ Tier 1

ตรวจสอบก่อน

ผูกกับพอร์ตหรือ watchlist

การส่งคืนฟิลด์เหล่านี้สำหรับพอร์ตหรือ watchlist ของลูกค้ารายใดรายหนึ่งโดยเฉพาะ ทำให้ข้อมูลผูกกับงานนั้น — ให้ถือว่าเป็น Tier 2/3 และเก็บไว้ภายใน SAM และ workflow ที่ได้รับอนุมัติ ตามที่ระบุใน ความปลอดภัยข้อมูล

SAM ရှိ အသစ် · Feature Guide

မျက်နှာပြင်များတွင် အပိုဒေတာအကွက် ၆၀ အထိ ထည့်သွင်းခြင်း

Portfolio analyst သည် ယခုအခါ ဘဏ္ဍာရေးနှင့် ကုမ္ပဏီဒေတာအကွက် ၆၀ ကျော် ထပ်မံပေးပို့နိုင်ပြီဖြစ်ပြီး ကျွန်ုပ်တို့၏ ဒေတာဘေ့စ် schema နှင့် အပြည့်အဝ ကိုက်ညီအောင် ပြုလုပ်ထားသည် — ပြီးတော့ အသစ်ဖြစ်သော include_fields ရွေးချယ်မှုက သင်လိုအပ်သည့်အကွက်များကိုသာ တောင်းဆိုနိုင်စေသည်။

အသစ်များ

  • အကွက်အသစ် ၆၀ ကျော် ဘဏ္ဍာရေးညွှန်းကိန်း၊ တန်ဖိုးဆန်းစစ်ညွှန်းကိန်း၊ ကုမ္ပဏီအခြေခံအချက်အလက် နှင့် စွမ်းဆောင်ရည်ညွှန်းကိန်းများ ပါဝင်သည် — portfolio analyst tool နှင့် database schema အကြား ကွာဟမှုကို ပိတ်ပေးသည်။
  • ရွေးချယ်၍ အကွက်ထည့်သွင်းခြင်း အသစ်ဖြစ်သော include_fields parameter မှတစ်ဆင့် query တစ်ခုသည် အချက်အလက်အားလုံးအစား သီးခြားအပိုင်းကိုသာ တောင်းဆိုနိုင်သည်။
  • နောက်ကြောင်းပြန်လိုက်ဖက်မှု: include_fields ကို မသတ်မှတ်ပါက tool သည် ယခင်ကအတိုင်း အကွက်အစုံကို ဆက်လက်ပေးပို့ပါမည်။

အကွက်အမျိုးအစားများ

ဘဏ္ဍာရေးညွှန်းကိန်းများ

ROA, ROE, Revenue Growth Rate နှင့် ဆက်စပ် အမြတ်အစွန်းနှင့် တိုးတက်မှု ညွှန်းကိန်းများ

တန်ဖိုးဆန်းစစ်ညွှန်းကိန်းများ

EV/EBITDA, Price-to-Sales, FCF Yield နှင့် အခြား valuation multiple များ

ကုမ္ပဏီအခြေခံအချက်အလက်

Market Cap, Industry, Country နှင့် အခြား ကုမ္ပဏီအဓိက identifier များ

စွမ်းဆောင်ရည်ညွှန်းကိန်းများ

သုံးလပတ် ဝင်ငွေတိုးတက်မှု၊ သုံးလပတ် အမြတ်တိုးတက်မှု နှင့် ဆက်စပ် period-over-period ညွှန်းကိန်းများ

ရွေးချယ်၍ အကွက်ထည့်သွင်းခြင်း

အသစ်ဖြစ်သော include_fields parameter က query တစ်ခုအား အကွက်အစုံအစား လိုအပ်သောဒေတာကိုသာ တောင်းဆိုနိုင်စေသည်။ ၎င်းက response size ကို လျှော့ချပြီး စွမ်းဆောင်ရည်ကို မြှင့်တင်ပေးသည် — portfolio ကြီးများ (သို့) batch query များတွင် အရေးအကြီးဆုံးဖြစ်သည်။

1

ချန်ထားခြင်း

include_fields ကို မသတ်မှတ်ခြင်းသည် ဤပြောင်းလဲမှုမတိုင်မီက tool ၏ အမူအကျင့်အတိုင်း အကွက်အစုံကို ပြန်ပေးသည်။

2

သတ်မှတ်ခြင်း

သီးခြားအကွက်များကို အမည်ဖော်ခြင်းသည် ၎င်းတို့ကိုသာ ပြန်ပေးသည် — အားလုံးမလိုအပ်သည့်အခါ သေးငယ်၍ မြန်ဆန်သော response များ ရရှိသည်။

မေးမြန်းနိုင်သည့်အရာများ

ဥပမာ

"ကျွန်ုပ်၏ portfolio ကုမ္ပဏီများအတွက် ROE နှင့် FCF yield ကို ပြပါ။"

ဥပမာ

"ဤ ticker များအတွက် market cap နှင့် industry ကိုသာ ပေးပါ — ကျွန်ုပ် ပရိုဖိုင်အပြည့်အစုံ မလိုအပ်ပါ။"

နောက်ကွယ်တွင် SAM သည် ဤကဲ့သို့သော တောင်းဆိုချက်ကို include_fields မှတစ်ဆင့် သက်ဆိုင်ရာ အကွက်များနှင့် ချိတ်ဆက်ပေးသဖြင့် သင်သည် နောက်မှ ကိုယ်တိုင် စစ်ထုတ်ရမည့် ဒေတာအလုံးအရင်းအစား ဦးတည်ချက်ရှိသော အဖြေကို ရရှိသည်။

ယုံကြည်စိတ်ချရမှု

နောက်ကြောင်းပြန်လိုက်ဖက်မှု။ include_fields ကို မသုံးသေးသော query များ ရှိနှင့်ပြီးသားများသည် ခံစားရမှု မရှိပါ။ စမ်းသပ်မှုများတွင် အကွက်အသစ်အားလုံးအတွက် ဒေတာမှန်ကန်စွာ ပြန်ပေးမှု၊ include_fields ပေါင်းစပ်မှုအမျိုးမျိုး နှင့် ရာခိုင်နှုန်း၊ ငွေကြေး၊ ရက်စွဲ format များ ပါဝင်သည်။ Field name များသည် လက်ရှိ database schema convention များနှင့် ကိုက်ညီသည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်း ဆက်လက်သက်ရောက်

အများပြည်သူ ဘဏ္ဍာရေးညွှန်းကိန်းများနှင့် ကုမ္ပဏီအခြေခံအချက်အလက်များကို တစ်ကိုယ်တည်းသုံးလျှင် ယေဘုယျအားဖြင့် Tier 1 ဖြစ်သည်။ scorecard နှင့် screening ကဲ့သို့ပင် ရလဒ်များကို ENVISAGE ၏ သီးခြား portfolio (သို့) client context နှင့် ချိတ်ဆက်လိုက်သောအခါ tier ပြောင်းလဲသွားသည်။

ပုံမှန်အားဖြင့် ကျန်

အများပြည်သူ ညွှန်းကိန်းများ၊ တစ်ကိုယ်တည်း

အများပြည်သူ စာရင်းဝင်ကုမ္ပဏီတစ်ခုအတွက် ROE, EV/EBITDA (သို့) market cap ကို တစ်ကိုယ်တည်းရယူခြင်းသည် Tier 1 နှင့် နီးစပ်စွာ ကျန်ရှိသည်။

အရင်စစ်ဆေးပါ

Portfolio (သို့) watchlist နှင့် ချိတ်ဆက်ထားလျှင်

ဤအကွက်များကို client တစ်ဦးချင်းစီ၏ portfolio (သို့) watchlist အတွက် ပြန်ပေးခြင်းသည် ဒေတာကို ထို engagement နှင့် ချိတ်ဆက်စေသည် — ၎င်းကို Tier 2/3 အဖြစ် သတ်မှတ်ပြီး SAM နှင့် ခွင့်ပြုထားသော workflow များအတွင်း ထိန်းသိမ်းပါ၊ ဒေတာလုံခြုံရေး အတိုင်း။
