Citation Deduplication & Tooltips

The engineering behind SAM's current citation behavior — fingerprint-based deduplication, per-message renumbering, real-time streaming links, and source tooltips, plus a chart-rendering memory leak fix along the way.

Backend

  • Fingerprint-based deduplication: the same source no longer appears twice, even across tools returning clashing source IDs.
  • Per-message citation mapping: numbering restarts from 1 in each response instead of climbing indefinitely.
  • Recursive payload extraction: handles nested tool-response structures so citations are found wherever they're nested.
  • Structured metrics: tracks available_total, used_total, used_unique, and invalid_count for observability.

Frontend

  • Citation superscripts become clickable as a response streams in, not only after it finishes.
  • Hovering a citation number shows the source title and domain via a tooltip before you click through.
  • Chart rendering was consolidated into shared utilities, fixing memory leaks from chart instances not being destroyed before re-render.

See Citation Management in SAM for the user-facing summary of this behavior.

Reliability

Extensive tests verify fingerprint deduplication, renumbering, metadata generation, real-time link processing, and chart rendering stability. Fully backward compatible with existing chat functionality.

Data Handling Still Applies

Better citation display is a convenience layer, not a change in sensitivity — see Data Security for what still needs verification before reaching a deliverable.

การลบ Citation ซ้ำและ Tooltip

เบื้องหลังทางวิศวกรรมของพฤติกรรม citation ปัจจุบันใน SAM — การลบข้อมูลซ้ำด้วย fingerprint การเรียงลำดับเลขใหม่ต่อข้อความ ลิงก์แบบ real-time ระหว่าง streaming และ tooltip แสดงแหล่งที่มา พร้อมทั้งแก้ปัญหา memory leak ในการเรนเดอร์กราฟไปในตัว

Backend

  • การลบข้อมูลซ้ำด้วย fingerprint: แหล่งข้อมูลเดียวกันจะไม่ปรากฏซ้ำสองครั้งอีกต่อไป แม้ในกรณีที่หลายเครื่องมือส่ง source ID ที่ชนกันมา
  • การจับคู่ citation ต่อข้อความ: การเรียงเลขจะเริ่มนับใหม่จาก 1 ในแต่ละคำตอบ แทนที่จะไล่เพิ่มขึ้นเรื่อย ๆ ไม่มีที่สิ้นสุด
  • การดึงข้อมูลแบบ recursive: รองรับโครงสร้าง tool-response ที่ซ้อนกันหลายชั้น เพื่อให้พบ citation ได้ไม่ว่าจะซ่อนอยู่ลึกแค่ไหน
  • ตัวชี้วัดแบบมีโครงสร้าง: ติดตามค่า available_total, used_total, used_unique และ invalid_count เพื่อการตรวจสอบระบบ

Frontend

  • ตัวเลข superscript ของ citation สามารถคลิกได้ทันทีขณะที่คำตอบกำลัง stream เข้ามา ไม่ต้องรอให้จบก่อน
  • เมื่อวางเมาส์บนเลข citation จะแสดงชื่อแหล่งที่มาและโดเมนผ่าน tooltip ก่อนที่คุณจะคลิกเข้าไปดู
  • การเรนเดอร์กราฟถูกรวมเข้าเป็น utility ที่ใช้ร่วมกัน ซึ่งแก้ปัญหา memory leak ที่เกิดจาก chart instance ไม่ถูกทำลายก่อนการ re-render

ดู การจัดการ Citation ใน SAM สำหรับสรุปพฤติกรรมนี้ในมุมมองผู้ใช้งาน

ความน่าเชื่อถือ

มีการทดสอบอย่างละเอียดเพื่อยืนยันการลบข้อมูลซ้ำด้วย fingerprint การเรียงเลขใหม่ การสร้าง metadata การประมวลผลลิงก์แบบ real-time และความเสถียรของการเรนเดอร์กราฟ ใช้งานร่วมกับฟังก์ชันแชทเดิมได้อย่างสมบูรณ์ (backward compatible)

กฎการจัดการข้อมูลยังคงบังคับใช้เหมือนเดิม

การแสดงผล citation ที่ดีขึ้นเป็นเพียงชั้นความสะดวกสบาย ไม่ใช่การเปลี่ยนแปลงระดับความอ่อนไหวของข้อมูล — ดู Data Security สำหรับสิ่งที่ยังต้องตรวจสอบก่อนนำไปใช้ใน deliverable

Citation ထပ်နေမှု ဖယ်ရှားခြင်းနှင့် Tooltip

SAM ၏ လက်ရှိ citation အပြုအမူနောက်ကွယ်ရှိ အင်ဂျင်နီယာအလုပ် — fingerprint အခြေခံ ထပ်နေမှုဖယ်ရှားခြင်း၊ message တစ်ခုချင်းစီအလိုက် နံပါတ်ပြန်သတ်မှတ်ခြင်း၊ real-time streaming link များ၊ source tooltip များ ထည့်သွင်းခြင်းအပြင် chart-rendering memory leak ပြဿနာကိုပါ ဖြေရှင်းပေးထားသည်။

Backend

  • Fingerprint အခြေခံ ထပ်နေမှုဖယ်ရှားခြင်း: tool များစွာမှ ပဋိပက္ခဖြစ်နေသော source ID များ ပြန်ပို့လာသည့်တိုင် တူညီသော source တစ်ခုသည် နှစ်ကြိမ် ထပ်ကာ ပေါ်တော့မည် မဟုတ်ပါ
  • Message တစ်ခုချင်းစီအလိုက် citation mapping: နံပါတ်စဉ်သည် response တစ်ခုစီတွင် ၁ မှ ပြန်စတင်သည် — အဆုံးမရှိ တိုးများလာခြင်း မရှိတော့ပါ
  • Recursive payload ထုတ်ယူခြင်း: tool-response ဖွဲ့စည်းပုံ အထပ်ထပ်ကို ကိုင်တွယ်နိုင်သဖြင့် citation များကို မည်သည့်အနက်ရှိနေစေကာမူ ရှာတွေ့နိုင်သည်
  • ဖွဲ့စည်းထားသော metric များ: available_total, used_total, used_unique နှင့် invalid_count ကို စောင့်ကြည့်ခြင်းရည်ရွယ်ချက်ဖြင့် ခြေရာခံသည်

Frontend

  • Citation superscript နံပါတ်များကို response stream ဝင်နေချိန်တွင်ပင် နှိပ်နိုင်တော့သည် — ပြီးဆုံးမှ စောင့်ရန် မလိုတော့ပါ
  • Citation နံပါတ်ပေါ်တွင် mouse ကို ထားလိုက်ပါက နှိပ်မကြည့်မီ tooltip မှတစ်ဆင့် source ခေါင်းစဉ်နှင့် domain ကို ပြသသည်
  • Chart rendering ကို shared utility များအဖြစ် စုစည်းလိုက်သဖြင့် re-render မလုပ်မီ chart instance များ မဖျက်ဘဲ ကျန်နေခြင်းကြောင့် ဖြစ်ပေါ်သော memory leak ပြဿနာများကို ဖြေရှင်းပြီးဖြစ်သည်

ဤအပြုအမူ၏ အသုံးပြုသူဘက်မှ အကျဉ်းချုပ်ကို SAM တွင် Citation စီမံခန့်ခွဲမှု တွင် ကြည့်ပါ

ယုံကြည်စိတ်ချရမှု

Fingerprint ထပ်နေမှုဖယ်ရှားခြင်း၊ နံပါတ်ပြန်သတ်မှတ်ခြင်း၊ metadata ထုတ်လုပ်ခြင်း၊ real-time link ကိုင်တွယ်ခြင်းနှင့် chart rendering တည်ငြိမ်မှုကို စစ်ဆေးရန် ကျယ်ကျယ်ပြန့်ပြန့် test များ ပြုလုပ်ထားသည်။ လက်ရှိ chat လုပ်ဆောင်ချက်များနှင့် အပြည့်အဝ backward compatible ဖြစ်သည်

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေသည်

Citation ကို ပိုမိုကောင်းမွန်စွာ ပြသခြင်းသည် အဆင်ပြေမှုအလွှာတစ်ခုသာဖြစ်ပြီး၊ ဒေတာ၏ အရေးကြီးမှုအဆင့်ကို ပြောင်းလဲခြင်း မဟုတ်ပါ — deliverable တစ်ခုသို့ မရောက်မီ မည်သည်ကို ဆက်လက်စစ်ဆေးရမည်ကို Data Security တွင် ကြည့်ပါ
