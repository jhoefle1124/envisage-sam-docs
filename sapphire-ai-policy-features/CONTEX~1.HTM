New in SAM · Feature Guide

Context Management Enhancements

SAM's chatbot now tracks how much of its context window a conversation is using, trims and summarizes history before it overflows, and shows you a real-time budget indicator — so long research conversations degrade gracefully instead of failing outright.

What's New

  • Context window management: a new budget tracking system monitors conversation size and manages it before limits are hit.
  • History summarization: long conversations are summarized in batches rather than simply cut off.
  • Model and configuration updates: default chat model upgraded, plus a new reasoning-effort setting.
  • Chart generation improvements: better validation and clearer tool-calling behavior for chart requests.
  • UI updates: a real-time context budget indicator and a clearer "compacting" state while history is summarized.

Context Window Management

Every chat has a finite context window — the amount of conversation and document content a model can hold at once. This update adds active management of that budget instead of letting a long conversation simply fail when it runs out:

Real-time budget indicator

The chat UI now shows how much of the context window a conversation is using, so you can see it coming rather than being surprised by a failure.

Dynamic trimming & prioritization

Message history and injected documents are trimmed and prioritized intelligently as a conversation approaches its budget, rather than dropped arbitrarily.

Batched history summarization

Once a conversation crosses a length threshold, older messages are summarized in batches — condensed, not deleted — and a "compacting" indicator shows this is happening.

Graceful overflow handling

If a conversation still exceeds the limit, you now get a clear, actionable message instead of a silent or confusing failure.

Chart Generation Improvements

Chart requests now go through explicit validation before generation — better handling of data series and categorical axes, and clearer instructions guiding the model's tool calls, meaning fewer malformed or unexpected charts.

Breaking Change: Company Financials Batch Limit

Reduced from 5 to 3 companies per request. This affects tool-calling behavior in the chatbot, any external integrations using this endpoint, and the performance characteristics of financial data queries. If you have workflows that assumed a 5-company batch, they now need to request in smaller groups.

This is separate from the scorecard batch increase (3 → 50 companies) covered on the Scorecard Rankings page — that change applies to scorecard retrieval specifically, while this change applies to the company financials tool. The two moved in opposite directions for different reasons; check which tool you're calling before assuming a batch size.

Reliability

Extensively tested. The update ships with 1,482 lines of context budget tests, 358 lines of context error-handling tests, and 219 lines of history summarizer tests, alongside enhanced validation coverage for the updated tools. A database migration is included; no manual data migration is required.

Data Handling Still Applies

Automatic summarization and trimming manage the size of a conversation, not its sensitivity. A summarized version of a conversation can still contain Tier 2 or Tier 3 content if the original did.

Summarization doesn't declassify. Don't rely on history compaction to "clean up" a conversation that drifted into sensitive territory. Anonymize before you type, the same as always — see Data Security. If Tier 3 content already reached the wrong tool, that's a same-day report to your project lead regardless of what the chat UI does with it afterward.
