Citation Management in SAM

SAM's citations are now deduplicated, renumbered per response, and clickable as the answer streams in — with tooltips showing you the source title and domain before you click. Citations are a starting point for verification, not a substitute for it.

What's New

  • Fingerprint-based deduplication: the same source no longer appears as multiple citations, even when different tools return overlapping results.
  • Per-message renumbering: citation numbers restart from 1 in each response instead of climbing indefinitely across a long conversation.
  • Context-aware citation metadata: citation titles now include the company name and date range behind a data pull, not just a generic source label.
  • Real-time citation links: citation superscripts become clickable as the response streams, not only after it finishes.
  • Source tooltips: hovering a citation number shows the source title and domain before you click through.
  • Chart rendering stability: consolidated chart logic fixes memory leaks and rendering glitches that could occur alongside heavy citation use.

How It Works

FeatureWhat It Does
Fingerprint deduplicationPrevents duplicate source citations, even across tools that return clashing source IDs
Per-message citation mappingEnsures numbering is accurate and restarts cleanly for each new response
Context-aware metadataAdds company names, date ranges, and result summaries to citation titles for tools like company financials, price metrics, and scorecards
Real-time linkingCitation superscripts are processed and made clickable as the AI response streams in
State managementCitation state clears between user messages, and stray "Sources" sections some models append are stripped automatically

Why It Matters

Clearer, more accurate citations make it faster to trace an answer back to where it came from — which is exactly what Cross-Verification asks you to do for anything going into a deliverable. A citation pointing at the right, deduplicated source with a descriptive title is a better starting point for that check than a generic or duplicated one.

A citation is a pointer, not proof. Better citation display doesn't reduce the need to verify a claim before it reaches a client — it just makes finding the source to verify against faster. Treat citation improvements as a convenience layer on top of the verification discipline already in place, not a replacement for it.

Reliability

Backward compatible, extensively tested. Structured metrics (available_total, used_total, used_unique, invalid_count, etc.) and detailed logging support ongoing validation of citation accuracy. Test coverage spans deduplication, renumbering, metadata generation, real-time link processing, and chart rendering stability.

Data Handling Still Applies

Citations surface the same underlying tool data covered elsewhere on this site — a citation to a company financials or scorecard result carries whatever tier that data carries. Seeing a source cited in SAM doesn't change its sensitivity.

Usually fine

Public source citations

Citations pointing to public news, filings, or public financial data stay close to Tier 1.

Check first

Citations tied to client data

If a citation traces back to a client-specific portfolio, watchlist, or engagement context, treat the underlying source the same as any other Tier 2/3 material, per Data Security.

การจัดการการอ้างอิงใน SAM

การอ้างอิง (citations) ของ SAM ตอนนี้ไม่ซ้ำกันแล้ว มีการเรียงหมายเลขใหม่ในแต่ละคำตอบ และสามารถคลิกได้ทันทีขณะที่คำตอบกำลัง stream เข้ามา — พร้อม tooltip ที่แสดงชื่อแหล่งที่มาและโดเมนก่อนที่คุณจะคลิก การอ้างอิงเป็นเพียงจุดเริ่มต้นสำหรับการตรวจสอบ ไม่ใช่สิ่งทดแทนการตรวจสอบ

มีอะไรใหม่

  • การลบข้อมูลซ้ำด้วย Fingerprint: แหล่งที่มาเดียวกันจะไม่ปรากฏเป็นการอ้างอิงหลายรายการอีกต่อไป แม้ว่าเครื่องมือต่างกันจะส่งคืนผลลัพธ์ที่ทับซ้อนกันก็ตาม
  • การเรียงหมายเลขใหม่ต่อข้อความ: หมายเลขการอ้างอิงจะเริ่มนับใหม่จาก 1 ในแต่ละคำตอบ แทนที่จะไล่เพิ่มขึ้นเรื่อย ๆ ตลอดบทสนทนาที่ยาว
  • ข้อมูลเมทาดาทาการอ้างอิงที่รับรู้บริบท: ชื่อการอ้างอิงตอนนี้รวมชื่อบริษัทและช่วงวันที่ของการดึงข้อมูลด้วย ไม่ใช่แค่ป้ายชื่อแหล่งที่มาทั่วไป
  • ลิงก์การอ้างอิงแบบเรียลไทม์: ตัวเลขยกกำลังของการอ้างอิงจะกลายเป็นลิงก์ที่คลิกได้ทันทีขณะคำตอบกำลัง stream ไม่ใช่หลังจาก stream เสร็จแล้วเท่านั้น
  • Tooltip แหล่งที่มา: การเลื่อนเมาส์ไปวางบนหมายเลขการอ้างอิงจะแสดงชื่อแหล่งที่มาและโดเมนก่อนที่คุณจะคลิกเข้าไป
  • ความเสถียรในการแสดงผลกราฟ: การรวมตรรกะการแสดงผลกราฟเข้าด้วยกันช่วยแก้ปัญหา memory leak และความผิดพลาดในการแสดงผลที่อาจเกิดขึ้นเมื่อมีการใช้การอ้างอิงจำนวนมาก

วิธีการทำงาน

ฟีเจอร์การทำงาน
Fingerprint deduplicationป้องกันการอ้างอิงแหล่งที่มาซ้ำ แม้ในเครื่องมือที่ส่งคืน source ID ที่ขัดแย้งกัน
Per-message citation mappingรับประกันว่าการเรียงหมายเลขถูกต้องและเริ่มต้นใหม่อย่างชัดเจนสำหรับคำตอบใหม่แต่ละครั้ง
Context-aware metadataเพิ่มชื่อบริษัท ช่วงวันที่ และสรุปผลลัพธ์ลงในชื่อการอ้างอิงสำหรับเครื่องมือ เช่น company financials, price metrics และ scorecards
Real-time linkingตัวเลขยกกำลังของการอ้างอิงจะถูกประมวลผลและทำให้คลิกได้ขณะที่คำตอบของ AI กำลัง stream เข้ามา
State managementState ของการอ้างอิงจะถูกล้างระหว่างข้อความของผู้ใช้แต่ละครั้ง และส่วน "Sources" ที่โมเดลบางตัวเพิ่มเข้ามาเองจะถูกลบออกโดยอัตโนมัติ

เหตุใดจึงสำคัญ

การอ้างอิงที่ชัดเจนและแม่นยำขึ้นช่วยให้ย้อนรอยคำตอบกลับไปยังที่มาได้รวดเร็วขึ้น — ซึ่งตรงกับสิ่งที่ Cross-Verification กำหนดให้คุณทำสำหรับทุกสิ่งที่จะนำไปใช้ใน deliverable การอ้างอิงที่ชี้ไปยังแหล่งที่มาที่ถูกต้อง ไม่ซ้ำกัน และมีชื่อที่บรรยายชัดเจน เป็นจุดเริ่มต้นที่ดีกว่าสำหรับการตรวจสอบนั้น เมื่อเทียบกับการอ้างอิงทั่วไปหรือที่ซ้ำกัน

การอ้างอิงเป็นตัวชี้ ไม่ใช่หลักฐาน การแสดงผลการอ้างอิงที่ดีขึ้นไม่ได้ลดความจำเป็นในการตรวจสอบข้อความก่อนที่จะถึงมือลูกค้า — มันเพียงทำให้การค้นหาแหล่งที่มาเพื่อตรวจสอบทำได้เร็วขึ้นเท่านั้น ให้มองว่าการปรับปรุงการอ้างอิงเป็นชั้นความสะดวกสบายที่เพิ่มเข้ามาบนวินัยการตรวจสอบที่มีอยู่แล้ว ไม่ใช่สิ่งทดแทนวินัยนั้น

ความน่าเชื่อถือ

เข้ากันได้กับระบบเดิม (Backward compatible) และผ่านการทดสอบอย่างละเอียด ตัวชี้วัดเชิงโครงสร้าง (available_total, used_total, used_unique, invalid_count เป็นต้น) และการบันทึก log อย่างละเอียด สนับสนุนการตรวจสอบความถูกต้องของการอ้างอิงอย่างต่อเนื่อง การทดสอบครอบคลุมการลบข้อมูลซ้ำ การเรียงหมายเลขใหม่ การสร้างเมทาดาทา การประมวลผลลิงก์แบบเรียลไทม์ และความเสถียรในการแสดงผลกราฟ

หลักการจัดการข้อมูลยังคงบังคับใช้

การอ้างอิงแสดงข้อมูลเครื่องมือชุดเดียวกันกับที่ครอบคลุมอยู่ในที่อื่นของเว็บไซต์นี้ — การอ้างอิงไปยังผลลัพธ์ company financials หรือ scorecard จะมี tier เดียวกันกับที่ข้อมูลนั้นมี การเห็นแหล่งที่มาถูกอ้างอิงใน SAM ไม่ได้เปลี่ยนระดับความอ่อนไหวของมัน

ปกติแล้วไม่มีปัญหา

การอ้างอิงจากแหล่งสาธารณะ

การอ้างอิงที่ชี้ไปยังข่าวสาธารณะ การยื่นเอกสาร (filings) หรือข้อมูลการเงินสาธารณะ ยังคงใกล้เคียงกับ Tier 1

ตรวจสอบก่อน

การอ้างอิงที่เกี่ยวข้องกับข้อมูลลูกค้า

หากการอ้างอิงย้อนกลับไปยัง portfolio เฉพาะของลูกค้า watchlist หรือบริบทของงาน (engagement) ให้ปฏิบัติต่อแหล่งที่มานั้นเช่นเดียวกับข้อมูล Tier 2/3 อื่น ๆ ตาม Data Security

SAM တွင် Citation စီမံခန့်ခွဲမှု

SAM ၏ citation များသည် ယခုအခါ ထပ်နေမှု ဖယ်ရှားပြီး (deduplicated)၊ တုံ့ပြန်မှုတစ်ခုစီအလိုက် နံပါတ်အသစ်ပြန်ချထားပြီး၊ အဖြေ stream ဖြစ်နေစဉ်ပင် နှိပ်နိုင်သည် — နှိပ်မီ ရင်းမြစ်ခေါင်းစဉ်နှင့် domain ကို ပြသည့် tooltip များနှင့်တကွ။ Citation သည် စစ်ဆေးမှုအတွက် အစပြုနေရာသာဖြစ်ပြီး စစ်ဆေးမှု၏ အစားထိုး မဟုတ်ပါ။

အသစ်ပါဝင်သည်များ

  • Fingerprint အခြေပြု ထပ်နေမှုဖယ်ရှားခြင်း: ကွဲပြားသော ကိရိယာများက ထပ်နေသော ရလဒ်များ ပြန်ပေးလင့်ကစား ရင်းမြစ်တူတစ်ခုသည် citation များစွာအဖြစ် နောက်တစ်ဖန် မပေါ်တော့ပါ။
  • Message တစ်ခုစီအလိုက် နံပါတ်ပြန်ချခြင်း: citation နံပါတ်များသည် စကားပြောရှည်တစ်ခုလုံးတွင် အဆက်မပြတ် တိုးလာမည့်အစား တုံ့ပြန်မှုတစ်ခုစီတွင် ၁ မှ ပြန်စတင်သည်။
  • Context သိရှိသော citation metadata: citation ခေါင်းစဉ်များတွင် ယခုအခါ generic ရင်းမြစ်ခေါင်းစဉ်တစ်ခုတည်းမက ဒေတာ pull တစ်ခု၏ နောက်ကွယ်ရှိ ကုမ္ပဏီအမည်နှင့် ရက်စွဲအပိုင်းအခြားကို ပါဝင်စေသည်။
  • Real-time citation link များ: citation superscript များသည် တုံ့ပြန်မှု ပြီးဆုံးမှသာမက stream ဖြစ်နေစဉ်ပင် နှိပ်နိုင်သောလင့်ခ်များ ဖြစ်လာသည်။
  • ရင်းမြစ် Tooltip များ: citation နံပါတ်တစ်ခုပေါ် mouse ရွှေ့ထားခြင်းဖြင့် နှိပ်မီ ရင်းမြစ်ခေါင်းစဉ်နှင့် domain ကို ပြသည်။
  • Chart ပြသမှု တည်ငြိမ်မှု: chart logic ကို ပေါင်းစည်းလိုက်ခြင်းဖြင့် citation အသုံးများသောအခါ ဖြစ်တတ်သည့် memory leak နှင့် ပြသမှုချို့ယွင်းချက်များကို ပြင်ဆင်ပေးသည်။

အလုပ်လုပ်ပုံ

ဝန်ဆောင်မှုဘာလုပ်ဆောင်သနည်း
Fingerprint deduplicationကွဲပြားသော source ID ပြန်ပေးသည့် ကိရိယာများကြားတွင်ပင် ရင်းမြစ် citation ထပ်နေမှုကို ကာကွယ်ပေးသည်
Per-message citation mappingတုံ့ပြန်မှုအသစ်တစ်ခုစီအတွက် နံပါတ်ချထားမှု မှန်ကန်ပြီး ရှင်းလင်းစွာ ပြန်စတင်ကြောင်း သေချာစေသည်
Context-aware metadatacompany financials, price metrics နှင့် scorecards ကဲ့သို့သော ကိရိယာများအတွက် ကုမ္ပဏီအမည်၊ ရက်စွဲအပိုင်းအခြားနှင့် ရလဒ်အကျဉ်းချုပ်များကို citation ခေါင်းစဉ်များတွင် ထည့်သွင်းသည်
Real-time linkingAI ၏ တုံ့ပြန်မှု stream ဖြစ်နေစဉ် citation superscript များကို ပြုပြင်ပြီး နှိပ်နိုင်စေသည်
State managementအသုံးပြုသူ message များကြား citation state ကို ရှင်းလင်းပေးပြီး၊ အချို့ model များ ထပ်ဆောင်း ပေါင်းထည့်လေ့ရှိသော "Sources" အပိုင်းများကို အလိုအလျောက် ဖယ်ရှားပေးသည်

အရေးကြီးရသည့်အကြောင်း

ပိုမိုရှင်းလင်းပြီး တိကျသော citation များသည် အဖြေတစ်ခုကို မူလရင်းမြစ်အထိ ပြန်လိုက်ရှာဖွေရာတွင် ပိုမြန်စေသည် — ၎င်းသည် deliverable တစ်ခုထဲ ထည့်မည့် မည်သည့်အရာမဆိုအတွက် Cross-Verification က လုပ်ခိုင်းသည့်အရာ အတိအကျဖြစ်သည်။ မှန်ကန်ပြီး ထပ်နေမှုမရှိသော ရင်းမြစ်တစ်ခုကို ညွှန်ပြသော၊ ဖော်ပြချက်ပါသည့် ခေါင်းစဉ်နှင့် citation တစ်ခုသည် generic သို့မဟုတ် ထပ်နေသော citation တစ်ခုထက် ထိုစစ်ဆေးမှုအတွက် ပိုကောင်းသော အစပြုနေရာ ဖြစ်သည်။

Citation တစ်ခုသည် ညွှန်ပြချက်တစ်ခုသာဖြစ်ပြီး သက်သေမဟုတ်ပါ။ Citation ပြသမှု ပိုကောင်းလာခြင်းသည် client ထံ မရောက်မီ claim တစ်ခုကို စစ်ဆေးရန် လိုအပ်ချက်ကို မလျှော့ချပါ — ၎င်းသည် စစ်ဆေးရန် ရင်းမြစ်ကို ရှာဖွေရာတွင် ပိုမြန်စေသည်သာ ဖြစ်သည်။ Citation တိုးတက်မှုများကို ရှိပြီးသား verification စည်းကမ်းအပေါ် ထပ်ထည့်သည့် အဆင့်တစ်ခုအဖြစ်သာ သတ်မှတ်ပါ၊ ၎င်း၏ အစားထိုးအဖြစ် မဟုတ်ပါ။

ယုံကြည်စိတ်ချရမှု

နောက်ကြောင်းပြန် လိုက်လျောညီထွေရှိပြီး (Backward compatible)၊ ကျယ်ကျယ်ပြန့်ပြန့် စမ်းသပ်ထားသည်။ ဖွဲ့စည်းထားသော metric များ (available_total, used_total, used_unique, invalid_count စသည်) နှင့် အသေးစိတ် log မှတ်တမ်းများသည် citation တိကျမှုကို စဉ်ဆက်မပြတ် အတည်ပြုနိုင်ရန် ကူညီပေးသည်။ စမ်းသပ်မှု အကျုံးဝင်မှုသည် ထပ်နေမှုဖယ်ရှားခြင်း၊ နံပါတ်ပြန်ချခြင်း၊ metadata ထုတ်လုပ်ခြင်း၊ real-time link ပြုပြင်ခြင်း နှင့် chart ပြသမှု တည်ငြိမ်မှုအထိ ကျယ်ပြန့်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် သက်ရောက်နေဆဲ

Citation များသည် ဤဝဘ်ဆိုက်၏ အခြားနေရာများတွင် ဖော်ပြထားသည့် underlying tool ဒေတာအတူတူကိုပင် ပြသသည် — company financials သို့မဟုတ် scorecard ရလဒ်တစ်ခုသို့ ညွှန်ပြသော citation တစ်ခုသည် ထိုဒေတာပါရှိသော tier မည်သည့်အဆင့်ဖြစ်စေ ထို tier ကိုပင် ဆောင်ရွက်သည်။ SAM တွင် citation ပြထားသည့် ရင်းမြစ်တစ်ခုကို မြင်ရခြင်းသည် ၎င်း၏ အာရုံခံနိုင်စွမ်း (sensitivity) ကို မပြောင်းလဲစေပါ။

ပုံမှန်အားဖြင့် ပြဿနာမရှိ

အများပြည်သူ ရင်းမြစ် Citation များ

အများပြည်သူ သတင်း၊ filing များ သို့မဟုတ် အများပြည်သူ ဘဏ္ဍာရေးဒေတာကို ညွှန်ပြသော citation များသည် Tier 1 နှင့် နီးစပ်စွာ ကျန်ရှိသည်

ဦးစွာ စစ်ဆေးပါ

ဖောက်သည်ဒေတာနှင့် ဆက်စပ်သော Citation များ

citation တစ်ခုသည် ဖောက်သည်အထူး portfolio၊ watchlist သို့မဟုတ် engagement context တစ်ခုသို့ ပြန်ဆက်နေပါက၊ underlying ရင်းမြစ်ကို Data Security အရ အခြား Tier 2/3 material များကဲ့သို့ပင် သတ်မှတ်ပါ
