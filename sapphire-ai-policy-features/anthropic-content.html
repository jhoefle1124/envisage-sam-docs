New in SAM · Feature Guide

Improving Anthropic Content and Complexity

SAM now extracts Claude's streaming responses more robustly — fewer stray fragments in chat, cleaner handling of "thinking" content, and a single, well-tested code path instead of scattered logic. You shouldn't have to think about any of this; that's the point.

What's New

  • Robust content extraction: new shared utility functions handle every shape Claude's streaming API can return — plain text, delta updates, lists of content blocks, and more.
  • Tool block detection: tool-related content (tool_use, tool_result, input_json_delta, etc.) is now properly identified and filtered out of what you see, instead of occasionally leaking through.
  • Cleaner "thinking" tag removal: an optimized pattern strips both raw and HTML-encoded thinking tags in a single pass.
  • Simplified streaming code: the event-processing logic that turns Claude's response stream into chat messages was refactored to use one centralized extraction path.

Why It Matters

This is a behind-the-scenes reliability change — you won't see a new button or setting. What you should notice is fewer rendering glitches in chat: no stray tool-call fragments or leftover "thinking" markup showing up where they shouldn't, and more consistent behavior regardless of how Claude structures a given streamed response.

What Changed Under the Hood

ComponentWhat It Does Now
extract_message_content()Single orchestrating function that handles all content extraction strategies
Delta & list handlingDedicated helpers extract text from delta updates and from lists of mixed content blocks
Tool block filteringTool-related blocks are identified and excluded from the visible message content
Streaming event processingRefactored to call the shared extraction function instead of duplicating logic in multiple places

Reliability

No breaking changes. This is a refactor with added robustness, not a behavior change you need to adjust to. It ships with 46 new tests (27 covering the LLM factory, 19 covering streaming event processing) alongside the full existing test suite, which continues to pass.

Data Handling Still Applies

This change is about how SAM renders Claude's responses more reliably — it doesn't change what data flows where. The same Data Security rules on what you type into SAM chat still apply exactly as before.

ใหม่ใน SAM · คู่มือฟีเจอร์

การปรับปรุงเนื้อหาและความซับซ้อนของ Anthropic

ขณะนี้ SAM ดึงเนื้อหาจาก streaming response ของ Claude ได้แม่นยำและเสถียรขึ้น — เศษข้อความที่หลุดเข้ามาในแชทน้อยลง จัดการเนื้อหา "thinking" ได้สะอาดขึ้น และใช้ code path เดียวที่ผ่านการทดสอบอย่างละเอียด แทนที่ logic ที่กระจัดกระจายหลายจุด คุณไม่จำเป็นต้องสนใจรายละเอียดเหล่านี้เลย — นั่นคือเป้าหมายของการเปลี่ยนแปลงนี้

มีอะไรใหม่

  • ดึงเนื้อหาได้แม่นยำขึ้น: ฟังก์ชัน utility ที่ใช้ร่วมกันตัวใหม่รองรับทุกรูปแบบที่ streaming API ของ Claude ส่งกลับมาได้ — ข้อความธรรมดา, delta update, รายการ content block และอื่น ๆ
  • ตรวจจับ tool block: เนื้อหาที่เกี่ยวข้องกับ tool (tool_use, tool_result, input_json_delta ฯลฯ) ถูกระบุและกรองออกจากสิ่งที่คุณเห็นอย่างถูกต้องแล้ว แทนที่จะหลุดเข้ามาเป็นครั้งคราวเหมือนก่อน
  • ลบแท็ก "thinking" ได้สะอาดขึ้น: รูปแบบที่ปรับให้เหมาะสมแล้วสามารถลบแท็ก thinking ทั้งแบบดิบและแบบเข้ารหัส HTML ได้ในรอบเดียว
  • โค้ด streaming ที่เรียบง่ายขึ้น: logic การประมวลผล event ที่แปลง response stream ของ Claude ให้เป็นข้อความแชท ถูก refactor ให้ใช้ extraction path ศูนย์กลางเพียงจุดเดียว

เหตุใดจึงสำคัญ

นี่คือการเปลี่ยนแปลงด้านความน่าเชื่อถือเบื้องหลัง — คุณจะไม่เห็นปุ่มหรือการตั้งค่าใหม่ สิ่งที่ควรสังเกตได้คือความผิดพลาดในการแสดงผลแชทลดลง ไม่มีเศษ tool-call หรือ markup "thinking" ที่หลงเหลือปรากฏในที่ที่ไม่ควรอยู่อีกต่อไป และพฤติกรรมของระบบสม่ำเสมอมากขึ้น ไม่ว่า Claude จะจัดโครงสร้าง streamed response นั้นอย่างไร

สิ่งที่เปลี่ยนแปลงเบื้องหลัง

องค์ประกอบการทำงานในปัจจุบัน
extract_message_content()ฟังก์ชันหลักเพียงจุดเดียวที่จัดการกลยุทธ์การดึงเนื้อหาทั้งหมด
การจัดการ Delta และรายการhelper เฉพาะทางดึงข้อความจาก delta update และจากรายการ content block แบบผสม
การกรอง Tool Blockบล็อกที่เกี่ยวข้องกับ tool ถูกระบุและตัดออกจากเนื้อหาข้อความที่มองเห็นได้
การประมวลผล Streaming Eventปรับให้เรียกใช้ฟังก์ชัน extraction ที่ใช้ร่วมกัน แทนการทำ logic ซ้ำซ้อนในหลายจุด

ความน่าเชื่อถือ

ไม่มีการเปลี่ยนแปลงที่กระทบการทำงานเดิม นี่คือการ refactor ที่เพิ่มความแข็งแรงให้ระบบ ไม่ใช่การเปลี่ยนพฤติกรรมที่คุณต้องปรับตัวตาม มาพร้อมกับชุดทดสอบใหม่ 46 รายการ (27 รายการครอบคลุม LLM factory และ 19 รายการครอบคลุมการประมวลผล streaming event) ควบคู่กับชุดทดสอบเดิมทั้งหมดที่ยังคงผ่านตามปกติ

กฎการจัดการข้อมูลยังคงบังคับใช้เหมือนเดิม

การเปลี่ยนแปลงนี้เกี่ยวข้องกับวิธีที่ SAM แสดงผล response ของ Claude ให้น่าเชื่อถือขึ้นเท่านั้น — ไม่ได้เปลี่ยนแปลงว่าข้อมูลไหลไปที่ใด กฎ Data Security เดิมเกี่ยวกับสิ่งที่คุณพิมพ์ลงในแชทของ SAM ยังคงบังคับใช้เหมือนเดิมทุกประการ

SAM တွင် အသစ် · ဝန်ဆောင်မှုလမ်းညွှန်

Anthropic ၏ အကြောင်းအရာနှင့် ရှုပ်ထွေးမှုကို မြှင့်တင်ခြင်း

SAM သည် ယခုအခါ Claude ၏ streaming response များမှ အကြောင်းအရာကို ပိုမိုခိုင်မာစွာ ထုတ်ယူနိုင်ပြီ — chat တွင် ကျန်ကျန်နေသော အပိုင်းအစများ လျော့နည်းလာသည်၊ "thinking" content ကို ပိုမိုသန့်ရှင်းစွာ ကိုင်တွယ်နိုင်သည်၊ ပြီးတော့ ပျံ့နှံ့နေသော logic များအစား စမ်းသပ်မှုကောင်းစွာ ပြီးစီးထားသော code path တစ်ခုတည်းသာ ရှိတော့သည်။ ဤအသေးစိတ်များကို သင် စဉ်းစားနေစရာ မလိုပါ — ဒါက ရည်ရွယ်ချက်ပါ။

အသစ်ပါဝင်သည့်အရာများ

  • ခိုင်မာသော content extraction: shared utility function အသစ်များသည် Claude ၏ streaming API ပြန်ပို့နိုင်သည့် ပုံစံအားလုံးကို ကိုင်တွယ်နိုင်သည် — plain text, delta update, content block စာရင်းများနှင့် အခြားများ
  • Tool block ရှာဖွေခြင်း: tool နှင့်ဆက်စပ်သော content (tool_use, tool_result, input_json_delta စသည်) ကို ယခုအခါ မှန်ကန်စွာ ခွဲခြားသတ်မှတ်ပြီး သင့်မြင်ရမည့်အရာမှ ဖယ်ထုတ်ပေးနေပြီ — ယခင်က တခါတရံ ပေါက်ကြားဝင်ရောက်လာသကဲ့သို့ မဟုတ်တော့ပါ
  • သန့်ရှင်းစွာ "thinking" tag ဖယ်ရှားခြင်း: အကောင်းဆုံးပြင်ဆင်ထားသော pattern သည် raw နှင့် HTML-encoded thinking tag နှစ်မျိုးလုံးကို တစ်ကြိမ်တည်းဖြင့် ဖယ်ရှားနိုင်သည်
  • ရိုးရှင်းသွားသော streaming code: Claude ၏ response stream ကို chat message များအဖြစ် ပြောင်းလဲပေးသည့် event-processing logic ကို ဗဟိုချုပ်ကိုင်သော extraction path တစ်ခုတည်းသုံးရန် ပြန်လည်ဖွဲ့စည်းလိုက်သည်

ဘာကြောင့် အရေးကြီးသလဲ

ဤသည်မှာ နောက်ကွယ်မှ ယုံကြည်စိတ်ချရမှု ပြောင်းလဲမှုတစ်ခုဖြစ်သည် — ခလုတ် သို့မဟုတ် setting အသစ်တစ်ခုကို သင်တွေ့မြင်ရမည် မဟုတ်ပါ။ သတိထားမိသင့်သည်မှာ chat တွင် rendering ချို့ယွင်းချက်များ လျော့နည်းလာခြင်းဖြစ်သည် — မှားယွင်းနေရာတွင် tool-call အပိုင်းအစများ သို့မဟုတ် ကျန်ရစ်နေသော "thinking" markup များ ပေါ်လာတော့မည် မဟုတ်ဘဲ Claude သည် streamed response ကို မည်သို့ပင် ဖွဲ့စည်းစေကာမူ ပိုမို တသမတ်တည်း ရှိသော အပြုအမူကို ရရှိမည်ဖြစ်သည်

နောက်ကွယ်တွင် ပြောင်းလဲသွားသည့်အရာများ

အစိတ်အပိုင်းယခုလုပ်ဆောင်ချက်
extract_message_content()content extraction ဗျူဟာအားလုံးကို ကိုင်တွယ်သည့် ဗဟိုစီမံသော function တစ်ခုတည်း
Delta နှင့် list ကိုင်တွယ်မှုကျယ်ကျယ်ပြန့်ပြန့် helper များသည် delta update များနှင့် ရောနှောထားသော content block စာရင်းများမှ စာသားကို ထုတ်ယူသည်
Tool Block စစ်ထုတ်ခြင်းtool နှင့်ဆက်စပ်သော block များကို ခွဲခြားသတ်မှတ်ပြီး မြင်ရသော message content မှ ထုတ်ပယ်သည်
Streaming Event ကိုင်တွယ်ခြင်းနေရာများစွာတွင် logic ထပ်ခါထပ်ခါ ရေးမည့်အစား shared extraction function ကို ခေါ်သုံးရန် ပြန်လည်ဖွဲ့စည်းထားသည်

ယုံကြည်စိတ်ချရမှု

ပျက်စီးစေမည့် ပြောင်းလဲမှု မရှိပါ။ ဤသည်မှာ ခိုင်မာမှုတိုးမြှင့်ထားသော refactor တစ်ခုဖြစ်ပြီး၊ သင်လိုက်လျောညီထွေဖြစ်ရန် လိုအပ်သော အပြုအမူပြောင်းလဲမှု မဟုတ်ပါ။ Test အသစ် ၄၆ ခု (LLM factory အတွက် ၂၇ ခု၊ streaming event processing အတွက် ၁၉ ခု) ပါဝင်ပြီး ရှိပြီးသား test suite အားလုံးနှင့်အတူ ဆက်လက်အောင်မြင်နေဆဲဖြစ်သည်

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်နေသည်

ဤပြောင်းလဲမှုသည် SAM မှ Claude ၏ response များကို ပိုမိုယုံကြည်စိတ်ချရအောင် ပြသနည်းနှင့် သက်ဆိုင်ပြီး — ဒေတာ မည်သည့်နေရာသို့ စီးဆင်းသည်ကို ပြောင်းလဲခြင်း မရှိပါ။ SAM chat တွင် သင်ရိုက်ထည့်သည့်အရာများအတွက် Data Security စည်းမျဉ်းများအတိုင်း ယခင်အတိုင်းပင် ဆက်လက်သက်ရောက်နေဆဲဖြစ်သည်
