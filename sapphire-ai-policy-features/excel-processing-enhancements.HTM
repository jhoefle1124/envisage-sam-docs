New in SAM · Feature Guide

Excel Processing Enhancements

Excel uploads now get intelligent row sampling instead of naive truncation, much larger file size limits, and a new GLM-4.5 model option — plus a form-submission bug fix so pressing Enter doesn't interrupt an upload in progress.

Smart Sampling Algorithm

Rather than reading only the first N rows of a large spreadsheet, uploads now sample intelligently:

  • First 100 rows — captures headers and initial data patterns.
  • Last 50 rows — captures the most recent data.
  • Smart middle sampling — time-based distribution for date columns, category distribution for categorical data (up to 50 unique values), or even spacing as a fallback.

Summaries are enriched with statistical analysis, a column overview with data types, date ranges, and categorical value distributions. Chunk size for Excel content doubled from 2,000 to 4,000 characters for better context retention.

File Handling

ChangeBeforeAfter
File size limit (unified)25MB100MB
Async processing threshold5MB2MB (faster UX trigger)

MIME type detection was also improved to correctly handle Excel/DOCX files that get misidentified as application/zip or application/octet-stream by the underlying detection library.

Reliability

Comprehensive test coverage for small/large DataFrames, duplicate removal, empty DataFrame edge cases, date detection, and category-based sampling. Also fixes: Enter-key no longer submits the form while a document is processing, and smart sampling now properly de-duplicates rows when combining its three samples.

Data Handling Still Applies

Larger file limits don't change what's safe to upload. A 100MB spreadsheet of client portfolio data is still Tier 3 — see Data Security before uploading anything beyond generic, public data.
