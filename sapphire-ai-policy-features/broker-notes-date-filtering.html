New in SAM · Feature Guide

Broker Notes MCP Integration

SAM can now search and fetch NTAsset broker research notes through a Pinecone-backed MCP server — gated by team-based access control so only approved teams see the guidance or the tools at all.

What's New

  • Two new tools: broker_notes_search for semantic search and broker_notes_fetch to retrieve full note content, communicating over SSE with the Pinecone MCP server.
  • Content truncation: responses capped at 15,000 characters to prevent context window overflow, with citation formatting including domain extraction.
  • Team-based access control: tool access is configurable per team via settings, filtered at LLM-call time so it stays compatible with LangGraph's compiled tool node architecture.
  • Dynamic system prompting: broker notes guidance only appears to teams that actually have access, with proactive instructions to check broker notes before falling back to web search.

Reliability

41 test cases across team permissions, MCP tool functionality, and system message awareness — all passing with proper isolation and mocking.

Data Handling Still Applies

Broker research notes are third-party content accessed through an approved integration — team access control determines who can query them, but the same Data Security rules apply to anything drawn from them into a deliverable.

การเชื่อมต่อ Broker Notes MCP

ตอนนี้ SAM สามารถค้นหาและดึงข้อมูล NTAsset broker research notes ผ่าน MCP server ที่ใช้ Pinecone ได้แล้ว — โดยควบคุมการเข้าถึงตามทีม เพื่อให้เฉพาะทีมที่ได้รับอนุมัติเท่านั้นที่จะเห็นคำแนะนำหรือแม้แต่เครื่องมือนี้เลย

มีอะไรใหม่

  • เครื่องมือใหม่สองตัว: broker_notes_search สำหรับการค้นหาเชิงความหมาย (semantic search) และ broker_notes_fetch สำหรับดึงเนื้อหา note แบบเต็ม โดยสื่อสารผ่าน SSE กับ Pinecone MCP server
  • การตัดทอนเนื้อหา: จำกัดการตอบกลับไว้ที่ 15,000 ตัวอักษร เพื่อป้องกันไม่ให้ context window ล้น พร้อมการจัดรูปแบบการอ้างอิง (citation) ที่รวมการดึง domain ด้วย
  • การควบคุมการเข้าถึงตามทีม: สามารถกำหนดค่าการเข้าถึงเครื่องมือได้ตามทีมผ่านการตั้งค่า โดยกรองในขณะเรียก LLM เพื่อให้ยังคงเข้ากันได้กับสถาปัตยกรรม compiled tool node ของ LangGraph
  • System prompt แบบไดนามิก: คำแนะนำเกี่ยวกับ broker notes จะปรากฏเฉพาะกับทีมที่มีสิทธิ์เข้าถึงจริงเท่านั้น พร้อมคำสั่งเชิงรุกให้ตรวจสอบ broker notes ก่อนที่จะย้อนกลับไปใช้การค้นหาเว็บ (web search)

ความน่าเชื่อถือ

เทสเคส 41 รายการ ครอบคลุมสิทธิ์การเข้าถึงของทีม การทำงานของเครื่องมือ MCP และการรับรู้ system message — ผ่านทั้งหมดโดยมีการแยกส่วน (isolation) และการทำ mocking ที่เหมาะสม

กฎการจัดการข้อมูลยังคงใช้บังคับ

Broker research notes เป็นเนื้อหาจากบุคคลที่สามที่เข้าถึงผ่านการเชื่อมต่อที่ได้รับอนุมัติ — การควบคุมการเข้าถึงตามทีมกำหนดว่าใครสามารถ query ข้อมูลนี้ได้ แต่กฎ Data Security เดียวกันยังคงใช้บังคับกับสิ่งใดก็ตามที่นำมาจากข้อมูลนี้เพื่อใช้ใน deliverable

Broker Notes MCP ချိတ်ဆက်မှု

SAM သည် ယခုအခါ Pinecone အခြေခံသော MCP server မှတစ်ဆင့် NTAsset broker research notes များကို ရှာဖွေပြီး ရယူနိုင်ပါပြီ — team အခြေခံ access control ဖြင့် ထိန်းချုပ်ထားသဖြင့် ခွင့်ပြုထားသော team များသာ လမ်းညွှန်ချက်ကို သော်လည်းကောင်း၊ ကိရိယာများကို သော်လည်းကောင်း မြင်နိုင်ပါသည်။

အသစ်ပါဝင်သည့်အချက်များ

  • ကိရိယာအသစ် နှစ်ခု — semantic search အတွက် broker_notes_search နှင့် note အကြောင်းအရာအပြည့်အစုံကို ရယူရန် broker_notes_fetch၊ Pinecone MCP server နှင့် SSE မှတစ်ဆင့် ဆက်သွယ်ပါသည်။
  • အကြောင်းအရာ ဖြတ်တောက်ခြင်း — context window ပြည့်လျှံမှုကို ကာကွယ်ရန် response များကို စာလုံး 15,000 ဖြင့် ကန့်သတ်ထားပြီး domain ထုတ်ယူမှုပါဝင်သော citation ပုံစံချထားသည်။
  • Team အခြေခံ access control — ကိရိယာအသုံးပြုခွင့်ကို settings မှတစ်ဆင့် team အလိုက် ပြင်ဆင်သတ်မှတ်နိုင်ပြီး LLM ခေါ်ယူချိန်တွင် filter လုပ်သဖြင့် LangGraph ၏ compiled tool node architecture နှင့် ဆက်လက် ကိုက်ညီနေပါသည်။
  • Dynamic system prompting — broker notes လမ်းညွှန်ချက်သည် တကယ် access ရှိသော team များထံသာ ပေါ်လာပြီး၊ web search သို့ ပြန်မသုံးမီ broker notes ကို ဦးစွာ စစ်ဆေးရန် ကြိုတင် ညွှန်ကြားချက် ပါဝင်ပါသည်။

ယုံကြည်စိတ်ချရမှု

Test case ၄၁ ခု — team ခွင့်ပြုချက်များ၊ MCP tool လုပ်ဆောင်ချက်များ နှင့် system message သိရှိမှုတို့ကို ခြုံငုံပြီး သင့်လျော်သော isolation နှင့် mocking ဖြင့် အားလုံး pass ဖြစ်ခဲ့သည်။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက် အသက်ဝင်နေဆဲ

Broker research notes များသည် ခွင့်ပြုထားသော ချိတ်ဆက်မှုမှတစ်ဆင့် ရယူနိုင်သော တတိယပါတီ အကြောင်းအရာများ ဖြစ်သည် — team access control က ဤအချက်အလက်များကို မည်သူ query လုပ်နိုင်သည်ကို သတ်မှတ်ပေးသော်လည်း၊ ၎င်းမှ deliverable တစ်ခုသို့ ထုတ်ယူအသုံးပြုသည့် မည်သည့်အရာအတွက်မဆို တူညီသော Data Security စည်းမျဉ်းများ ဆက်လက် အသက်ဝင်နေဆဲ ဖြစ်သည်။
