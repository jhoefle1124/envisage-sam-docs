Portfolio Analyst Field Parity

The portfolio analyst tool now exposes all 60+ financial and company fields in the database schema, with a new include_fields parameter for requesting just the subset you need.

What's New

  • 60+ new fields for full parity with the database schema — financial metrics (ROA, ROE, revenue growth rate), valuation metrics (EV/EBITDA, price-to-sales, FCF yield), company fundamentals (market cap, industry, country), and performance indicators (quarterly revenue/earnings growth).
  • Selective field inclusion: the new include_fields parameter requests specific subsets of data, reducing response size and improving performance. Defaults to all fields for backward compatibility.

Reliability

Verified that all new fields return correct data, that include_fields works across combinations, and that formatting is correct for percentages, currency, and dates. Backward compatible when the parameter is omitted.

Data Handling Still Applies

More fields doesn't mean less sensitivity — portfolio company fundamentals tied to a client engagement remain Tier 3 per Data Security, regardless of how many fields are requested.

ความเท่าเทียมของฟิลด์ใน Portfolio Analyst

เครื่องมือ Portfolio Analyst ในขณะนี้เปิดให้เข้าถึงฟิลด์ข้อมูลทางการเงินและข้อมูลบริษัททั้งหมดกว่า 60 ฟิลด์ตาม schema ของฐานข้อมูล พร้อมพารามิเตอร์ใหม่ include_fields สำหรับขอเฉพาะชุดข้อมูลย่อยที่ต้องการ

มีอะไรใหม่

  • ฟิลด์ใหม่กว่า 60 ฟิลด์ เพื่อให้เทียบเท่ากับ schema ของฐานข้อมูลอย่างสมบูรณ์ — ตัวชี้วัดทางการเงิน (ROA, ROE, อัตราการเติบโตของรายได้), ตัวชี้วัดมูลค่า (EV/EBITDA, price-to-sales, FCF yield), ข้อมูลพื้นฐานของบริษัท (market cap, industry, ประเทศ) และตัวชี้วัดผลการดำเนินงาน (การเติบโตของรายได้/กำไรรายไตรมาส)
  • การเลือกรวมฟิลด์แบบเจาะจง: พารามิเตอร์ include_fields ใหม่ช่วยให้ขอชุดข้อมูลย่อยที่ต้องการได้อย่างเจาะจง ลดขนาดของ response และช่วยเพิ่มประสิทธิภาพการทำงาน โดยค่าเริ่มต้นจะดึงข้อมูลทุกฟิลด์เพื่อความเข้ากันได้กับระบบเดิม

ความน่าเชื่อถือ

ตรวจสอบยืนยันแล้วว่าฟิลด์ใหม่ทั้งหมดส่งคืนข้อมูลที่ถูกต้อง include_fields ทำงานได้ถูกต้องในทุกชุดค่าผสม และการจัดรูปแบบถูกต้องสำหรับเปอร์เซ็นต์ สกุลเงิน และวันที่ ระบบยังคงเข้ากันได้กับเวอร์ชันเดิมเมื่อไม่ระบุพารามิเตอร์นี้

หลักการจัดการข้อมูลยังคงเดิม

ฟิลด์ที่มากขึ้นไม่ได้แปลว่าความอ่อนไหวลดลง — ข้อมูลพื้นฐานของบริษัทในพอร์ตที่ผูกกับงานของลูกค้ายังคงเป็น Tier 3 ตามที่ระบุใน Data Security ไม่ว่าจะมีการขอฟิลด์จำนวนเท่าใดก็ตาม

SAM တွင် အသစ် · ဝန်ဆောင်မှု လမ်းညွှန်

Portfolio Analyst ကွက် ညီညွတ်မှု

Portfolio Analyst tool သည် database schema ရှိ ဘဏ္ဍာရေးနှင့် ကုမ္ပဏီဆိုင်ရာ ကွက် 60 ကျော်လုံးကို ယခုအခါ ဖော်ပြပေးနိုင်ပြီဖြစ်ပြီး၊ လိုအပ်သော subset ကိုသာ တောင်းဆိုနိုင်ရန် include_fields ပါရာမီတာအသစ်ကိုလည်း ထည့်သွင်းထားသည်။

အသစ်ပါဝင်လာသည့်အချက်များ

  • ကွက် 60 ကျော် အသစ် — database schema နှင့် အပြည့်အဝ ညီညွတ်စေရန်။ ဘဏ္ဍာရေး metric များ (ROA, ROE, ၀င်ငွေ တိုးတက်နှုန်း), တန်ဖိုးဆန်းစစ်ချက် metric များ (EV/EBITDA, price-to-sales, FCF yield), ကုမ္ပဏီ အခြေခံအချက်များ (market cap, industry, နိုင်ငံ), နှင့် စွမ်းဆောင်ရည် ညွှန်းကိန်းများ (သုံးလပတ် ၀င်ငွေ/အမြတ် တိုးတက်နှုန်း)။
  • ရွေးချယ်၍ ကွက်ထည့်ခြင်း — response size ကို လျှော့ချပြီး စွမ်းဆောင်ရည်ကို မြှင့်တင်ရန် ဒေတာ subset အတိအကျကို တောင်းဆိုနိုင်သော include_fields ပါရာမီတာအသစ်။ backward compatibility အတွက် ပါရာမီတာကို ချန်လှပ်ထားပါက ကွက်အားလုံးကို default အဖြစ် ပြန်ပေးသည်။

ယုံကြည်စိတ်ချရမှု

ကွက်အသစ်အားလုံးသည် မှန်ကန်သော ဒေတာကို ပြန်ပေးကြောင်း၊ include_fields သည် ပေါင်းစပ်မှု အမျိုးမျိုးတွင် အလုပ်လုပ်ကြောင်း၊ နှင့် ရာခိုင်နှုန်း၊ ငွေကြေး၊ ရက်စွဲများအတွက် ပုံစံချထားမှု မှန်ကန်ကြောင်း စစ်ဆေးအတည်ပြုပြီးဖြစ်သည်။ ပါရာမီတာကို ချန်လှပ်ထားပါက backward compatible ဖြစ်သည်။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

ကွက်ပိုများလာခြင်းသည် အထိခိုက်ခံနိုင်မှု လျော့ကျခြင်း မဟုတ်ပါ — ဖောက်သည် engagement တစ်ခုနှင့် ဆက်စပ်နေသော portfolio ကုမ္ပဏီ အခြေခံအချက်များသည် Data Security အရ Tier 3 အဖြစ် ဆက်လက်ရှိနေမည်ဖြစ်ပြီး၊ ကွက် မည်မျှ တောင်းဆိုသည်ဖြစ်စေ ၎င်းအတိုင်းဖြစ်သည်။
