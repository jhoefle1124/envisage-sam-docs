New in SAM · Feature Guide

Firecrawl Web Extraction Tool

SAM can now pull content from JavaScript-rendered websites — single pages, full site crawls, or parallel batches of URLs — through a new Firecrawl tool with built-in caching and retry logic.

Three Modes

Scrape

Single page extraction with full JavaScript rendering.

Crawl

Full site crawling with configurable depth and path filters.

Batch

Multiple URLs extracted in parallel in one request.

Supports multiple output formats (markdown, HTML, structured) with built-in caching and retry logic, plus comprehensive error handling.

Reliability & Code Quality

  • Fixed a bug where crawl mode returned empty content when using the structured format.
  • Refactored the core method into configuration classes (CrawlConfig, ExtractionRequest), mode-specific handlers, a unified retry wrapper, and utility methods for cache keys and payloads — eliminating all complexity warnings.
  • 94 new unit tests, all passing, alongside integration tests and manual verification of all three modes.

Data Handling Still Applies

Content pulled from the open web is generally Tier 1 on its own. It becomes Tier 2/3 the moment it's combined with ENVISAGE analysis, client framing, or is crawled from a site containing non-public information — the same rule that applies to any other OSINT source on Usage of OSINT.
