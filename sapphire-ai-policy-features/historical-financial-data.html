Historical Financial Data

The company financials tool now reaches back in time — income statements, cash flows, balance sheets, and analyst forecasts — with a new data_types parameter and intelligent caching tuned for how often each kind of data actually changes.

What You Can Ask For

data_types valueReturns
overview (default)Current financial metrics
income_statementHistorical profit & loss data
cash_flowHistorical cash flow statements
balance_sheetHistorical balance sheets
forecastAnalyst consensus estimates
allEvery available data type in one call

A years parameter controls how far back to go (1–10, default 5), and a metrics parameter narrows the response to specific fields.

Caching

  • 5-minute TTL for overview (current) data.
  • 24-hour TTL for historical data, which changes far less often.
  • Cache keys include every parameter, so requests for different data types or years don't collide.

Reliability

~200 lines of duplicated code eliminated by extracting a generic _fetch_financial_statements method, and the main run method shrank from 316 lines to roughly 60 through helper extraction. All 57 existing tests continue to pass.

Data Handling Still Applies

Historical financials for a publicly listed company are generally Tier 1 on their own. The moment a query is framed around a specific client engagement, treat it as Tier 2/3 per Data Security.

ใหม่ใน SAM · คู่มือฟีเจอร์

ข้อมูลการเงินย้อนหลัง

เครื่องมือข้อมูลการเงินของบริษัทตอนนี้สามารถย้อนดูข้อมูลในอดีตได้แล้ว — งบกำไรขาดทุน กระแสเงินสด งบดุล และประมาณการของนักวิเคราะห์ — ด้วยพารามิเตอร์ data_types ใหม่ และระบบแคชอัจฉริยะที่ปรับตามความถี่ในการเปลี่ยนแปลงจริงของข้อมูลแต่ละประเภท

สิ่งที่คุณสามารถขอได้

ค่า data_typesผลลัพธ์ที่ได้
overview (ค่าเริ่มต้น)ข้อมูลการเงินปัจจุบัน
income_statementข้อมูลกำไรขาดทุนย้อนหลัง
cash_flowงบกระแสเงินสดย้อนหลัง
balance_sheetงบดุลย้อนหลัง
forecastประมาณการที่นักวิเคราะห์เห็นพ้องต้องกัน
allข้อมูลทุกประเภทที่มีในการเรียกครั้งเดียว

พารามิเตอร์ years ใช้กำหนดว่าจะย้อนดูข้อมูลย้อนหลังไปกี่ปี (1–10 ค่าเริ่มต้นคือ 5) และพารามิเตอร์ metrics ใช้จำกัดผลลัพธ์ให้แสดงเฉพาะฟิลด์ที่ต้องการ

การแคชข้อมูล

  • TTL 5 นาที สำหรับข้อมูล overview (ข้อมูลปัจจุบัน)
  • TTL 24 ชั่วโมง สำหรับข้อมูลย้อนหลัง ซึ่งเปลี่ยนแปลงน้อยกว่ามาก
  • คีย์ของแคชครอบคลุมพารามิเตอร์ทุกตัว คำขอที่ใช้ data_types หรือ years ต่างกันจึงไม่ชนกัน

ความน่าเชื่อถือ

ลดโค้ดที่ซ้ำซ้อนไปได้ประมาณ 200 บรรทัด ด้วยการแยกเมธอด _fetch_financial_statements แบบทั่วไปออกมาใช้ร่วมกัน และเมธอด run หลักก็ลดขนาดจาก 316 บรรทัด เหลือประมาณ 60 บรรทัด ผ่านการแยก helper ทดสอบเดิมทั้ง 57 รายการยังคงผ่านทั้งหมด

หลักการจัดการข้อมูลยังคงใช้ได้เหมือนเดิม

ข้อมูลการเงินย้อนหลังของบริษัทที่จดทะเบียนในตลาดหลักทรัพย์โดยทั่วไปถือเป็น Tier 1 เมื่อใช้ตามลำพัง ทันทีที่คำขอถูกกำหนดกรอบโดยอิงกับงานของลูกค้ารายใดรายหนึ่งโดยเฉพาะ ให้ถือว่าเป็น Tier 2/3 ตามที่ระบุใน Data Security

SAM တွင် အသစ် · ဝန်ဆောင်မှု လမ်းညွှန်

ငွေကြေးရာဇဝင် အချက်အလက်များ

ကုမ္ပဏီ ငွေကြေးအချက်အလက် ကိရိယာသည် ယခုအခါ အချိန်ကာလနောက်ပြန်၍ ကြည့်ရှုနိုင်ပြီ — ဝင်ငွေထုတ်ပြန်ချက်များ၊ ငွေသားစီးဆင်းမှုများ၊ လက်ကျန်ရှင်းတမ်းများ နှင့် သုံးသပ်သူ ခန့်မှန်းချက်များ — data_types parameter အသစ်တစ်ခုနှင့် ဒေတာအမျိုးအစားတစ်ခုစီ မည်မျှမကြာခဏ ပြောင်းလဲသည်ကို လိုက်၍ ချိန်ညှိထားသော ဉာဏ်ရည်ရှိ caching စနစ်ဖြင့်။

မည်သည့်အချက်အလက်ကို တောင်းဆိုနိုင်သနည်း

data_types တန်ဖိုးရလဒ်
overview (default)လက်ရှိ ငွေကြေးဆိုင်ရာ metrics များ
income_statementရာဇဝင် ဝင်ငွေထုတ်ပြန်ချက် ဒေတာ
cash_flowရာဇဝင် ငွေသားစီးဆင်းမှု ထုတ်ပြန်ချက်များ
balance_sheetရာဇဝင် လက်ကျန်ရှင်းတမ်းများ
forecastသုံးသပ်သူများ သဘောတူညီသော ခန့်မှန်းချက်များ
allရနိုင်သော ဒေတာအမျိုးအစား အားလုံးကို တစ်ကြိမ်တည်း ခေါ်ယူခြင်း

years parameter သည် မည်မျှ အချိန်နောက်ပြန်ကြည့်မည်ကို ထိန်းချုပ်ပေးသည် (1–10, ပုံသေတန်ဖိုး 5) ။ metrics parameter သည် ရလဒ်ကို သတ်မှတ်ထားသော field များအထိသာ ကျဉ်းသွားစေသည်။

Caching

  • overview (လက်ရှိ) ဒေတာအတွက် TTL 5 မိနစ်
  • ပို၍ ရှားရှားပါးပါးသာ ပြောင်းလဲတတ်သော ရာဇဝင်ဒေတာအတွက် TTL 24 နာရီ
  • Cache key များတွင် parameter တိုင်းပါဝင်သဖြင့် data_types သို့မဟုတ် years မတူညီသော တောင်းဆိုမှုများသည် အချင်းချင်း တိုက်နေခြင်း မရှိပါ

ယုံကြည်စိတ်ချရမှု

ထပ်နေသော code လိုင်း ~200 ခန့်ကို ဖယ်ရှားနိုင်ခဲ့သည် — generic _fetch_financial_statements method တစ်ခု ထုတ်ယူခြင်းဖြင့် ဖြစ်သည်။ helper extraction မှတစ်ဆင့် run method အဓိကသည် လိုင်း 316 မှ ခန့်မှန်း 60 အထိ ကျုံ့သွားခဲ့သည်။ လက်ရှိစမ်းသပ်မှု 57 ခုလုံး ဆက်လက်အောင်မြင်နေဆဲဖြစ်သည်။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

အများပြည်သူ စာရင်းဝင် ကုမ္ပဏီတစ်ခု၏ ရာဇဝင် ငွေကြေးအချက်အလက်များသည် တစ်ကိုယ်တည်း အသုံးပြုသောအခါ ယေဘုယျအားဖြင့် Tier 1 ဖြစ်သည်။ မေးမြန်းချက်တစ်ခုကို သီးခြား client engagement တစ်ခုအတွက် ဘောင်ဖွဲ့လိုက်သည်နှင့်တစ်ပြိုင်နက် Data Security အရ Tier 2/3 အဖြစ် သတ်မှတ်ပါ
