New in SAM · Technical Reference

Model Fallback Chains

If your chosen model becomes unavailable mid-conversation, SAM now falls back automatically through a multi-level chain instead of failing outright — part of a broader update that also migrated the embedding model and hardened document processing.

Model Fallback System

  • Multi-level fallback chain: automatic runtime fallback when a saved session model is stale or rejected, with a toast notification telling you the model was auto-switched.
  • Context rebudgeting: when falling back to a model with a different token limit, the context budget is recalculated rather than assumed.
  • Original model state is restored if the fallback itself fails, and silent failures were replaced with explicit errors throughout model resolution.

Model & Infrastructure Updates

  • Title generation switched to gpt-oss-120b, with tool-announcement messages filtered out of what it summarizes.
  • Embedding model migrated to Qwen3-Embedding-8B at 1024 dimensions — existing embeddings are re-processed via a data migration.
  • Distributed document processing locks with staleness recovery, preventing two processes from working the same document at once.

UI Improvements

Message creation date and time now display under chat bubbles (date visible, full timestamp on hover), document chips are clickable to open the file directly, and stalled uploads show an amber warning icon instead of spinning indefinitely.

Reliability

5,680+ lines of new tests across document locks, fallback chains, context rebudgeting, the embedding service, and the migration itself, spanning 52 files.

Data Handling Still Applies

Switching models automatically doesn't change what tier of content may be sent to it — the fallback chain still respects the same Data Security tool and model rules as a manually selected model.

ห่วงโซ่การสำรองโมเดล (Model Fallback Chains)

หากโมเดลที่คุณเลือกใช้งานไม่พร้อมใช้งานระหว่างการสนทนา ตอนนี้ SAM จะสำรอง (fallback) ไปยังโมเดลอื่นโดยอัตโนมัติผ่านห่วงโซ่หลายระดับ แทนที่จะล้มเหลวไปเลย — ซึ่งเป็นส่วนหนึ่งของการอัปเดตที่ครอบคลุมกว่าเดิม ซึ่งรวมถึงการย้าย embedding model และการเสริมความแข็งแกร่งให้กับการประมวลผลเอกสารด้วย

ระบบ Model Fallback

  • ห่วงโซ่การสำรองหลายระดับ: ระบบจะสำรองโมเดลโดยอัตโนมัติขณะรันไทม์ เมื่อโมเดลที่บันทึกไว้ในเซสชันล้าสมัยหรือถูกปฏิเสธ พร้อมแจ้งเตือนแบบ toast บอกคุณว่ามีการสลับโมเดลให้โดยอัตโนมัติ
  • การคำนวณ context budget ใหม่: เมื่อสำรองไปยังโมเดลที่มี token limit ต่างกัน ระบบจะคำนวณ context budget ใหม่แทนที่จะสันนิษฐานเอาเอง
  • สถานะโมเดลเดิมจะถูกกู้คืนหากการสำรองโมเดลเองล้มเหลว และข้อผิดพลาดแบบเงียบ (silent failures) ถูกแทนที่ด้วยข้อผิดพลาดที่ชัดเจนตลอดกระบวนการหาโมเดล (model resolution)

การอัปเดตโมเดลและโครงสร้างพื้นฐาน

  • การสร้างชื่อหัวข้อ (title generation) เปลี่ยนไปใช้ gpt-oss-120b โดยกรองข้อความแจ้งเตือนการใช้เครื่องมือ (tool-announcement messages) ออกจากสิ่งที่นำมาสรุป
  • ย้าย embedding model ไปใช้ Qwen3-Embedding-8B ที่ 1024 มิติ — embeddings เดิมจะถูกประมวลผลใหม่ผ่านการทำ data migration
  • ใช้ระบบล็อกการประมวลผลเอกสารแบบกระจาย (distributed) พร้อมการกู้คืนจากสถานะค้าง (staleness recovery) เพื่อป้องกันไม่ให้สองกระบวนการทำงานกับเอกสารเดียวกันพร้อมกัน

การปรับปรุง UI

วันที่และเวลาที่สร้างข้อความจะแสดงใต้ chat bubble แล้ว (แสดงวันที่ตลอด และแสดง timestamp เต็มเมื่อวางเมาส์ค้าง) ชิปเอกสาร (document chips) สามารถคลิกเพื่อเปิดไฟล์ได้โดยตรง และการอัปโหลดที่ค้างจะแสดงไอคอนเตือนสีเหลืองอำพันแทนที่จะหมุนวนไม่รู้จบ

ความน่าเชื่อถือ

เทสต์ใหม่มากกว่า 5,680 บรรทัด ครอบคลุมการล็อกเอกสาร ห่วงโซ่การสำรองโมเดล การคำนวณ context budget ใหม่ บริการ embedding และตัว migration เอง รวมทั้งหมด 52 ไฟล์

การจัดการข้อมูลยังคงบังคับใช้เหมือนเดิม

การสลับโมเดลโดยอัตโนมัติไม่ได้เปลี่ยนแปลงว่าเนื้อหาระดับใด (tier) สามารถส่งไปยังโมเดลนั้นได้ — ห่วงโซ่การสำรองโมเดลยังคงปฏิบัติตามกฎเครื่องมือและโมเดลของ Data Security เช่นเดียวกับการเลือกโมเดลด้วยตนเอง

Model Fallback Chains

သင်ရွေးချယ်ထားသော model သည် စကားပြောနေစဉ်အတွင်း မရရှိနိုင်တော့ပါက SAM သည် ယခုအခါ လုံးဝ fail မဖြစ်တော့ဘဲ၊ level များစွာပါဝင်သော chain တစ်ခုမှတစ်ဆင့် အလိုအလျောက် fallback ပြုလုပ်ပေးပါသည် — ဤသည်မှာ embedding model ကို ပြောင်းရွှေ့ခြင်းနှင့် document processing ကို ပိုမိုကြံ့ခိုင်အောင် ပြုလုပ်ခြင်းလည်း ပါဝင်သော ပိုမိုကျယ်ပြန့်သည့် update ၏ တစ်စိတ်တစ်ပိုင်း ဖြစ်ပါသည်။

Model Fallback စနစ်

  • Level များစွာပါ Fallback Chain: သိမ်းဆည်းထားသော session model သည် ရက်ကုန်သွားခြင်း သို့မဟုတ် ငြင်းပယ်ခံရခြင်းရှိပါက runtime တွင် အလိုအလျောက် fallback ပြုလုပ်ပြီး၊ model ကို auto-switch လုပ်ခဲ့ကြောင်း toast notification ဖြင့် အသိပေးပါသည်။
  • Context Rebudgeting: token limit မတူညီသော model တစ်ခုသို့ fallback ပြုလုပ်သည့်အခါ context budget ကို ယူဆမည့်အစား ပြန်လည်တွက်ချက်ပေးပါသည်။
  • Fallback ကိုယ်တိုင် မအောင်မြင်ပါက မူလ model အခြေအနေကို ပြန်လည်ရယူပေးပြီး၊ model resolution တစ်လျှောက်လုံးတွင် silent failure များအား ရှင်းလင်းသော error များဖြင့် အစားထိုးထားပါသည်။

Model နှင့် Infrastructure အပ်ဒိတ်များ

  • Title generation ကို gpt-oss-120b သို့ ပြောင်းလိုက်ပြီး၊ အကျဉ်းချုပ်ရာတွင် tool-announcement message များကို ဖယ်ထုတ်ထားသည်။
  • Embedding model ကို Qwen3-Embedding-8B (dimension 1024) သို့ ပြောင်းရွှေ့ထားပြီး — ရှိပြီးသား embedding များကို data migration တစ်ခုမှတစ်ဆင့် ပြန်လည်ပြုပြင်ပါသည်။
  • Document processing lock များကို distributed ပုံစံဖြင့် staleness recovery ပါဝင်စွာ အသုံးပြုထားသဖြင့် process နှစ်ခုက document တစ်ခုတည်းကို တစ်ပြိုင်နက်တည်း ကိုင်တွယ်မှု မဖြစ်ပေါ်စေပါ။

UI တိုးတက်မှုများ

Message ဖန်တီးသည့် ရက်စွဲနှင့် အချိန်ကို chat bubble အောက်တွင် ယခုပြသပါပြီ (ရက်စွဲကို မြင်ရပြီး၊ mouse ကို hover လုပ်လျှင် timestamp အပြည့်အစုံပြပါသည်)၊ document chip များကို click နှိပ်ပြီး ဖိုင်ကို တိုက်ရိုက်ဖွင့်နိုင်ပါသည်၊ ရပ်တန့်နေသော upload များကို အဆုံးမရှိ လည်နေမည့်အစား အဝါရောင် သတိပေးအိုင်ကွန်ဖြင့် ပြသပါသည်။

ယုံကြည်စိတ်ချရမှု

Test line 5,680+ ကို အသစ်ရေးထားသည် document lock, fallback chain, context rebudgeting, embedding service နှင့် migration ကိုယ်တိုင်အပါအဝင် ဖိုင် 52 ဖိုင်ကို လွှမ်းခြုံထားသည်။

ဒေတာ ကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

Model ကို အလိုအလျောက် ပြောင်းလဲခြင်းသည် ၎င်းသို့ မည်သည့် tier အဆင့်ရှိ content ကို ပို့နိုင်သည်ကို မပြောင်းလဲစေပါ — fallback chain သည် model ကို ကိုယ်တိုင်ရွေးချယ်ခြင်းနှင့် အတူတူပင် Data Security ကိရိယာနှင့် model စည်းမျဉ်းများကို ဆက်လက်လိုက်နာပါသည်။
