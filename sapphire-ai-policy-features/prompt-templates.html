Prompt Studio Modal & Company Autocomplete

The chat input got a modal-based prompt studio with a categorized template catalog, an EasyMDE markdown editor, and company autocomplete — plus placeholder validation that blocks submission until every {{blank}} is filled.

Advanced Prompt Editor

  • Modal-based prompt studio with an EasyMDE markdown editor and an accordion-organized template catalog by category.
  • Smart placeholder detection with visual highlighting, and validation that blocks submitting a prompt with unfilled placeholders.
  • Bidirectional sync between the simple chat input and the full prompt editor.

Company Integration

An autocomplete selector in the main chat input adds quick company context; the same autocomplete is available for ad-hoc company insertion inside the prompt editor, with support for MIC-coded tickers in identifier resolution.

Reliability

33 files changed, 5,579 insertions. A management command creates sample prompt templates; most changes are additive and existing functionality remains intact. See also Prompt Studio for the admin-side authoring workflow.

Data Handling Still Applies

Autocomplete inserts a company name into your prompt — it doesn't change what's safe to type. Anonymize client-specific framing exactly as described on Data Security.

Prompt Studio Modal และระบบเติมชื่อบริษัทอัตโนมัติ

ช่องแชทได้รับ Prompt Studio แบบ modal พร้อมแคตตาล็อกเทมเพลตที่จัดหมวดหมู่ ตัวแก้ไข markdown แบบ EasyMDE และระบบเติมชื่อบริษัทอัตโนมัติ — พร้อมการตรวจสอบ placeholder ที่ป้องกันการส่งจนกว่าทุก {{blank}} จะถูกกรอกครบ

ตัวแก้ไข Prompt ขั้นสูง

  • Prompt Studio แบบ modal พร้อมตัวแก้ไข markdown แบบ EasyMDE และแคตตาล็อกเทมเพลตที่จัดเรียงแบบ accordion ตามหมวดหมู่
  • ตรวจจับ placeholder อย่างชาญฉลาดพร้อมไฮไลต์ให้เห็นภาพ และการตรวจสอบที่ป้องกันการส่ง prompt ที่ยังกรอก placeholder ไม่ครบ
  • การซิงค์สองทิศทางระหว่างช่องแชทแบบง่ายกับตัวแก้ไข prompt แบบเต็มรูปแบบ

การผสานรวมข้อมูลบริษัท

ตัวเลือกเติมอัตโนมัติในช่องแชทหลักช่วยเพิ่มบริบทของบริษัทได้อย่างรวดเร็ว โดยระบบเติมอัตโนมัติเดียวกันนี้ยังใช้ได้กับการแทรกชื่อบริษัทเฉพาะกิจภายในตัวแก้ไข prompt พร้อมรองรับ ticker ที่มีรหัส MIC ในการระบุตัวตน

ความน่าเชื่อถือ

แก้ไข 33 ไฟล์ เพิ่มโค้ด 5,579 บรรทัด มีคำสั่งจัดการ (management command) สำหรับสร้างเทมเพลต prompt ตัวอย่าง โดยการเปลี่ยนแปลงส่วนใหญ่เป็นการเพิ่มเติมและฟังก์ชันเดิมยังคงทำงานได้ตามปกติ ดูเพิ่มเติมที่ Prompt Studio สำหรับ workflow การเขียนฝั่งผู้ดูแลระบบ

หลักการจัดการข้อมูลยังคงมีผล

ระบบเติมอัตโนมัติเพียงแทรกชื่อบริษัทลงใน prompt ของคุณ — ไม่ได้เปลี่ยนแปลงว่าอะไรปลอดภัยที่จะพิมพ์ ควรทำให้บริบทเฉพาะของลูกค้าไม่ระบุตัวตนตามที่อธิบายไว้ใน ความปลอดภัยของข้อมูล

Prompt Studio Modal နှင့် ကုမ္ပဏီအမည် အလိုအလျောက်ဖြည့်ခြင်း

Chat input သည် အမျိုးအစားလိုက် စီထားသော template catalog၊ EasyMDE markdown editor နှင့် ကုမ္ပဏီအမည် autocomplete ပါဝင်သော modal-based prompt studio ရရှိလာသည် — {{blank}} တိုင်း ဖြည့်စွက်ပြီးမှသာ တင်သွင်းခွင့်ပြုသော placeholder validation နှင့်တကွ။

အဆင့်မြင့် Prompt တည်းဖြတ်စနစ်

  • EasyMDE markdown editor နှင့် အမျိုးအစားအလိုက် accordion ဖြင့် စီစဉ်ထားသော template catalog ပါဝင်သည့် modal-based prompt studio
  • Placeholder များကို ရှုမြင်နိုင်သော highlighting ဖြင့် စမတ်ကျစွာ ရှာဖွေခြင်းနှင့် ဖြည့်စွက်မထားသော placeholder ပါသော prompt တင်သွင်းခြင်းကို ပိတ်ဆို့ပေးသော validation
  • ရိုးရှင်းသော chat input နှင့် prompt editor အပြည့်အစုံ ကြားရှိ နှစ်ဖက်စလုံး ထပ်တူညီစေခြင်း

ကုမ္ပဏီ ပေါင်းစပ်ချိတ်ဆက်မှု

အဓိက chat input ရှိ autocomplete selector က ကုမ္ပဏီ context ကို လျင်မြန်စွာ ထည့်သွင်းပေးသည်၊ prompt editor အတွင်း ad-hoc ကုမ္ပဏီ ထည့်သွင်းမှုအတွက်လည်း autocomplete တူတစ်ခုကို ရရှိနိုင်ပြီး identifier resolution တွင် MIC-coded ticker များကို ပံ့ပိုးထားသည်။

ယုံကြည်စိတ်ချရမှု

ဖိုင် 33 ဖိုင် ပြောင်းလဲထားပြီး 5,579 လိုင်း ထပ်ပေါင်းထားသည်။ Management command တစ်ခုက နမူနာ prompt template များ ဖန်တီးပေးသည်၊ ပြောင်းလဲမှု အများစုမှာ ထပ်ပေါင်းသည့်သဘောသာ ဖြစ်ပြီး ရှိပြီးသား လုပ်ဆောင်ချက်များ ဆက်လက် အလုပ်လုပ်နေသည်။ Admin ဘက်မှ prompt ရေးသားရေး workflow အတွက် Prompt Studio ကိုလည်း ကြည့်ပါ။

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်

Autocomplete သည် ကုမ္ပဏီအမည်ကို သင့် prompt ထဲသို့ ထည့်သွင်းပေးရုံသာ — ဘာစာရိုက်ရန် ဘေးကင်းသည်ဆိုသည်ကို ပြောင်းလဲပေးသည် မဟုတ်ပါ။ ဒေတာလုံခြုံရေး တွင် ဖော်ပြထားသည့်အတိုင်း ဖောက်သည်-သီးခြား framing ကို အမည်ဝှက်ပါ။
