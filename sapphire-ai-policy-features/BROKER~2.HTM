Broker Notes Date Filtering & Opus 4.5

Broker notes search gained date-range filtering and configurable result limits, alongside a cache key fix and an upgrade of the Claude Opus model powering the chatbot to version 4.5.

What's New

  • Date filtering: start_date and end_date parameters (YYYY-MM-DD) narrow results to a date range, defaulting to the last 12 months when unspecified.
  • Result limits: a new max_results parameter (1–50, default 10) controls how many results come back.
  • Validation: date range validation ensures the start date is on or before the end date, with tolerance for LLM-generated artifacts like quoted strings or stray whitespace.

Reliability Improvements

  • Null values in API responses (title, text, url) now fall back correctly instead of surfacing errors.
  • Cache keys now hash the entire JSON payload (query, result count, dates) instead of just the query string, preventing cache collisions.

Claude Opus was upgraded from 4.1 to 4.5, bringing improved reasoning and performance to the chatbot.

Reliability

20+ new unit tests covering date validation, range validation, result-limit bounds, null handling, and cache key generation. Backward compatible — existing calls without date or limit parameters keep working with sensible defaults.

Data Handling Still Applies

Filtering by date doesn't change what's safe to do with the results — see Broker Notes MCP Integration and Data Security.
