New in SAM · Feature Guide

Improving Anthropic Content and Complexity

SAM now extracts Claude's streaming responses more robustly — fewer stray fragments in chat, cleaner handling of "thinking" content, and a single, well-tested code path instead of scattered logic. You shouldn't have to think about any of this; that's the point.

What's New

  • Robust content extraction: new shared utility functions handle every shape Claude's streaming API can return — plain text, delta updates, lists of content blocks, and more.
  • Tool block detection: tool-related content (tool_use, tool_result, input_json_delta, etc.) is now properly identified and filtered out of what you see, instead of occasionally leaking through.
  • Cleaner "thinking" tag removal: an optimized pattern strips both raw and HTML-encoded thinking tags in a single pass.
  • Simplified streaming code: the event-processing logic that turns Claude's response stream into chat messages was refactored to use one centralized extraction path.

Why It Matters

This is a behind-the-scenes reliability change — you won't see a new button or setting. What you should notice is fewer rendering glitches in chat: no stray tool-call fragments or leftover "thinking" markup showing up where they shouldn't, and more consistent behavior regardless of how Claude structures a given streamed response.

What Changed Under the Hood

ComponentWhat It Does Now
extract_message_content()Single orchestrating function that handles all content extraction strategies
Delta & list handlingDedicated helpers extract text from delta updates and from lists of mixed content blocks
Tool block filteringTool-related blocks are identified and excluded from the visible message content
Streaming event processingRefactored to call the shared extraction function instead of duplicating logic in multiple places

Reliability

No breaking changes. This is a refactor with added robustness, not a behavior change you need to adjust to. It ships with 46 new tests (27 covering the LLM factory, 19 covering streaming event processing) alongside the full existing test suite, which continues to pass.

Data Handling Still Applies

This change is about how SAM renders Claude's responses more reliably — it doesn't change what data flows where. The same Data Security rules on what you type into SAM chat still apply exactly as before.
