New in SAM · Feature Guide

Citation, UI & Screener Improvements

A broad pass across three areas: citation accuracy and tool-specific attribution, a snappier streaming and auto-scroll experience, and a modernized stock screener with query complexity protection.

Citation System Enhancements

  • Tool-specific attribution: citations now carry prefixes like "SAM Database:" or "Stock Screener:" for clarity.
  • Generic SAM database citations: standardized to prevent duplication across a response.
  • Smart prioritization: database citations get lower numbers, which prevents the LLM from generating its own duplicate source list.
  • Safer dict handling: citations arriving as plain dictionaries no longer raise an AttributeError.

Performance & UX

Artificial streaming delays were removed, auto-scrolling is smoother during token streaming, and the LangGraph recursion limit was raised from 50 to 75 with graceful handling — including automatic title generation and response persistence when the limit is hit, so a long conversation never silently loses its answer.

Stock Screener Modernization

ChangeDetail
Modular architectureReorganized into a dedicated module with a field configuration registry and an extracted SQL builder class
Query complexity limitsAdded database protection through query complexity scoring
Result limitsMaximum results increased from 50 to 100
Quick filtersRemoved — an underutilized feature that added complexity

A new FinancialFormatter class standardizes financial data formatting across the app, and company_financials was migrated from raw SQL to Django ORM.

Critical Bug Fixes

Dividend yield was displaying 100x too high (41.69% instead of 0.42%) because the database stores yields as already-percentage values while the tool treated them as decimals. Fixed with corrected documentation and proper formatting functions per field.

Also fixed: a citation AttributeError that crashed multi-turn conversations when citations arrived as dictionaries instead of objects.

Data Handling Still Applies

Better citations and a modernized screener don't change data sensitivity. Screener results built from public company fundamentals stay close to Tier 1; results tied to a client portfolio or watchlist follow the same Tier 2/3 rules as everywhere else on Data Security.

การปรับปรุงการอ้างอิง UI และ Screener

การปรับปรุงครอบคลุมสามด้าน ได้แก่ ความแม่นยำของการอ้างอิง (citation) และการระบุแหล่งที่มาตามแต่ละเครื่องมือ ประสบการณ์การ streaming และ auto-scroll ที่ลื่นไหลขึ้น และ stock screener ที่ปรับปรุงใหม่พร้อมการป้องกัน query ที่ซับซ้อนเกินไป

การปรับปรุงระบบการอ้างอิง

  • การระบุแหล่งที่มาตามเครื่องมือ: การอ้างอิงตอนนี้มีคำนำหน้า เช่น "SAM Database:" หรือ "Stock Screener:" เพื่อความชัดเจน
  • การอ้างอิงฐานข้อมูล SAM แบบทั่วไป: ทำให้เป็นมาตรฐานเพื่อป้องกันการซ้ำซ้อนภายในคำตอบเดียว
  • การจัดลำดับความสำคัญอัจฉริยะ: การอ้างอิงจากฐานข้อมูลจะได้หมายเลขต่ำกว่า ซึ่งช่วยป้องกันไม่ให้ LLM สร้างรายการแหล่งที่มาซ้ำของตัวเอง
  • การจัดการ dict ที่ปลอดภัยขึ้น: การอ้างอิงที่มาในรูปแบบ dictionary ธรรมดาจะไม่ก่อให้เกิด AttributeError อีกต่อไป

ประสิทธิภาพและประสบการณ์ผู้ใช้

ลบความหน่วงเทียมระหว่างการ streaming ออกแล้ว การ auto-scroll ลื่นไหลขึ้นระหว่าง token streaming และเพิ่มขีดจำกัด recursion ของ LangGraph จาก 50 เป็น 75 พร้อมการจัดการอย่างนุ่มนวล — รวมถึงการสร้างชื่อหัวข้ออัตโนมัติและการบันทึกคำตอบเมื่อถึงขีดจำกัด เพื่อไม่ให้บทสนทนายาว ๆ สูญเสียคำตอบไปเงียบ ๆ

การปรับปรุง Stock Screener ให้ทันสมัย

การเปลี่ยนแปลงรายละเอียด
สถาปัตยกรรมแบบโมดูลจัดระเบียบใหม่เป็นโมดูลเฉพาะ พร้อม field configuration registry และแยกคลาส SQL builder ออกมาต่างหาก
ขีดจำกัดความซับซ้อนของ queryเพิ่มการป้องกันฐานข้อมูลด้วยการให้คะแนนความซับซ้อนของ query
ขีดจำกัดผลลัพธ์เพิ่มจำนวนผลลัพธ์สูงสุดจาก 50 เป็น 100
ตัวกรองด่วน (Quick filters)ถูกนำออก — เป็นฟีเจอร์ที่ใช้งานน้อยและเพิ่มความซับซ้อนโดยไม่จำเป็น

คลาสใหม่ FinancialFormatter ทำให้การจัดรูปแบบข้อมูลการเงินเป็นมาตรฐานเดียวกันทั่วทั้งแอป และ company_financials ถูกย้ายจาก raw SQL มาเป็น Django ORM

การแก้ไขบั๊กสำคัญ

Dividend yield แสดงผลสูงเกินจริงถึง 100 เท่า (41.69% แทนที่จะเป็น 0.42%) เนื่องจากฐานข้อมูลเก็บค่า yield เป็นเปอร์เซ็นต์อยู่แล้ว แต่เครื่องมือกลับปฏิบัติกับค่านั้นเสมือนเป็นทศนิยม แก้ไขแล้วพร้อมเอกสารประกอบที่ถูกต้องและฟังก์ชันจัดรูปแบบเฉพาะแต่ละฟิลด์

นอกจากนี้ยังแก้ไข: AttributeError ของการอ้างอิงที่ทำให้บทสนทนาหลายรอบล่มเมื่อการอ้างอิงมาในรูปแบบ dictionary แทนที่จะเป็น object

การจัดการข้อมูลยังคงบังคับใช้

การอ้างอิงที่ดีขึ้นและ screener ที่ทันสมัยขึ้นไม่ได้เปลี่ยนแปลงระดับความอ่อนไหวของข้อมูล ผลลัพธ์จาก screener ที่สร้างจากข้อมูลพื้นฐานสาธารณะของบริษัทยังคงใกล้เคียง Tier 1 ส่วนผลลัพธ์ที่ผูกกับพอร์ตโฟลิโอหรือ watchlist ของลูกค้าจะยังคงอยู่ภายใต้กฎ Tier 2/3 เช่นเดียวกับที่อื่น ๆ ตามหน้า Data Security

SAM တွင် အသစ် · ဝန်ဆောင်မှုလမ်းညွှန်

Citation၊ UI နှင့် Screener တိုးတက်မှုများ

နယ်ပယ်သုံးခုအတွက် ကျယ်ပြန့်စွာ ပြင်ဆင်မှု — citation တိကျမှု နှင့် ကိရိယာအလိုက် source ဖော်ပြမှု၊ streaming နှင့် auto-scroll အတွေ့အကြုံ ပိုမိုချောမွေ့လာမှု၊ query ရှုပ်ထွေးမှုကို ကာကွယ်ပေးသော stock screener ခေတ်မီပြင်ဆင်မှု တို့ ပါဝင်သည်။

Citation စနစ် တိုးတက်မှုများ

  • ကိရိယာအလိုက် source ဖော်ပြမှု — citation များတွင် ယခုအခါ "SAM Database:" သို့မဟုတ် "Stock Screener:" ကဲ့သို့ ရှေ့ဆက်စာလုံးများ ရှင်းလင်းစွာ ပါရှိသည်
  • ယေဘုယျ SAM database citation — အဖြေတစ်ခုတည်းတွင် ထပ်နေမှု မဖြစ်စေရန် စံသတ်မှတ်ထားသည်
  • ထက်မြက်သော ဦးစားပေးစီစဉ်မှု — database citation များသည် နံပါတ်နည်းသည့်အနေအထားရရှိပြီး LLM မှ မိမိကိုယ်ပိုင် source list ထပ်ဖန်တီးခြင်းကို ကာကွယ်ပေးသည်
  • ပို၍ဘေးကင်းသော dict ကိုင်တွယ်မှု — citation များသည် dictionary အနေဖြင့် ရောက်ရှိလာသောအခါ AttributeError ထပ်မံဖြစ်ပေါ်တော့မည် မဟုတ်ပါ

စွမ်းဆောင်ရည် နှင့် အသုံးပြုသူအတွေ့အကြုံ

Streaming အတွင်း ဖန်တီးထားသော ကြန့်ကြာချိန်များကို ဖယ်ရှားလိုက်ပြီး token streaming အတွင်း auto-scroll ပိုမိုချောမွေ့လာသည်။ LangGraph ၏ recursion ကန့်သတ်ချက်ကို ၅၀ မှ ၇၅ သို့ တိုးမြှင့်ပြီး ချောမွေ့စွာ ကိုင်တွယ်ပေးသည် — ကန့်သတ်ချက်သို့ ရောက်သောအခါ ခေါင်းစဉ် အလိုအလျောက်ဖန်တီးခြင်းနှင့် အဖြေထိန်းသိမ်းခြင်းလည်း ပါဝင်သဖြင့် စကားဝိုင်း ရှည်ကြာမှုများသည် အဖြေကို တိတ်တဆိတ် ဆုံးရှုံးမည် မဟုတ်ပါ။

ပြောင်းလဲမှုအသေးစိတ်

Query ရှုပ်ထွေးမှု ကန့်သတ်ချက်Query ရှုပ်ထွေးမှုကို အမှတ်ပေးစနစ်ဖြင့် database ကာကွယ်မှု ထပ်တိုးထည့်ထားသည်
ရလဒ်ကန့်သတ်ချက်အများဆုံးရလဒ်ကို ၅၀ မှ ၁၀၀ သို့ တိုးမြှင့်ထားသည်

Class အသစ် FinancialFormatter က app တစ်ခုလုံးတွင် ဘဏ္ဍာရေးဒေတာ ပုံစံချမှုကို စံသတ်မှတ်ပေးပြီး company_financials ကို raw SQL မှ Django ORM သို့ ပြောင်းရွှေ့ထားသည်

အရေးကြီးသော bug ပြင်ဆင်မှုများ

Dividend yield ကို ၁၀၀ ဆ မှားယွင်းကာ မြင့်စွာ ပြသနေခဲ့သည် (0.42% အစား 41.69% ဖြစ်နေသည်) — database တွင် yield တန်ဖိုးများကို ရာခိုင်နှုန်းအနေဖြင့် သိမ်းဆည်းထားသော်လည်း ကိရိယာက ယင်းကို ဒဿမကိန်းအဖြစ် ကိုင်တွယ်နေသောကြောင့် ဖြစ်သည်။ မှန်ကန်သော documentation နှင့် field တစ်ခုစီအလိုက် ပုံစံချသော function များဖြင့် ပြင်ဆင်ပြီးပါပြီ။

ထို့အပြင် ပြင်ဆင်ခဲ့သည့်အချက် — citation များသည် object အစား dictionary အနေဖြင့် ရောက်ရှိလာသောအခါ စကားဝိုင်းများကို ပြိုကျစေခဲ့သော citation AttributeError

ဒေတာကိုင်တွယ်မှု စည်းမျဉ်းများ ဆက်လက်သက်ရောက်သည်

Citation ပိုမိုကောင်းမွန်လာခြင်းနှင့် screener ခေတ်မီလာခြင်းသည် ဒေတာ၏ အထွေထွေအရေးကြီးမှု အဆင့်ကို ပြောင်းလဲစေခြင်း မရှိပါ။ အများပြည်သူ ကုမ္ပဏီ အခြေခံဒေတာမှ ဖန်တီးထားသော screener ရလဒ်များသည် Tier 1 နှင့် နီးကပ်စွာ ကျန်ရှိနေသည်။ ဖောက်သည်၏ ပေါ်တဖိုလီယို သို့မဟုတ် watchlist နှင့် ချိတ်ဆက်ထားသော ရလဒ်များမူ Data Security စာမျက်နှာရှိ အခြားနေရာများနည်းတူ Tier 2/3 စည်းမျဉ်းများကို လိုက်နာရမည်
