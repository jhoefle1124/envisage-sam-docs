Section 2

Minimizing Token Waste

Tokens are the metered unit of everything we send to and receive from AI tools. Most waste comes from a handful of habits — fix these and team-wide spend drops sharply with no loss of quality.

Where Tokens Actually Go

Every conversation re-sends its entire history with each new message. That means a long chat with three pasted documents re-transmits all of it every single turn. The three biggest sources of waste in research work:

1

Oversized input

Pasting whole documents when one section is relevant; re-uploading files already provided; long conversations that drag history behind every message.

2

Regeneration loops

Vague prompts that need 5 attempts; "try again" instead of a precise correction; regenerating full documents to fix one paragraph.

3

Premium tools on trivial work

Using Claude tokens for tasks a free tool handles identically — generic formatting, simple lookups, boilerplate text.

The Efficiency Habits

1. One task, one chat

Start a fresh conversation for each distinct task. When a chat exceeds roughly a dozen exchanges, ask Claude to summarize the state of work in a short paragraph, copy it, and continue in a new chat with that summary. You carry forward 150 tokens instead of 15,000.

2. Excerpts, not documents

Wasteful

Pasting a 40-page assessment and asking, "What does this say about labor conditions in the northern provinces?"

Efficient

Finding the two relevant sections yourself (Ctrl+F) and pasting only those: ~600 tokens instead of ~30,000 — and a more focused answer.

3. Let Cowork read files locally

In Cowork, Claude reads files from the task folder directly and reads only what it needs. For any file-based work, connecting a folder beats pasting contents into Chat.

4. Put stable context in Projects, not prompts

Templates, methodology, terminology — written once into Project knowledge — stop costing you tokens in every conversation. See Claude & Cowork.

5. Specify the output before generating

State length, format, and audience in the first prompt: "500 words, three sections, for a donor audience, no jargon." The most expensive output is a long, beautiful draft in the wrong shape.

6. Correct precisely, never "try again"

"Try again" regenerates everything and usually repeats the misunderstanding. Instead: "Keep the structure and tone. Fix only: (1) the date range should be 2023–2025, (2) shorten the intro to two sentences."

7. Batch related questions

Five small questions in one message cost less than five separate exchanges, because each separate exchange re-sends the conversation history.

8. Route Tier 1 work to free tools

Generic public-domain tasks — summarizing a news article, drafting boilerplate, explaining a Python error — go to DeepSeek, Kimi, or other free tools. Save Claude capacity for work that needs our context, judgment, and confidentiality. See the AI Toolkit.

9. Don't use AI for what software does better

Sorting a spreadsheet, counting rows, exact find-and-replace — Excel and your editor do these instantly and for free. Use AI to decide what to do, not as a calculator.

Quick Reference: Cost of Common Habits

HabitApproximate Token Effect
Pasting a full 40-page report vs the 2 relevant pages~30,000 vs ~1,500 input tokens — then re-sent with every following message in that chat
Continuing a 30-turn chat vs starting fresh with a summaryEach new message carries the full history; a fresh chat with a 150-token summary cuts per-message cost by 90%+
"Try again" ×4 on a 2,000-token draft~8,000 wasted output tokens vs ~400 for one precise correction
Outline-first on long deliverables~150 tokens to approve an outline versus the full cost of a wrong-shaped draft redone from scratch

ลดการสูญเสีย Token ให้เหลือน้อยที่สุด

Token คือหน่วยวัดปริมาณของทุกสิ่งที่เราส่งไปและรับกลับจากเครื่องมือ AI การสูญเสียส่วนใหญ่เกิดจากนิสัยเพียงไม่กี่อย่าง — แก้ไขนิสัยเหล่านี้แล้วค่าใช้จ่ายของทั้งทีมจะลดลงอย่างชัดเจน โดยไม่เสียคุณภาพงานแต่อย่างใด

Token ถูกใช้ไปกับอะไรบ้าง

การสนทนาทุกครั้งจะส่งประวัติการสนทนาทั้งหมดซ้ำไปพร้อมกับข้อความใหม่ทุกข้อความ นั่นหมายความว่าแชทยาวที่มีเอกสารแนบสามฉบับจะส่งข้อมูลทั้งหมดซ้ำในทุกรอบการสนทนา สามสาเหตุหลักของการสูญเสียในงานวิจัยคือ:

1

ข้อมูลนำเข้าที่มากเกินจำเป็น

การวางเอกสารทั้งฉบับทั้งที่เกี่ยวข้องเพียงส่วนเดียว; การอัปโหลดไฟล์ที่ให้ไว้แล้วซ้ำอีกครั้ง; การสนทนาที่ยืดยาวจนลากประวัติทั้งหมดติดไปกับทุกข้อความ

2

การสร้างซ้ำวนไป

Prompt ที่คลุมเครือจนต้องลองถึง 5 ครั้ง; การพิมพ์ "ลองใหม่อีกครั้ง" แทนที่จะแก้ไขให้ตรงจุด; การสร้างเอกสารทั้งฉบับใหม่ทั้งที่ต้องแก้เพียงย่อหน้าเดียว

3

ใช้เครื่องมือระดับพรีเมียมกับงานเล็กน้อย

การใช้ Token ของ Claude กับงานที่เครื่องมือฟรีก็ให้ผลลัพธ์เหมือนกัน — เช่น การจัดรูปแบบทั่วไป การค้นหาข้อมูลง่าย ๆ หรือข้อความสำเร็จรูป

นิสัยเพื่อประสิทธิภาพในการใช้งาน

1. หนึ่งงาน หนึ่งแชท

เริ่มการสนทนาใหม่สำหรับแต่ละงานที่แยกจากกัน เมื่อแชทหนึ่งมีการโต้ตอบเกินประมาณสิบกว่าครั้ง ให้ขอให้ Claude สรุปสถานะของงานเป็นย่อหน้าสั้น ๆ คัดลอกไว้ แล้วเริ่มแชทใหม่พร้อมสรุปนั้น คุณจะพกพาไปเพียง 150 Token แทนที่จะเป็น 15,000 Token

2. ใช้ส่วนที่ตัดมา ไม่ใช่เอกสารทั้งฉบับ

สิ้นเปลือง

การวางเอกสารประเมินผลยาว 40 หน้าทั้งฉบับ แล้วถามว่า "เอกสารนี้พูดถึงสภาพแรงงานในจังหวัดทางภาคเหนืออย่างไร"

มีประสิทธิภาพ

ค้นหาสองส่วนที่เกี่ยวข้องด้วยตนเองก่อน (Ctrl+F) แล้ววางเฉพาะส่วนนั้น: ใช้ประมาณ 600 Token แทนที่จะเป็น 30,000 Token — และได้คำตอบที่ตรงประเด็นมากขึ้น

3. ให้ Cowork อ่านไฟล์จากเครื่องโดยตรง

ใน Cowork นั้น Claude จะอ่านไฟล์จากโฟลเดอร์งานโดยตรง และอ่านเฉพาะส่วนที่จำเป็นเท่านั้น สำหรับงานที่อ้างอิงไฟล์ทุกประเภท การเชื่อมต่อโฟลเดอร์ให้ Claude เข้าถึงย่อมดีกว่าการวางเนื้อหาไฟล์ลงใน Chat

4. เก็บบริบทที่คงที่ไว้ใน Projects ไม่ใช่ใน Prompt

Template วิธีการทำงาน และคำศัพท์เฉพาะ — เมื่อเขียนไว้ครั้งเดียวใน Project knowledge — จะไม่เสีย Token ซ้ำในทุกการสนทนาอีกต่อไป ดูเพิ่มเติมที่ Claude & Cowork

5. ระบุลักษณะผลลัพธ์ก่อนให้สร้างงาน

ระบุความยาว รูปแบบ และกลุ่มผู้อ่านไว้ใน prompt แรก: "500 คำ สามหัวข้อ สำหรับกลุ่มผู้บริจาค ไม่ใช้ศัพท์เทคนิค" ผลลัพธ์ที่แพงที่สุดคือร่างงานที่สวยงามแต่ยาวและผิดรูปแบบตั้งแต่ต้น

6. แก้ไขให้ตรงจุดเสมอ อย่าเพียงพิมพ์ว่า "ลองใหม่"

"ลองใหม่อีกครั้ง" จะสร้างทุกอย่างขึ้นใหม่ทั้งหมดและมักจะเข้าใจผิดซ้ำเดิม แทนที่จะทำเช่นนั้น: "คงโครงสร้างและน้ำเสียงเดิมไว้ แก้เฉพาะ: (1) ช่วงวันที่ควรเป็น 2023–2025 (2) ย่อบทนำให้เหลือสองประโยค"

7. รวมคำถามที่เกี่ยวข้องไว้ในครั้งเดียว

คำถามเล็ก ๆ ห้าข้อในข้อความเดียวมีต้นทุนต่ำกว่าการถามแยกกันห้าครั้ง เพราะการสนทนาแต่ละรอบจะส่งประวัติการสนทนาทั้งหมดซ้ำอีกครั้ง

8. ส่งงาน Tier 1 ไปยังเครื่องมือฟรี

งานทั่วไปที่เป็นข้อมูลสาธารณะ — เช่น การสรุปข่าว การร่างข้อความสำเร็จรูป หรือการอธิบาย error ของ Python — ควรส่งไปยัง DeepSeek, Kimi หรือเครื่องมือฟรีอื่น ๆ สงวนกำลังของ Claude ไว้สำหรับงานที่ต้องใช้บริบท วิจารณญาณ และการรักษาความลับของเรา ดูเพิ่มเติมที่ AI Toolkit

9. อย่าใช้ AI ทำงานที่ซอฟต์แวร์ทำได้ดีกว่า

การเรียงลำดับข้อมูลในสเปรดชีต การนับแถว หรือการค้นหาและแทนที่แบบตรงตัว — Excel และโปรแกรมแก้ไขข้อความของคุณทำสิ่งเหล่านี้ได้ทันทีและฟรี ใช้ AI เพื่อตัดสินใจว่าควรทำอะไร ไม่ใช่ใช้เป็นเครื่องคิดเลข

อ้างอิงด่วน: ต้นทุนของนิสัยที่พบบ่อย

นิสัยผลกระทบต่อ Token โดยประมาณ
การวางรายงานฉบับเต็ม 40 หน้า เทียบกับการวางเฉพาะ 2 หน้าที่เกี่ยวข้องประมาณ 30,000 เทียบกับ 1,500 input token — จากนั้นจะถูกส่งซ้ำในทุกข้อความถัดไปของแชทนั้น
การสนทนาต่อในแชทที่มี 30 รอบ เทียบกับการเริ่มแชทใหม่พร้อมบทสรุปทุกข้อความใหม่จะพกพาประวัติทั้งหมดไปด้วย ส่วนแชทใหม่ที่มีบทสรุปเพียง 150 Token จะลดต้นทุนต่อข้อความลงมากกว่า 90%
"ลองใหม่" 4 ครั้งกับร่างงานความยาว 2,000 Tokenสูญเสีย output token ประมาณ 8,000 เทียบกับประมาณ 400 สำหรับการแก้ไขที่ตรงจุดเพียงครั้งเดียว
ทำ Outline ก่อนสำหรับงานที่ต้องส่งยาวใช้ประมาณ 150 Token เพื่ออนุมัติ Outline เทียบกับต้นทุนทั้งหมดของการทำร่างงานผิดรูปแบบใหม่ทั้งหมดตั้งแต่ต้น

အပိုင်း 2

Token အလေအလွင့် လျှော့ချခြင်း

Token သည် AI ကိရိယာများသို့ ပို့သည့်နှင့် ရရှိသည့် အရာအားလုံး၏ တိုင်းတာနိုင်သော ယူနစ်ဖြစ်သည်။ အလေအလွင့်အများစုသည် အလေ့အထအနည်းငယ်ကြောင့်ဖြစ်သည် — ၎င်းတို့ကို ပြင်ဆင်ပါက အသင်းလုံး၏ အသုံးစရိတ်သည် အရည်အသွေး မထိခိုက်ဘဲ သိသိသာသာ ကျဆင်းသွားမည်ဖြစ်သည်။

Token များ အမှန်တကယ် ဘယ်ကိုသွားသနည်း

စကားဝိုင်းတိုင်းသည် စကားစမြည်သမိုင်းကြောင်းတစ်ခုလုံးကို မက်ဆေ့ချ်အသစ်တိုင်းနှင့်အတူ ထပ်မံပို့ပေးသည်။ ဆိုလိုသည်မှာ စာတမ်းသုံးခုကို ကူးထည့်ထားသော စကားဝိုင်းရှည်တစ်ခုသည် ၎င်းအားလုံးကို အလှည့်တိုင်းတွင် ထပ်ပို့နေသည်ဟု ဆိုလိုသည်။ သုတေသနအလုပ်တွင် အလေအလွင့်၏ အကြီးဆုံးအကြောင်းရင်းသုံးချက်မှာ —

1

လိုအပ်သည်ထက် ပိုကြီးသော input

သက်ဆိုင်သည့်အပိုင်းတစ်ခုသာရှိသော်လည်း စာတမ်းတစ်ခုလုံးကို ကူးထည့်ခြင်း; ပေးပြီးသား ဖိုင်များကို ထပ်မံ တင်ခြင်း; မက်ဆေ့ချ်တိုင်းနောက်သို့ သမိုင်းကြောင်းတစ်ခုလုံး ဆွဲလာသော စကားဝိုင်းရှည်များ

2

ထပ်ခါထပ်ခါ ပြန်ဆွဲခြင်း

ကြိုးစားမှု 5 ကြိမ်လိုအပ်သော ရေရေရာရာမဲ့သည့် prompt များ; တိကျသော ပြင်ဆင်မှုအစား "ထပ်ကြိုးစားပါ" ဟု ပြောခြင်း; စာပိုဒ်တစ်ခုတည်းကို ပြင်ရန် စာတမ်းတစ်ခုလုံးကို ပြန်ဆွဲခြင်း

3

သေးငယ်သောအလုပ်တွင် Premium ကိရိယာအသုံးပြုခြင်း

အခမဲ့ကိရိယာတစ်ခုနှင့် ရလဒ်တူညီစွာရနိုင်သော အလုပ်များအတွက် Claude Token သုံးခြင်း — ဥပမာ ယေဘုယျပုံစံချခြင်း၊ ရိုးရှင်းသော ရှာဖွေမှုများ၊ boilerplate စာသားများ

ထိရောက်စွာ အသုံးပြုရန် အလေ့အထများ

1. တစ်လုပ်ငန်း၊ တစ်စကားဝိုင်း

သီးခြားလုပ်ငန်းတစ်ခုစီအတွက် စကားဝိုင်းအသစ်ကို စတင်ပါ။ စကားဝိုင်းတစ်ခုသည် အလှည့်တစ်ဒါဇင်ခန့်ကျော်လာသောအခါ Claude ကို အလုပ်အခြေအနေကို စာပိုဒ်တိုတစ်ခုဖြင့် အနှစ်ချုပ်ခိုင်းပြီး ကူးယူထားပါ၊ ထို့နောက် ထိုအနှစ်ချုပ်ဖြင့် စကားဝိုင်းအသစ်တွင် ဆက်လုပ်ပါ။ ဤနည်းဖြင့် Token 15,000 အစား 150 ကိုသာ ရှေ့ဆက်ယူဆောင်သွားရသည်။

2. စာတမ်းတစ်ခုလုံးမဟုတ်ဘဲ သက်ဆိုင်ရာအပိုင်းသာ

အလေအလွင့်

စာမျက်နှာ 40 ရှိသော အကဲဖြတ်စာတမ်းတစ်ခုလုံးကို ကူးထည့်ပြီး "ဤစာတမ်းက မြောက်ပိုင်းတိုင်းများရှိ လုပ်သားအခြေအနေများအကြောင်း ဘာပြောထားသနည်း" ဟု မေးခြင်း

ထိရောက်မှု

သက်ဆိုင်ရာ အပိုင်းနှစ်ပိုင်းကို ကိုယ်တိုင်ရှာပြီး (Ctrl+F) ၎င်းတို့ကိုသာ ကူးထည့်ခြင်း: Token 30,000 အစား 600 ခန့်သာ သုံးရပြီး — ပိုမိုတိကျသော အဖြေရရှိသည်

3. Cowork ကို ဖိုင်များကို ကွန်ပျူတာမှ တိုက်ရိုက် ဖတ်ခိုင်းပါ

Cowork တွင် Claude သည် task folder မှ ဖိုင်များကို တိုက်ရိုက်ဖတ်ပြီး လိုအပ်သည့်အပိုင်းကိုသာ ဖတ်သည်။ ဖိုင်အခြေခံအလုပ် မှန်သမျှအတွက် folder ချိတ်ဆက်ခြင်းသည် Chat ထဲသို့ အကြောင်းအရာ ကူးထည့်ခြင်းထက် ပိုကောင်းသည်။

4. ပုံမှန်ရှိနေမည့် context ကို Prompt တွင်မဟုတ်ဘဲ Projects တွင် သိမ်းထားပါ

Template များ၊ လုပ်ဆောင်နည်း၊ အသုံးအနှုန်းများ — Project knowledge တွင် တစ်ကြိမ်တည်း ရေးထည့်ထားခြင်းဖြင့် — စကားဝိုင်းတိုင်းတွင် Token ထပ်မံကုန်ကျမှု ရပ်တန့်သွားသည်။ Claude & Cowork ကို ကြည့်ပါ

5. ရလဒ်မထုတ်မီ ပုံစံသတ်မှတ်ပါ

ပထမဆုံး prompt တွင် အရှည်၊ ပုံစံနှင့် ပရိသတ်ကို ဖော်ပြပါ — "စကားလုံး 500၊ အပိုင်းသုံးပိုင်း၊ လှူဒါန်းသူ ပရိသတ်အတွက်၊ ဗန်းစကားမပါ" ဟူ၍။ အကုန်ကျဆုံးရလဒ်မှာ ပုံစံမှားနေသော လှပသည့် ရေရှည်မူကြမ်းတစ်ခုပင်ဖြစ်သည်။

6. "ထပ်ကြိုးစားပါ" ဟု မဆိုဘဲ တိကျစွာ ပြင်ဆင်ပါ

"ထပ်ကြိုးစားပါ" ဆိုသည်မှာ အားလုံးကို ပြန်ဆွဲစေပြီး နားလည်မှုမှားယွင်းချက်ကို ထပ်ခါထပ်ခါ ဖြစ်စေတတ်သည်။ ယင်းအစား — "ဖွဲ့စည်းပုံနှင့် အသံအနေအထားကို ဆက်ထားပါ။ ဤအချက်များသာ ပြင်ပါ: (1) ရက်စွဲကာလ 2023–2025 ဖြစ်သင့်သည်၊ (2) နိဒါန်းကို စာကြောင်းနှစ်ကြောင်းအထိ တိုစေပါ" ဟု ပြောပါ

7. ဆက်စပ်သော မေးခွန်းများကို စုပြီး တစ်ခါတည်းမေးပါ

မက်ဆေ့ချ်တစ်ခုတည်းတွင် သေးငယ်သော မေးခွန်းငါးခု ထည့်သွင်းမေးခြင်းသည် သီးခြားစီ ငါးကြိမ် မေးခြင်းထက် ကုန်ကျစရိတ် နည်းသည်၊ အကြောင်းမှာ သီးခြားစီ မေးမြန်းမှုတိုင်းသည် စကားဝိုင်း သမိုင်းကြောင်းတစ်ခုလုံးကို ထပ်မံပို့နေသောကြောင့် ဖြစ်သည်။

8. Tier 1 အလုပ်များကို အခမဲ့ကိရိယာများသို့ ပို့ပါ

သတင်းဆောင်းပါးတစ်ခု အနှစ်ချုပ်ခြင်း၊ boilerplate ရေးဆွဲခြင်း၊ Python error ရှင်းပြခြင်း ကဲ့သို့ အများပြည်သူဆိုင်ရာ ယေဘုယျအလုပ်များကို DeepSeek၊ Kimi သို့မဟုတ် အခြားအခမဲ့ကိရိယာများသို့ ပို့ပါ။ ကျွန်ုပ်တို့၏ context၊ ဆင်ခြင်တုံတရား နှင့် လျှို့ဝှက်ထိန်းသိမ်းမှု လိုအပ်သည့် အလုပ်များအတွက် Claude ၏ စွမ်းဆောင်ရည်ကို သိမ်းထားပါ။ AI Toolkit ကို ကြည့်ပါ

9. Software ပိုကောင်းစွာ လုပ်နိုင်သည့်အရာအတွက် AI ကို မသုံးပါနှင့်

Spreadsheet စီစဉ်ခြင်း၊ row အရေအတွက် ရေတွက်ခြင်း၊ တိကျသော find-and-replace — Excel နှင့် သင့် editor သည် ဤအရာများကို ချက်ချင်းနှင့် အခမဲ့ လုပ်ဆောင်ပေးနိုင်သည်။ AI ကို ဘာလုပ်ရမည်ကို ဆုံးဖြတ်ရန် သုံးပါ၊ calculator အဖြစ် မသုံးပါနှင့်

လျင်မြန်သော ကိုးကားချက်: အလေ့အထ အများစု၏ ကုန်ကျစရိတ်

အလေ့အထခန့်မှန်း Token အကျိုးသက်ရောက်မှု
စာမျက်နှာ 40 ပါ အစီရင်ခံစာအပြည့် ကူးထည့်ခြင်း နှင့် သက်ဆိုင်ရာ စာမျက်နှာ 2 ချက်သာ ကူးထည့်ခြင်း နှိုင်းယှဉ်မှုinput token ခန့်မှန်း 30,000 နှင့် 1,500 — ထို့နောက် ထိုစကားဝိုင်း၏ နောက်ဆက်တွဲ မက်ဆေ့ချ်တိုင်းတွင် ထပ်မံပို့ခြင်းခံရသည်
အလှည့် 30 ရှိသော စကားဝိုင်းကို ဆက်လက်လုပ်ဆောင်ခြင်း နှင့် အနှစ်ချုပ်ဖြင့် စကားဝိုင်းအသစ်စတင်ခြင်း နှိုင်းယှဉ်မှုမက်ဆေ့ချ်အသစ်တိုင်းသည် သမိုင်းကြောင်းအပြည့်ကို သယ်ဆောင်သည်; Token 150 ရှိသော အနှစ်ချုပ်ဖြင့် စကားဝိုင်းအသစ်သည် မက်ဆေ့ချ်တစ်ခုချင်းစီ၏ ကုန်ကျစရိတ်ကို 90%+ လျှော့ချပေးသည်
"ထပ်ကြိုးစားပါ" ကို Token 2,000 ရှိသော မူကြမ်းတွင် 4 ကြိမ်သုံးခြင်းoutput token ခန့်မှန်း 8,000 ဆုံးရှုံးမှုကို တိကျသော ပြင်ဆင်မှုတစ်ကြိမ်အတွက် 400 ခန့်နှင့် နှိုင်းယှဉ်ပါက
