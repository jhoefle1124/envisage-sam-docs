AI Tools Directory

Every major AI coding agent, IDE, extension, and app builder — with a plain-language explanation of when to choose each one over the alternatives.

How to use this directory: Each tool includes a link and a single paragraph explaining its primary differentiator. Use this to decide which tool fits your specific task — not every tool is right for every job. Always apply ENVISAGE data tier rules before connecting any tool to files or databases.

⌨️ Terminal & CLI Coding Agents

Run from the command line. Developer-oriented. Give explicit control over which files and directories the AI can access.

CLI Agent

Claude Code — Anthropic

Best-in-class for reasoning-heavy, multi-step refactors and complex coding tasks. Choose this when you're already in the Anthropic ecosystem and need the most capable model for nuanced, multi-file work. Confirmed compatible with SAM.

CLI Agent

OpenAI Codex CLI — OpenAI

Rust-native with a cloud sandbox option that keeps code execution off your local machine. Choose this when you want OpenAI models in a fast CLI with an optional isolated execution environment. Confirmed compatible with SAM.

CLI Agent

Gemini CLI — Google

The only free CLI agent with a 1M-token context window. Choose this when you need to load an entire large codebase or long research document into a single session without paying per token. Open-source and auditable.

CLI Agent

GitHub Copilot CLI — GitHub

Best when your team already has GitHub Enterprise licensing. Integrates naturally with your repo, pull requests, and CI workflows without additional setup — no new account or billing required if Copilot is already active.

CLI Agent

Sourcegraph Amp — Sourcegraph

Composable sub-agent system (Oracle for search, Librarian for context, Painter for generation). Choose this for large, complex codebases where different tasks need specialised agents rather than one general-purpose model.

CLI Agent

Mistral Vibe — Mistral

EU-hosted, open-weight model option. Choose this when your team has European data residency requirements or prefers a non-US AI provider with full transparency into model weights.

CLI Agent

Rovo Dev — Atlassian

Best when your team runs on Jira and Confluence. Coding context connects directly to your project management and documentation ecosystem — tasks, tickets, and specs inform the AI's understanding of your work automatically.

CLI Agent

Tabnine CLI — Tabnine

Docker-distributed, enterprise-licensed, fully self-hostable. Choose this for environments where no code can leave your infrastructure — models run locally and data never touches external servers. Strongest privacy guarantee in this category.

CLI Agent

Mentat

Cloud-native managed agent — no local infrastructure to maintain. Choose this when you want a capable coding agent without the DevOps overhead of running models locally or managing CLI toolchains yourself.

CLI Agent

Aider

The gold standard for git-native pair programming. Every AI change is committed automatically with a clean message, giving you a full rollback-friendly history of every AI edit. Best choice when auditability of changes matters.

CLI Agent

OpenCode

Go-based TUI supporting 75+ AI providers. The most portable CLI option — switch between Claude, Gemini, Mistral, and local models with a config change. Confirmed compatible with SAM. Best choice when you must avoid vendor lock-in at the CLI level.

CLI Agent

Goose — Block

YAML "Recipes" encode repeatable workflows as reusable scripts. Choose this when you want to standardise how your team approaches common coding tasks — a Recipe runs the same workflow consistently every time.

CLI Agent

Plandex

2M-token context with a diff sandbox. Best for very large-scale, multi-file refactors — you plan the entire change set first, review diffs, then apply. The diff sandbox prevents runaway changes from reaching your actual codebase.

CLI Agent

gptme

Persistent autonomous agent with git-backed memory across sessions. Choose this when you need an agent that remembers prior conversations, decisions, and context from previous work sessions — ideal for long-running research or development projects.

CLI Agent

Warp

AI built into the terminal itself rather than bolted on as a plugin. Choose this when you want the shell experience to be AI-native — command suggestions, error explanations, and agent execution all happen inside a modern terminal UI.

CLI Agent

SWE-agent — Princeton

Research-grade agent built specifically for automated bug fixing on GitHub issues. Best for teams who want to automate issue resolution — feed it a GitHub issue URL and it attempts to write and test the fix autonomously.

CLI Agent

Smol Developer

Designed to scaffold an entire new codebase from a high-level spec in one shot. Choose this for greenfield projects where you want a complete project structure generated from a prompt — not for iterating on existing code.

CLI Agent

ForgeCode

Supports 300+ AI models in a single CLI. Choose this when your team uses many different models for different tasks — route heavy reasoning to Claude, fast tasks to a smaller model, and specialist queries to domain-specific models, all from one tool.

CLI Agent

Codebuff

Multi-agent collaborative CLI — multiple AI agents work the same codebase simultaneously. Choose this over single-agent tools when tasks can be parallelised, such as writing tests while another agent writes the implementation.

CLI Agent

Trae Agent — ByteDance

ByteDance's terminal-focused coding agent, distinct from their Trae IDE. Choose this when working within ByteDance's AI ecosystem or when you want a capable free-tier option from a major AI lab with strong coding benchmarks.

CLI Agent

Kimi CLI — Moonshot

Strong long-context capabilities from Moonshot AI. Choose this for tasks involving very long documents or large codebases where context window size is the limiting factor. Use Tier 1 public data only per ENVISAGE policy.

CLI Agent

Qwen Code — Alibaba

Alibaba's coding-tuned model as a CLI agent. Choose this when working with Chinese-market data, needing a non-Western AI provider, or running comparative checks against Western models to identify bias in output. Tier 1 data only.

🖥 AI-Native IDEs

Full development environments rebuilt around AI. Replace or supplement VS Code, JetBrains, etc.

AI IDE

Cursor

The market leader for AI-native development with the most mature feature set and largest community. Choose this as your primary switch from VS Code — it has the best balance of AI capability, familiarity, and third-party plugin support.

AI IDE

Windsurf — OpenAI-acquired

Formerly Codeium, now owned by OpenAI. Choose this for deep integration with GPT-4 and o-series models in a polished IDE — best if your team is invested in OpenAI's ecosystem and wants the tightest model-IDE pairing available.

AI IDE

Zed

Rust-built for raw speed with real-time collaboration and bring-your-own-model (BYOM) support. Choose this when latency is critical or when you need real-time multiplayer editing. The fastest editor in this list by a significant margin.

AI IDE

Kiro — AWS

Spec-driven development — requirements are defined first, code is generated from them. Choose this when you want tighter alignment between documentation and implementation, or when working within AWS infrastructure where deep service integration matters.

AI IDE

Trae — ByteDance

Fully free AI IDE with adaptive mode that adjusts assistance based on your experience level. Best free alternative to Cursor — choose this when budget is a constraint or when onboarding less-experienced developers who benefit from adaptive guidance.

AI IDE

Replit

Browser-based — zero local setup required. Choose this for rapid prototyping, teaching, or any scenario where you need a full development environment accessible from any device without installation. Includes Replit Agent for autonomous full-stack building.

AI IDE

PearAI

Open-source Cursor alternative with community governance. Choose this when transparency and auditability of the IDE itself matter — the full source is inspectable, and the community steers development rather than a commercial entity.

AI IDE

BLACKBOX

Broad language and framework support with a focus on speed and simplicity. Choose this as a lightweight entry point to AI-assisted development, particularly for teams that find Cursor or Windsurf overly complex for straightforward tasks.

AI IDE

MarsCode — ByteDance

ByteDance's AI IDE built for the Chinese developer market. Choose this when your team operates primarily in the Chinese AI ecosystem or when you need strong support for frameworks and libraries popular in China-based development environments.

🔌 IDE Extensions & Plugins

Add AI to your existing editor — VS Code, JetBrains, and others. Lowest disruption path to AI-assisted development.

Extension

GitHub Copilot — GitHub/Microsoft

The default enterprise choice if your organisation is on GitHub. Now multi-model with Claude and Codex backends. Choose this when your team needs AI assistance that integrates directly with existing GitHub repos, PRs, and code review workflows.

Extension

Cline

Best VS Code extension for complex agentic tasks. Its Plan/Act architecture separates planning from execution — the AI lays out what it will do before touching any code, letting you review and approve the plan first. BYOM with strong Claude integration.

Extension

Roo Code

Cline fork with multiple operating modes (Code, Architect, Ask, Debug). Choose Roo over Cline when you need the same agentic power but want to switch context — ask questions without triggering code changes, or plan architecture without writing a line.

Extension

Kilo Code

Adds an Orchestrator mode that routes sub-tasks to specialised agents. Choose this for complex multi-step workflows in VS Code where different parts of a task — search, write, test, document — benefit from different models or agent behaviours.

Extension

Continue

Open-source with a pivot toward AI-assisted CI checks. Choose this when you want full control over model selection, transparent source code, and are building toward AI integration in your continuous integration pipeline rather than just interactive coding.

Extension

Augment Code

Purpose-built for very large codebases — 100K+ files. Choose this over other extensions when your project is too large for standard context windows and you need an AI that can genuinely understand the full scope of an enterprise-scale repository.

Extension

Amazon Q Developer — AWS

Deepest integration with AWS services — CloudFormation, Lambda, CDK, and the full AWS ecosystem. Choose this when your infrastructure is on AWS and you want an AI that understands your cloud architecture, not just your code.

Extension

Gemini Code Assist — Google

Best for teams on Google Cloud. Integrates with GCP services, BigQuery, and Firebase, and benefits from Google's long-context models. Choose this when your data and infrastructure live in the Google ecosystem.

Extension

JetBrains Junie — JetBrains

Native to IntelliJ, PyCharm, GoLand, and other JetBrains IDEs. Choose this if your team is invested in the JetBrains ecosystem and doesn't want to switch editors — it provides deep AI integration without losing JetBrains' powerful refactoring and inspection tooling.

Extension

Tabnine

Privacy-first, self-hostable, fine-tuneable on your own codebase. Choose this for regulated industries or security-sensitive environments where no code can be transmitted externally — run the model on your own servers, train it on your own code.

Extension

Sourcegraph Cody

Grounded in Sourcegraph's code-search index rather than raw model memory. Choose this when your codebase is too large for any context window — Cody searches the code rather than loading it, so it always works at any scale.

Extension

Supermaven

Fastest autocomplete in any extension tested, with a 1M-token context window. Choose this specifically for inline completion speed — if you find Copilot or Codeium suggestions lag behind your typing, Supermaven is the fix.

Extension

Bito AI

Specialises in code explanation and test generation rather than general completion. Use Bito alongside your primary coding assistant — not as a replacement — when you need to understand unfamiliar code or rapidly generate test coverage for existing functions.

Extension

Phind

Combines web search with code generation — answers technical questions with cited sources, then writes the code. Choose this when you need to look something up and implement it in the same step, rather than switching between a browser and your editor.

Extension

CodeGPT

Multi-provider extension for VS Code and JetBrains — switch between OpenAI, Anthropic, local models, and others from the same interface. Choose this when your team uses different models for different tasks and needs one consistent UI across all of them.

Extension

Refact.ai

Open-source, self-hostable, focused on refactoring and code quality rather than generation volume. Choose this when your priority is improving existing code rather than generating new code — it analyses, restructures, and improves rather than just autocompleting.

Extension

Tabby

Fully self-hosted, open-source AI assistant. The best choice for air-gapped environments or teams with absolute data sovereignty requirements — the model runs on your infrastructure and zero data leaves your network.

Extension

GitLab Duo — GitLab

AI across the full DevSecOps lifecycle — code, review, security scanning, and deployment. Choose this if your team's DevOps runs on GitLab and you want AI assistance that spans the entire development pipeline, not just the editor.

Extension

Jupyter AI

The only AI assistant built natively for Jupyter notebooks. Choose this for data science, research, or any analysis workflow in Jupyter — it understands notebook structure, cell outputs, and data context in a way no general-purpose coding assistant does.

Extension

Stagewise

Browser-aware bridge between your live app and your coding tools. Unique in this list — you select a UI element directly in Chrome, click "fix this," and it sends the exact context to Cursor, Copilot, Cline, or Windsurf. Use it to eliminate the copy-paste step in frontend debugging.

Extension

CodeGeeX — Zhipu AI

Open-source with support for 600+ programming languages — the broadest language coverage in this category. Choose this for polyglot teams or when working with niche, domain-specific, or lesser-known programming languages that other tools handle poorly.

✨ Vibe-Coding & Prompt-to-App Builders

Describe what you want in plain language — the AI builds the app. No coding experience required for basic use. Best for prototyping and non-developer staff.

Vibe-Coding

Lovable

The most mature and highest-quality full-stack vibe-coding platform ($6.6B valuation). Choose this as your default when a non-developer needs to build a real, deployable web application from a chat description — it produces the most production-ready output in this category.

Vibe-Coding

Bolt.new — StackBlitz

Instant browser-based app building with no local setup. Choose this for the fastest path from idea to working prototype — open a tab, describe what you want, and have a running app in minutes. Best for rapid iteration and early-stage exploration.

Vibe-Coding

v0 — Vercel

Best specifically for React and Next.js UI generation. Produces production-ready shadcn/ui components that drop directly into Vercel-hosted projects. Choose this when the deliverable is a React component or page, not a full application.

Vibe-Coding

Replit Agent — Replit

Vibe-coding plus hosting in one place. Choose this when you want the AI to build and deploy the app without leaving a single platform — no separate hosting setup, no export step, no deployment pipeline to configure.

Vibe-Coding

Tempo

Visual React editor where you interact with rendered components while the AI generates the underlying code. Choose this when you want to see and manipulate the UI visually rather than working from code or text descriptions alone.

Vibe-Coding

Magic Patterns

Generates multiple UI variants simultaneously on a shared canvas. Choose this for rapid design exploration when you want to compare four or five different visual approaches to the same feature before committing to one direction.

Vibe-Coding

Figma Make — Figma

Adds interactivity and code to existing Figma designs without leaving the Figma environment. Choose this when you're a designer who already works in Figma and wants to make designs interactive or generate component code without switching to a separate tool.

Vibe-Coding

Hostinger Horizons

Prompt-to-app plus instant hosting as a single integrated workflow. Choose this when you need a simple public-facing app or landing page deployed quickly — the hosting is built in and there is no separate deployment step or server to configure.

Vibe-Coding

Taskade Genesis

Builds internal business apps — CRMs, dashboards, project trackers — rather than consumer-facing products. Choose this when the deliverable is an operational tool for internal team use rather than a public-facing website or application.

Vibe-Coding

A0.dev

Mobile-first app generation. Choose this when the target platform is iOS or Android rather than the web — most vibe-coding tools default to browser apps, making A0.dev a rare specialist for native mobile prototyping from a text description.

Vibe-Coding

Canva Code — Canva

Designed for non-developers already inside Canva who want to add simple interactive elements to their designs. Choose this when your team lives in Canva and the ask is a small interactive widget rather than a full application.

Vibe-Coding

Emergent

India-based vibe-coding platform with regional pricing and support. Choose this for teams in the South Asian market who want local support, regional compliance, or simply a provider whose pricing is calibrated for that market.

Reminder — ENVISAGE Data Rules Apply to All Tools Above: Before connecting any tool to files, databases, or cloud storage, check which data tier applies. Tools from non-Western providers (DeepSeek, Kimi, Qwen Code, Kimi For Coding, CodeGeeX, Tongyi Lingma, Baidu Comate, MarsCode) must be used with Tier 1 publicly available data only. ไดเร็กทอรีเครื่องมือ AI

ตัวแทน AI สำหรับการเขียนโค้ด, IDE, ส่วนขยาย และตัวสร้างแอปทุกตัว — พร้อมคำอธิบายภาษาธรรมดาว่าเมื่อใดควรเลือกแต่ละตัวแทนทางเลือกอื่น

วิธีใช้ไดเร็กทอรีนี้: แต่ละเครื่องมือมีลิงก์และย่อหน้าเดียวที่อธิบายสิ่งที่แตกต่างหลักของมัน ใช้สิ่งนี้เพื่อตัดสินใจว่าเครื่องมือใดเหมาะกับงานเฉพาะของคุณ ปฏิบัติตามกฎระดับข้อมูลของ ENVISAGE ก่อนเชื่อมต่อเครื่องมือใดกับไฟล์หรือฐานข้อมูล

⌨️ ตัวแทน CLI สำหรับเขียนโค้ด

ทำงานจากบรรทัดคำสั่ง เหมาะสำหรับนักพัฒนา ให้การควบคุมที่ชัดเจนว่าไฟล์และไดเร็กทอรีใดที่ AI สามารถเข้าถึงได้

AI ကိရိယာလမ်းညွှန်

ပြည့်စုံသော AI coding agent, IDE, extension နှင့် app builder များ — မည်သည့်အချိန်တွင် တစ်ခုကို အခြားတစ်ခုအပေါ် ရွေးချယ်ရမည်ကို ရိုးရှင်းသောဘာသာစကားဖြင့် ရှင်းပြချက်ဖြင့်

ဤလမ်းညွှန်ကို မည်သို့သုံးမည်နည်း: ကိရိယာတစ်ခုချင်းစီတွင် လင့်ခ်နှင့် ၎င်း၏ အဓိကကွဲပြားချက်ကို ရှင်းပြသောပိုဒ်ကြီးတစ်ပိုဒ် ပါဝင်သည်။ ကိုင်တွယ်ဆောင်ရွက်ရေးကိရိယာ သို့မဟုတ် ဒေတာဘေ့စ်နှင့် မချိတ်ဆက်မီ ENVISAGE ဒေတာ tier စည်းကမ်းများ အမြဲတမ်း လိုက်နာပါ။
