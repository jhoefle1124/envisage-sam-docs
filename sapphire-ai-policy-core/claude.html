Claude & Claude Cowork Best Practices

Claude is our primary AI workhorse. Use Chat for thinking and Cowork for doing — and set up each research stream once so you never pay to re-explain context.

Chat vs Cowork: Pick the Right Mode

Claude Chat

For thinking, drafting, and analysis

  • Reasoning through a research question or analytical framework
  • Drafting and editing report sections, summaries, briefings
  • Reviewing translations from Burmese, Thai, Shan, and other languages
  • Interpreting patterns in data you describe or paste as excerpts
  • Structuring interview notes or field observations (anonymized)
Claude Cowork

For working directly on files

  • Building deliverables: Word reports, Excel workbooks, slide decks
  • Cleaning, restructuring, and converting data files
  • Multi-step tasks: "process these 12 CSVs and produce a summary table"
  • Repetitive document work across many files in a project folder
  • Writing and testing scripts (Python, SQL) against sample files

Rule of thumb: if the answer belongs in a file, start in Cowork. If the answer belongs in your head, start in Chat. Pasting file contents into Chat wastes tokens twice — once to paste, once when Claude repeats it back.

Set Up One Project per Research Stream

Claude Projects hold persistent context so you stop paying to re-explain it in every conversation.

Create a Project for each long-running engagement or workstream — for example: a monitoring-and-analysis engagement, a supply-chain risk study, SAM product development, or a mapping project. In each Project's knowledge/instructions, store the stable context once:

  • The engagement's purpose, scope, and client-approved terminology (anonymized to the level the tier rules allow)
  • Our methodology notes — e.g., how we verify sources, our scoring rubric, our visualization standards
  • Output templates: report structure, citation style, standard caveats
  • Schema descriptions for relevant datasets (structure only, no data rows)
Good — context lives in the Project

Project instructions contain the report template and methodology. Each new chat: "Draft the Q2 trend section using the attached excerpt." ~200 tokens of prompt.

Wasteful — context re-pasted every time

Every chat opens with 3,000 words of background, the full template, and last quarter's report pasted in before the actual request.

Cowork Folder Discipline

Cowork works on folders you explicitly connect. Treat that connection like handing over a filing cabinet:

  • One task folder per engagement. Create a dedicated working folder containing only the files the task needs. Connect that — never a parent folder, your Desktop, or your whole Drive sync folder.
  • No client deliverable folders. Final confidential deliverables live in their own protected location that is never connected to Cowork. Work on drafts in the task folder; move finals out when done.
  • Strip before you connect. Before connecting a folder, scan it: any file naming sources, field staff, or persons of interest that the task doesn't require gets moved out first.
  • Review what Cowork produced before moving files onward. You are the quality gate, exactly as with a junior analyst's work.

Connector Rules

ConnectorStatusWhy
Email (Gmail / Outlook)ProhibitedEmail holds client correspondence, contracts, and source contacts. No AI tool connects to ENVISAGE email, ever.

Prompting Patterns for Research Work

Give the role, the source, the output shape — once

You are assisting an ENVISAGE analyst on a supply-chain study.
Source: the excerpt below (public news reporting, translated).
Task: extract company names, locations, and alleged practice
into a 3-column table. Flag low-confidence extractions.

Make Claude ask before generating long output

For big deliverables, add: "Before drafting, list your assumptions and an outline in under 150 words, and wait for my go-ahead." A wrong assumption caught at the outline stage costs 150 tokens to fix; the same mistake caught after a full draft costs the whole draft.

แนวปฏิบัติที่ดีที่สุดของ Claude & Claude Cowork

Claude คือเครื่องมือ AI หลักของเรา ใช้ Chat สำหรับการคิดวิเคราะห์ และใช้ Cowork สำหรับการลงมือทำ — และตั้งค่าแต่ละสายงานวิจัยไว้เพียงครั้งเดียว เพื่อไม่ต้องเสีย token ไปกับการอธิบายบริบทซ้ำทุกครั้ง

Chat กับ Cowork: เลือกโหมดให้ถูกต้อง

Claude Chat

สำหรับการคิด ร่างงาน และวิเคราะห์

  • คิดวิเคราะห์คำถามงานวิจัยหรือกรอบการวิเคราะห์
  • ร่างและแก้ไขส่วนต่าง ๆ ของรายงาน บทสรุป และ briefing
  • ตรวจทานคำแปลจากภาษาพม่า ไทย ไทใหญ่ และภาษาอื่น ๆ
  • ตีความรูปแบบข้อมูลที่คุณอธิบายหรือวางเป็น excerpt
  • จัดโครงสร้างบันทึกการสัมภาษณ์หรือข้อสังเกตภาคสนาม (ที่ทำให้ไม่ระบุตัวตนแล้ว)
Claude Cowork

สำหรับการทำงานกับไฟล์โดยตรง

  • สร้าง deliverable: รายงาน Word, Excel workbook, slide deck
  • ทำความสะอาด จัดโครงสร้างใหม่ และแปลงไฟล์ข้อมูล
  • งานหลายขั้นตอน เช่น "ประมวลผลไฟล์ CSV ทั้ง 12 ไฟล์นี้ แล้วสร้างตารางสรุป"
  • งานเอกสารที่ทำซ้ำ ๆ ในไฟล์จำนวนมากภายในโฟลเดอร์โครงการ
  • เขียนและทดสอบสคริปต์ (Python, SQL) กับไฟล์ตัวอย่าง

หลักง่าย ๆ ในการเลือกใช้: ถ้าคำตอบควรอยู่ในไฟล์ ให้เริ่มที่ Cowork ถ้าคำตอบควรอยู่ในความคิดของคุณ ให้เริ่มที่ Chat การวางเนื้อหาไฟล์ลงใน Chat เป็นการสิ้นเปลือง token สองต่อ — ครั้งแรกตอนวาง และอีกครั้งเมื่อ Claude พิมพ์เนื้อหานั้นกลับมาซ้ำ

ตั้งค่าหนึ่ง Project ต่อหนึ่งสายงานวิจัย

Claude Projects เก็บบริบทไว้อย่างต่อเนื่อง เพื่อให้คุณไม่ต้องเสีย token อธิบายซ้ำในทุกบทสนทนา

สร้าง Project หนึ่งรายการสำหรับแต่ละงานหรือสายงานที่ดำเนินต่อเนื่องระยะยาว เช่น งาน monitoring-and-analysis งานศึกษาความเสี่ยง supply-chain การพัฒนาผลิตภัณฑ์ SAM หรือโครงการทำแผนที่ ในส่วน knowledge/instructions ของแต่ละ Project ให้บันทึกบริบทที่คงที่ไว้เพียงครั้งเดียว:

  • วัตถุประสงค์ ขอบเขต และคำศัพท์เฉพาะที่ลูกค้าอนุมัติแล้วของงาน (ทำให้ไม่ระบุตัวตนตามระดับที่กฎ tier อนุญาต)
  • บันทึก methodology ของเรา เช่น วิธีตรวจสอบแหล่งข่าว เกณฑ์การให้คะแนน มาตรฐานการแสดงผลข้อมูล
  • template ผลลัพธ์: โครงสร้างรายงาน รูปแบบการอ้างอิง ข้อควรระวังมาตรฐาน
  • คำอธิบาย schema ของชุดข้อมูลที่เกี่ยวข้อง (เฉพาะโครงสร้าง ไม่มีแถวข้อมูลจริง)
ดี — บริบทอยู่ใน Project

คำสั่งของ Project มี template รายงานและ methodology อยู่แล้ว ทุกบทสนทนาใหม่เพียงพิมพ์ว่า: "ร่างส่วนแนวโน้มไตรมาส 2 โดยใช้ excerpt ที่แนบมา" ใช้ prompt เพียงประมาณ 200 token

สิ้นเปลือง — วางบริบทซ้ำทุกครั้ง

ทุกบทสนทนาเริ่มต้นด้วยการวางเนื้อหาพื้นหลังยาว 3,000 คำ template ฉบับเต็ม และรายงานไตรมาสก่อนหน้า ก่อนจะถึงคำขอจริง

วินัยการใช้โฟลเดอร์ใน Cowork

Cowork ทำงานกับโฟลเดอร์ที่คุณเชื่อมต่อไว้อย่างชัดเจนเท่านั้น ควรปฏิบัติต่อการเชื่อมต่อนั้นเหมือนกับการส่งมอบตู้เก็บเอกสารทั้งตู้:

  • หนึ่งโฟลเดอร์งานต่อหนึ่งภารกิจ สร้างโฟลเดอร์ทำงานเฉพาะที่มีเฉพาะไฟล์ที่งานนั้นต้องใช้ แล้วเชื่อมต่อเฉพาะโฟลเดอร์นั้น — ห้ามเชื่อมต่อโฟลเดอร์แม่ Desktop หรือโฟลเดอร์ sync ของ Drive ทั้งหมด
  • ห้ามใช้โฟลเดอร์ deliverable ของลูกค้า deliverable ลับฉบับสมบูรณ์ต้องเก็บไว้ในตำแหน่งที่ได้รับการป้องกันของตัวเองซึ่งไม่เชื่อมต่อกับ Cowork เลย ทำงานร่างในโฟลเดอร์งาน แล้วย้ายฉบับสมบูรณ์ออกเมื่อเสร็จ
  • ตรวจสอบก่อนเชื่อมต่อ ก่อนเชื่อมต่อโฟลเดอร์ใด ให้ตรวจสอบก่อนว่า ไฟล์ใดที่ระบุชื่อแหล่งข่าว เจ้าหน้าที่ภาคสนาม หรือบุคคลเป้าหมายที่งานนั้นไม่จำเป็นต้องใช้ ให้ย้ายออกก่อน
  • ตรวจสอบผลงานที่ Cowork สร้างขึ้น ก่อนย้ายไฟล์ต่อไป คุณคือด่านตรวจคุณภาพ เช่นเดียวกับการตรวจงานของนักวิเคราะห์รุ่นน้อง

กฎการเชื่อมต่อเครื่องมือ

ตัวเชื่อมต่อสถานะเหตุผล
Email (Gmail / Outlook)ห้ามใช้Email เก็บการติดต่อกับลูกค้า สัญญา และข้อมูลติดต่อแหล่งข่าว ห้ามเชื่อมต่อเครื่องมือ AI ใด ๆ กับ email ของ ENVISAGE โดยเด็ดขาด

ระบุบทบาท แหล่งข้อมูล และรูปแบบผลลัพธ์ — เพียงครั้งเดียว

အပိုင်း ၁

Claude & Claude Cowork အသုံးပြုမှု အကောင်းဆုံးလမ်းညွှန်

Claude သည် ကျွန်ုပ်တို့၏ အဓိက AI လုပ်ငန်းကိရိယာ ဖြစ်သည်။ တွေးခေါ်ဆင်ခြင်ရန်အတွက် Chat ကိုသုံးပြီး၊ လက်တွေ့ဆောင်ရွက်ရန်အတွက် Cowork ကိုသုံးပါ — ထို့ပြင် သုတေသနလုပ်ငန်းစဉ်တစ်ခုစီအတွက် context ကို တစ်ကြိမ်တည်း စနစ်တကျ ထားရှိပါက context ကို ထပ်ခါထပ်ခါ ရှင်းပြရန် token ကုန်ကျစရာ မလိုတော့ပါ။

Chat နှင့် Cowork — မှန်ကန်သော mode ကို ရွေးချယ်ခြင်း

Claude Chat

တွေးခေါ်ခြင်း၊ မူကြမ်းရေးခြင်း နှင့် ခွဲခြမ်းစိတ်ဖြာခြင်းအတွက်

  • သုတေသနမေးခွန်း သို့မဟုတ် ခွဲခြမ်းစိတ်ဖြာမှု framework တစ်ခုကို ဆင်ခြင်ခြင်း
  • report အပိုင်းများ၊ အကျဉ်းချုပ်များ၊ briefing များ မူကြမ်းရေးခြင်းနှင့် တည်းဖြတ်ခြင်း
  • မြန်မာ၊ ထိုင်း၊ ရှမ်း နှင့် အခြားဘာသာစကားများမှ ဘာသာပြန်ချက်များကို ပြန်လည်စစ်ဆေးခြင်း
  • သင်ဖော်ပြထားသော သို့မဟုတ် excerpt အဖြစ်ကူးထည့်ထားသော ဒေတာရှိ ပုံစံများကို အဓိပ္ပာယ်ဖွင့်ခြင်း
  • အင်တာဗျူးမှတ်စု သို့မဟုတ် လယ်ကွင်းအကဲဖြတ်ချက်များကို (အမည်ဖုံးကွယ်ထားသည့်) ဖွဲ့စည်းပြခြင်း
Claude Cowork

ဖိုင်များကို တိုက်ရိုက်လုပ်ဆောင်ရန်အတွက်

  • deliverable များ တည်ဆောက်ခြင်း — Word report, Excel workbook, slide deck
  • ဒေတာဖိုင်များကို သန့်ရှင်းခြင်း၊ ပြန်လည်ဖွဲ့စည်းခြင်း နှင့် ပြောင်းလဲခြင်း
  • အဆင့်ဆင့်လုပ်ဆောင်ရမည့် လုပ်ငန်းများ — ဥပမာ "ဤ CSV ဖိုင် ၁၂ ခုကို လုပ်ဆောင်ပြီး summary table တစ်ခု ထုတ်ပေးပါ"
  • project folder တစ်ခုအတွင်း ဖိုင်များစွာတွင် ထပ်တလဲလဲလုပ်ရသည့် document အလုပ်များ
  • နမူနာဖိုင်များနှင့် script (Python, SQL) များ ရေးသားခြင်းနှင့် စမ်းသပ်ခြင်း

အခြေခံစည်းမျဉ်း — အဖြေသည် ဖိုင်တစ်ခုထဲတွင် ရှိသင့်လျှင် Cowork မှ စတင်ပါ။ အဖြေသည် သင့်စိတ်ထဲတွင်သာ ရှိသင့်လျှင် Chat မှ စတင်ပါ။ ဖိုင်အကြောင်းအရာများကို Chat ထဲသို့ ကူးထည့်ခြင်းသည် token ကို နှစ်ကြိမ် ဖြုန်းစေသည် — တစ်ကြိမ်မှာ ကူးထည့်စဉ်၊ နောက်တစ်ကြိမ်မှာ Claude မှ ထိုအရာကို ပြန်ရေးပြသောအခါ။

သုတေသနလုပ်ငန်းစဉ်တစ်ခုစီအတွက် Project တစ်ခုစီ ထားရှိခြင်း

Claude Projects များသည် context ကို ဆက်လက်တည်ရှိစေသောကြောင့် စကားဝိုင်းတိုင်းတွင် ထပ်ခါထပ်ခါ ရှင်းပြရန် token ကုန်ကျမှု ရပ်တန့်စေသည်။

ရေရှည်လုပ်ငန်း သို့မဟုတ် လုပ်ငန်းစဉ်တစ်ခုစီအတွက် Project တစ်ခုစီ ဖန်တီးပါ — ဥပမာ monitoring-and-analysis လုပ်ငန်း၊ supply-chain အန္တရာယ်လေ့လာမှု၊ SAM ထုတ်ကုန်ဖွံ့ဖြိုးမှု သို့မဟုတ် မြေပုံရေးဆွဲမှုပရောဂျက် စသည်တို့။ Project တစ်ခုစီ၏ knowledge/instructions အပိုင်းတွင် တည်ငြိမ်သော context ကို တစ်ကြိမ်တည်း သိမ်းဆည်းထားပါ —

  • လုပ်ငန်း၏ ရည်ရွယ်ချက်၊ နယ်ပယ် နှင့် ဖောက်သည်အတည်ပြုထားသော ဝေါဟာရများ (tier စည်းမျဉ်းများ ခွင့်ပြုသည့်ပမာဏအထိ အမည်ဖုံးကွယ်ထားသည်)
  • ကျွန်ုပ်တို့၏ methodology မှတ်စုများ — ဥပမာ ရင်းမြစ်များကို မည်သို့စစ်ဆေးသည်၊ ကျွန်ုပ်တို့၏ scoring rubric၊ visualization စံနှုန်းများ
  • output template များ — report ဖွဲ့စည်းပုံ၊ ကိုးကားပုံစံ၊ standard caveat များ
  • သက်ဆိုင်ရာ dataset များ၏ schema ဖော်ပြချက် (ဖွဲ့စည်းပုံသာ၊ ဒေတာ row များ မပါ)
ကောင်းသောနမူနာ — context သည် Project ထဲတွင် ရှိနေသည်

Project instructions တွင် report template နှင့် methodology ပါဝင်ပြီးသားဖြစ်သည်။ စကားဝိုင်းသစ်တိုင်း — "ပူးတွဲထားသော excerpt ကိုသုံးပြီး Q2 လမ်းကြောင်း အပိုင်းကို မူကြမ်းရေးပါ" ဟု prompt token ခန့် ၂၀၀ ခန့်ဖြင့်သာ ရေးရသည်။

ဖြုန်းတီးမှု — context ကို အကြိမ်တိုင်း ပြန်ကူးထည့်ခြင်း

စကားဝိုင်းတိုင်းသည် နောက်ခံအချက်အလက် စကားလုံး ၃,၀၀၀ ၊ template အပြည့်အစုံ နှင့် ယခင်သုံးလပတ် report ကို တကယ့်တောင်းဆိုချက်မတိုင်မီ ကူးထည့်ခြင်းဖြင့် စတင်သည်။

Cowork Folder စည်းကမ်း

Cowork သည် သင်တိကျစွာ ချိတ်ဆက်ပေးသော folder များပေါ်တွင်သာ လုပ်ဆောင်သည်။ ထိုချိတ်ဆက်မှုကို ဖိုင်ဗီရိုတစ်လုံးလုံး လွှဲပြောင်းပေးသကဲ့သို့ သဘောထားပါ —

  • လုပ်ငန်းတစ်ခုလျှင် folder တစ်ခု။ ထိုလုပ်ငန်းအတွက် လိုအပ်သည့်ဖိုင်များသာ ပါဝင်သော အထူးလုပ်ငန်း folder ကို ဖန်တီးပါ။ ထို folder ကိုသာ ချိတ်ဆက်ပါ — parent folder၊ Desktop သို့မဟုတ် Drive sync folder တစ်ခုလုံးကို လုံးဝ မချိတ်ဆက်ပါနှင့်။
  • ဖောက်သည် deliverable folder များ မပါ။ နောက်ဆုံး လျှို့ဝှက် deliverable များသည် Cowork နှင့် လုံးဝ မချိတ်ဆက်ရသော ကာကွယ်ထားသည့် နေရာသီးသန့်တွင် ရှိရမည်။ မူကြမ်းများကို လုပ်ငန်း folder တွင် လုပ်ဆောင်ပြီး ပြီးစီးသောအခါ နောက်ဆုံးဗားရှင်းကို ထုတ်ယူပါ။
  • ချိတ်ဆက်မီ ရှင်းလင်းပါ။ folder တစ်ခုကို မချိတ်ဆက်မီ စစ်ဆေးပါ — ထိုလုပ်ငန်းအတွက် မလိုအပ်သော ရင်းမြစ်၊ လယ်ကွင်းဝန်ထမ်း သို့မဟုတ် စိတ်ဝင်စားဖွယ်ပုဂ္ဂိုလ်များ အမည်ဖော်ပြထားသော ဖိုင်မှန်သမျှကို ဦးစွာ ရွှေ့ထုတ်ပါ။
  • Cowork မှ ထုတ်လုပ်ထားသော ရလဒ်ကို စစ်ဆေးပါ ဖိုင်များကို ရှေ့ဆက်မထားမီ။ သင်သည် junior analyst တစ်ဦး၏ အလုပ်ကို စစ်ဆေးသကဲ့သို့ ပင် အရည်အသွေးထိန်းသိမ်းသူ ဖြစ်သည်။

Connector စည်းမျဉ်းများ

Connectorအခြေအနေအကြောင်းရင်း
Email (Gmail / Outlook)တားမြစ်ထားသည်Email တွင် ဖောက်သည်ဆက်သွယ်မှု၊ စာချုပ်များ နှင့် ရင်းမြစ်ဆက်သွယ်ရန် အချက်အလက်များ ပါဝင်သည်။ AI ကိရိယာ မည်သည်မဆို ENVISAGE ၏ email နှင့် ဘယ်တော့မှ မချိတ်ဆက်ရပါ။
Google Driveကန့်သတ်ထားသည်ကြိုတင်အတည်ပြုထားသော လုပ်ငန်း folder အထူးများကိုသာ ခွင့်ပြုသည် — root access ကို ဘယ်တော့မှ ခွင့်မပြုပါ။ ဖောက်သည် deliverable နှင့် HR/ဘဏ္ဍာရေး folder များကို အမြဲထုတ်ပယ်ထားသည်။ project lead ၏ အတည်ပြုချက် လိုအပ်သည်။
Document / spreadsheet မျှဝေခြင်းတားမြစ်ထားသည်အတွင်းပိုင်း Docs/Sheets ၏ တိုက်ရိုက်လင့်ခ်များကို AI ကိရိယာ မည်သည်နှင့်မျှ မမျှဝေပါနှင့်၊ လျှို့ဝှက်စာတမ်းအပြည့်အစုံကိုလည်း မကူးထည့်ပါနှင့်။ sanitize လုပ်ထားသော excerpt များ သို့မဟုတ် Cowork လုပ်ငန်း folder များကို အသုံးပြုပါ။
local task folder များ (Cowork)အတည်ပြုပြီးအထက်ပါ စည်းကမ်းအတိုင်း လုပ်ငန်းနှင့် သက်ဆိုင်သောဖိုင်များသာ ပါဝင်သည့် ရည်ရွယ်ချက်အတွက် သီးသန့်ဖန်တီးထားသော folder များ။
